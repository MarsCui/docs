# 配置账号与权限

::: info 文档信息
版本：v1.0
更新日期：2026-07-13
:::

## 流程概述

本文覆盖一次账号与权限配置端到端流程：先确认租户和账号信息，再准备角色，添加或核对成员，按需创建访问凭证，最后通过操作日志验证变更是否完成。涉及 API 调用账号时，还需要确认 Key 类型、有效期、额度或流控影响。

#### 新手理解

账号与权限配置像给新成员办理门禁：先确认成员属于哪个租户，再决定他拿哪类角色权限，之后检查他能不能进入目标页面，最后在操作日志里确认配置记录。

#### 术语速查

| 术语 | 含义 | 处理建议 |
| --- | --- | --- |
| 租户 | 成员和权限归属的业务主体。 | 先确认租户和管理员。 |
| 成员 | 需要访问平台的账号。 | 添加前确认邮箱、手机号和身份。 |
| 角色 | 成员可访问菜单和操作的权限集合。 | 先建角色再分配成员。 |
| Key | 接口调用使用的访问凭证。 | 按用途拆分并设置有效期。 |
| 操作日志 | 记录成员、角色和配置变更的审计线索。 | 变更后用它复核。 |

## 前提条件

1. 当前账号具备设置运营管理员权限。
2. 已明确目标租户、目标成员、需要授权的角色和业务用途。
3. 若涉及接口调用，已确认需要 Model API Key 还是 System API AK/SK。
4. 若涉及新增成员、角色授权、Key 创建、登录策略变更、删除、重置或 Key 轮换，已确认审批依据和回退方案。

## 参数说明

| 字段名称 | 是否必填 | 字段类型 | 示例 | 说明 |
| --- | --- | --- | --- | --- |
| 目标租户 | 必填 | 文本 | `<tenant_name>` | 成员、角色和权限归属范围。 |
| 目标成员 | 必填 | 文本 | `<member_account>` | 需要添加、核对或授权的账号。 |
| 角色 | 必填 | 枚举 | `<role_name>` | 成员可访问菜单和操作权限集合。 |
| Key 类型 | 条件必填 | 枚举 | `Model API Key` | 涉及接口调用时需要确认。 |
| 登录策略 | 条件必填 | 配置项 | `验证码` | 登录、注册或找回异常时需要核对。 |
| 操作日志时间 | 条件必填 | 时间范围 | `变更时间窗口` | 复核变更记录时用于筛选。 |

## 流程步骤

#### 确认租户

1. 进入 `设置 > 运营管理员 > 租户&设置 > 租户`。
2. 按租户名称、管理员信息或页面筛选条件定位目标租户。
3. 确认租户状态、管理员、角色范围和业务归属。
4. 如租户不存在或归属不清晰，先不要继续授权。

#### 准备角色

1. 进入 `设置 > 运营管理员 > 成员与角色 > 角色`。
2. 查看已有角色能否满足业务需要。
3. 如需新增或修改角色，先确认菜单权限、操作权限和适用成员范围。
4. 删除、扩大或收窄角色前，确认是否仍有成员正在使用。

#### 核对成员

1. 进入 `设置 > 运营管理员 > 成员与角色 > 成员`。
2. 搜索目标成员，确认成员状态、邮箱、手机号、角色和租户归属。
3. 如成员不存在，按页面入口添加成员。
4. 如成员无法登录，先检查状态、角色和登录配置，再考虑重置密码。

#### 配置个人访问凭证

1. 成员需要接口调用时，进入 `设置 > 个人 > 我的 Keys`。
2. 根据调用类型选择 `Model API Key` 或 `System API AK/SK Pair`。
3. 设置用途、有效期、权限范围和额度限制。
4. 创建或轮换后，确认调用方已通过受控渠道替换配置。

#### 复核登录策略

1. 如成员登录、验证码或账号找回异常，进入 `设置 > 运营管理员 > 系统设置 > 登录配置`。
2. 检查登录安全、注册验证码、账号找回和第三方登录配置。
3. 调整登录策略前，先确认影响用户范围、验证方式和回退方案。

#### 查看操作日志

1. 进入 `设置 > 运营管理员 > 消息与日志 > 操作日志`。
2. 按时间、操作者、操作对象、请求方法和结果筛选。
3. 确认新增成员、角色授权、Key 创建或登录策略变更已记录。
4. 如日志缺失，扩大时间范围并确认当前账号日志权限。

## 结果校验

| 检查项 | 成功表现 | 异常时处理 |
| --- | --- | --- |
| 租户明确 | 能定位目标租户和管理员 | 继续配置角色 |
| 角色可用 | 角色权限覆盖目标菜单和操作 | 分配给成员 |
| 成员可登录 | 成员状态正常且角色已绑定 | 进入目标业务页面验证 |
| Key 可使用 | Key 类型、有效期、权限范围和额度符合调用需求 | 通知调用方替换配置 |
| 日志可追溯 | 操作日志能查到本次配置记录 | 归档脱敏后的变更记录 |

## 失败分支与排查路径

| 失败现象 | 优先排查页面 | 下一步 |
| --- | --- | --- |
| 成员看不到菜单 | [角色](../../operator/members-roles/roles/) | 检查角色权限和成员绑定 |
| 成员无法登录 | [成员](../../operator/members-roles/members/) | 检查状态、密码和登录配置 |
| 验证码收不到 | [登录配置](../../operator/system-settings/login-properties/) | 检查验证码和通知配置 |
| Key 调用失败 | [我的 Keys](../../user/personal/my-keys/) | 检查 Key 状态、额度和过期时间 |
| 变更无法追溯 | [操作日志](../../operator/activity-notifications/operation-logs/) | 放宽时间范围并核对日志权限 |

## 踩坑提示

- 先确认租户再分配角色，错误租户下的授权不会解决目标页面不可见问题。
- 新增成员、角色授权、Key 创建和登录策略变更都属于高风险配置链路，应先确认审批依据和影响范围。
- 高权限角色不要长期给临时成员保留，授权完成后应按最小权限复核。
- Key 创建后只代表凭据生成成功，调用方仍需替换配置并确认额度、过期时间和接口权限。
- 操作日志缺失时不要立即重复操作，先放宽时间范围并确认日志权限。

## 常见问题

#### 成员已经添加但仍看不到菜单怎么办？

**问题现象：**

成员可以登录，但目标菜单没有显示。

**可能原因：**

成员没有绑定正确角色，角色权限不包含目标菜单，或成员处于错误租户上下文。

**处理方式：**

先核对成员租户和角色绑定；再进入角色页确认菜单权限；必要时重新登录刷新权限。

#### Key 创建后仍调用失败怎么办？

**问题现象：**

调用方已拿到 Key，但接口仍返回鉴权或额度相关错误。

**可能原因：**

Key 类型不匹配、权限范围不足、额度不足、Key 已过期，或调用方仍使用旧凭证。

**处理方式：**

核对 Key 类型、状态、过期时间、权限范围和额度限制；让调用方确认配置已替换；仍失败时结合操作日志和业务接口错误继续排查。

- 新增成员、角色授权、Key 创建、登录策略变更、删除成员、重置密码和轮换 Key 前，应确认通知、窗口和回退方式。

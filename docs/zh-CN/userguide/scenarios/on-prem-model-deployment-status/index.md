# 场景概览-On-Prem 模型部署与状态检查

本场景介绍用户如何基于推理模板创建模型实例，并从实例、作业、节点和设备四个层面判断部署是否成功。对于 4 张 NPU 卡，本场景负责验证模板是否真的申请到预期卡数。

## 适用角色

- 模型提供方
- 最终用户
- 参与排障的运营管理员

## 场景目标

- 使用 NPU 推理模板创建模型实例。
- 判断实例处于创建、运行、失败还是停止状态。
- 确认实际分配的 NPU 数量与模板一致。

## 开始前准备

- 已有可用推理模板和足够的租户配额。
- 目标集群有足够的空闲 NPU。
- 已准备模型文件、镜像和启动参数。

## 完成标志

- 模型实例进入运行状态。
- 作业实际占用卡数与模板规格一致。
- 服务健康检查和访问入口正常。

# 用户管理

## 文档说明

本文档说明管理员如何在管理门户中维护用户，包括用户创建、角色授权、账号状态控制、重置操作和审计校验。

## 开始前准备

- 确认当前管理员具备用户管理权限。
- 确认目标租户、组织、项目或资源范围。
- 准备用户姓名、邮箱或登录标识、所属组织和联系方式。
- 确认需要分配的角色以及分配原因。
- 确认账号创建后是否需要立即启用。

## 用户生命周期

| 阶段 | 主要操作 | 校验方式 |
|------|----------|----------|
| 创建 | 新增用户并填写必要资料。 | 用户记录出现在列表中。 |
| 授权 | 分配角色和资源范围。 | 可见菜单和资源范围符合预期。 |
| 通知 | 通过合规渠道告知登录入口和初始访问方式。 | 用户可完成首次登录。 |
| 维护 | 按职责变化更新资料、角色、范围或状态。 | 审计日志记录本次变更。 |
| 禁用 | 用户离开授权范围时关闭访问能力。 | 用户无法继续访问受限功能。 |

## 常见操作

### 创建用户

1. 进入 `管理门户 > 用户管理`。
2. 点击新建或添加用户入口。
3. 填写姓名、邮箱、组织、联系方式等必要资料。
4. 选择用户所属租户或组织范围。
5. 确认账号是否创建后立即启用。
6. 保存用户。

保存后，在用户列表中搜索该用户，确认资料和状态正确。

### 分配角色和资源范围

1. 打开目标用户记录。
2. 进入角色或权限配置区域。
3. 选择符合用户职责的角色模板。
4. 根据平台配置分配租户、组织、项目、资源或菜单范围。
5. 保存变更。
6. 让用户重新登录；如果平台提供权限刷新入口，也可以执行权限刷新。

不要只为了显示某个缺失菜单就直接分配管理员或宽泛运营角色。应先确认问题来自角色、资源范围还是菜单配置。

### 启用、禁用或重置访问

谨慎使用状态和重置类操作：

| 操作 | 适用场景 | 保存前检查 |
|------|----------|------------|
| 启用 | 新建用户或曾被禁用的用户需要恢复访问。 | 确认用户仍属于授权组织。 |
| 禁用 | 用户离开项目、职责变化或需要临时关闭访问。 | 确认没有进行中的交付或审批任务依赖该账号。 |
| 重置访问 | 需要处理登录恢复或账号排障。 | 确认请求人身份，并记录操作原因。 |

## 验证清单

- [ ] 用户可正常登录。
- [ ] 分配权限符合预期角色和资源范围。
- [ ] 用户只能看到预期菜单。
- [ ] 用户无法访问授权范围外资源。
- [ ] 创建、更新、禁用或重置操作已记录在审计日志中。
- [ ] 高风险变更有审批原因或可追溯请求记录。

## 常见问题

| 问题 | 可能原因 | 处理方法 |
|------|----------|----------|
| 用户无法登录 | 账号被禁用、登录标识错误，或密码 / 重置流程未完成 | 确认账号状态，并按合规流程重新重置 |
| 用户看不到预期菜单 | 缺少角色或菜单权限 | 确认角色分配和菜单授权 |
| 用户能看到菜单但没有数据 | 未授权资源范围 | 确认租户、组织、项目或资源授权 |
| 角色变更未生效 | 权限缓存或当前会话未刷新 | 让用户退出后重新登录，或使用平台权限刷新入口 |
| 审计日志缺失 | 操作未保存、日志同步延迟，或当前账号缺少审计可见权限 | 重新检查操作结果，并确认审计权限 |

## 功能概述

`框架配置` 用于维护推理或运行框架，包括框架名称、版本、镜像、启动命令、端口、环境变量和支持的模型类型。框架决定模型实例启动后由什么服务进程承载。

| 项目 | 内容 |
| --- | --- |
| 适用角色 | 运营方 |
| 导航路径 | 模板 > 框架配置 |
| 页面路由 | `/powerone/fast-build-v2/frameworks` |
| 管理对象 | 推理框架、框架版本、运行镜像、端口、启动命令和环境变量 |
| 典型用途 | 为推理模板提供可复用的运行框架 |

### 新手理解

框架配置像服务启动器的说明书，提前写清镜像、启动命令、端口和健康检查，后续模板才能稳定把模型服务拉起来。

### 术语速查

| 术语 | 说明 |
| --- | --- |
| 推理框架 | 承载模型服务的运行框架，例如 vLLM、TGI 或自研服务框架。 |
| 启动命令 | 容器启动后执行的命令，决定服务如何加载模型。 |
| 端口 | 框架对外提供服务的监听端口。 |
| 环境变量 | 影响框架运行行为的键值配置。 |
| 参数触发条件 | 根据模型、规格或用户输入动态显示或启用参数的规则。 |

## 前提条件

1. 框架镜像已准备并能被目标集群拉取。
2. 已明确框架支持的模型类型、量化方式、端口和启动参数。
3. 已确认默认命令、环境变量和可暴露参数不会泄露敏感信息。
4. 当前账号具备模板管理权限。

## 添加或维护框架

### 操作前确认

1. 已确认框架所需运行镜像、基础依赖和启动命令。
2. 已确认服务监听端口、健康检查路径和探活方式。
3. 已确认框架适配的模型类型、推理协议和资源规格。
4. 镜像仓库凭据、环境变量和启动参数均已脱敏。

### 操作步骤

1. 进入 `模板 > 框架配置`。
2. 点击新增、编辑或页面提供的维护入口。
3. 在基础信息 Tab 中填写框架名称、版本、描述和支持场景。
4. 在运行配置 Tab 中选择镜像、端口、启动命令和环境变量。
5. 在参数配置 Tab 中维护用户可填写参数、默认值和触发条件。
6. 保存后在推理模板中引用该框架。

### 参数说明

| 字段名称 | 是否必填 | 字段类型 | 示例 | 说明 |
| --- | --- | --- | --- | --- |
| 框架名称 | 是 | 文本 | `vllm-runtime` | 用于在推理模板中选择运行框架。 |
| 运行镜像 | 是 | 镜像地址 | `registry.example.com/runtime/vllm:1.0` | 框架启动时使用的容器镜像。 |
| 启动命令 | 是 | 命令行 | `python -m vllm.entrypoints.openai.api_server` | 容器启动后执行的服务命令。 |
| 服务端口 | 是 | 数字 | `8000` | 平台探测和路由流量的监听端口。 |
| 健康检查 | 条件必填 | 路径 / 命令 | `/health` | 用于判断框架服务是否启动成功。 |
| 环境变量 | 否 | 键值对 | `HF_HOME=/models/cache` | 传入框架运行时的环境配置。 |

### 踩坑提示

- 启动命令必须能以前台进程运行，避免容器启动后立即退出。
- 服务端口要与框架实际监听端口一致，否则健康检查通过不了。
- 运行镜像内必须包含框架依赖、模型加载依赖和必要系统库。

### 结果校验

1. 框架出现在列表中。
2. 推理模板可以选择该框架。
3. 使用该框架创建测试服务时，镜像可拉取、命令可执行、端口可访问。

## 常见问题

### 推理模板中选不到框架

**问题现象：**

配置推理模板时，框架下拉列表没有目标框架。

**可能原因：**

- 框架未启用或版本不可用。
- 框架支持的模型类型与当前模型不匹配。
- 框架镜像或配置未通过校验。

**处理方式：**

1. 检查框架状态和版本。
2. 确认模型类型、量化方式和框架支持范围。
3. 保存框架配置后重新进入推理模板。

### 服务启动后端口不可访问

**问题现象：**

模型实例运行中，但访问服务端口失败。

**可能原因：**

- 框架监听端口与模板端口不一致。
- 启动命令没有绑定 `0.0.0.0`。
- 容器启动成功但服务进程异常退出。

**处理方式：**

1. 核对框架端口和推理模板端口。
2. 检查启动命令和日志。
3. 确认服务监听地址和健康检查配置。

# 场景概览 - 模型消费与收益

本场景介绍调用方如何核对模型用量，模型提供方如何核对客户调用和收益，以及双方如何按相同账期完成对账。

## 适用角色

- 平台用户、模型提供方

## 场景目标

- 用量数据能够对应到调用日志。
- 收益数据能够回溯到有效调用和价格规则。
- 异常能够定位到模型、调用方、账期或计费配置。

## 场景流程

**主线：** 统一对账范围 → 核对调用日志 → 核对模型用量 → 核对消费与收益

| 阶段 | 核心结果 |
| --- | --- |
| 1. 统一范围 | 用户、模型、供应方、币种和时间范围一致 |
| 2. 核对调用 | 成功、失败、Token 或时长记录可追溯 |
| 3. 核对用量 | 汇总用量与同一范围内的有效调用对应 |
| 4. 核对结算 | 用户消费、提供方收益和发布价格规则能够解释彼此 |

## 开始前准备

- 明确模型、时间范围、账期和计费单位。
- 准备脱敏后的请求时间或调用标识。

## 推荐阅读顺序

1. 查看调用日志
2. 查看模型用量
3. 提供方查看模型收益
4. 对照计费配置完成核对

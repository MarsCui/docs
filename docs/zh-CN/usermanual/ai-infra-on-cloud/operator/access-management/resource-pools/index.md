# 资源池

## 功能概述

`资源池` 用于维护云账号资源池、地域、实例规格、库存和同步状态，支撑多云调度、资源授权和模型部署流程。

| 项目 | 内容 |
| --- | --- |
| 适用角色 | 运营方 |
| 导航路径 | 接入管理 > 资源池 |
| 页面路由 | /operator/access-management/resource-pools |
| 管理对象 | 云账号资源池、地域、实例规格、库存和同步状态 |
| 典型用途 | 按云账号和地域组织可部署资源 |

### 新手理解

资源池像把云账号里的机器、加速卡和存储按地域装进货架。用户部署时看到的是货架里的可用资源，而不是云厂商账号本身。

### 术语速查

| 术语 | 说明 |
| --- | --- |
| 资源池 | 按云账号、地域和资源类型组织的资源集合。 |
| 库存 | 当前可用于部署的实例或加速卡容量。 |
| 同步任务 | 平台从云厂商拉取资源状态的任务。 |
| 资源边界 | 资源池开放给哪些租户或业务地域使用。 |

## 前提条件

1. 云账号已接入并通过校验。
2. 目标地域资源已同步。
3. 资源池容量、资源类型和启用状态已确认。
## 页面说明

页面用于查看和维护从云账号同步来的资源池，包括云平台、地域、资源类型、容量水位、启用状态和授权状态。运营方应按地域和业务用途拆分资源池，保证后续授权和调度策略可控。

用于查看资源池地域、容量、水位和启用状态。

## 主要操作

### 操作步骤

1. 进入 `接入管理 > 资源池`。
2. 按云平台、地域、资源类型或状态筛选资源池。
3. 同步或新增资源池后核对容量、规格和资源类型。
4. 为资源池设置启用状态、用途说明和可授权范围。
5. 进入授权页面将资源池开放给租户或业务地域。

关键步骤截图：

![编辑资源池名称](./images/edit-name.png)

编辑名称前确认不会影响授权和用户侧识别。

### 参数说明

| 字段名称 | 是否必填 | 字段类型 | 示例 | 说明 |
| --- | --- | --- | --- | --- |
| 资源池名称 | 是 | 文本 | `gpu-cn-shanghai-prod` | 建议体现云平台、地域和用途。 |
| 云账号 | 是 | 下拉选择 | `prod-cloud-account` | 资源池所属云账号。 |
| 地域 | 是 | 下拉选择 | `cn-shanghai` | 云资源所在地域。 |
| 资源类型 | 是 | 多选 | `GPU / CPU` | 资源池可承载的资源类型。 |
| 启用状态 | 是 | 枚举 | `启用` | 控制是否可被授权和调度。 |

### 踩坑提示

- 资源池容量正常不代表租户已有权限，还需要完成授权。
- 跨地域资源池不要混用，避免部署时选择到不可达资源。
- 停用资源池前先确认是否存在运行中部署实例。

### 结果校验

1. 资源池列表显示容量、地域和启用状态。
2. 授权页面可以选择目标资源池。
3. 用户部署页能在授权范围内看到对应资源。

## 常见问题

### 资源池不可授权

**问题现象：**

在租户授权或业务地域授权页面找不到目标资源池。

**可能原因：**

- 资源池未启用。
- 资源池所属云账号校验失败。
- 资源池地域与授权对象不匹配。

**处理方式：**

1. 确认资源池启用状态。
2. 检查云账号校验和同步任务。
3. 核对授权对象的云平台和地域。

### 资源池容量显示异常

**问题现象：**

资源池容量为 0 或与云厂商控制台不一致。

**可能原因：**

- 同步任务延迟。
- 云厂商资源标签或规格不符合识别规则。
- 资源已被其他业务占用。

**处理方式：**

1. 刷新或重新触发同步。
2. 核对资源标签、规格和状态。
3. 结合云厂商控制台检查实际可用容量。

## 后续操作

1. 配置租户云授权。
2. 配置业务地域授权。
3. 维护调度策略和部署资产。

## 注意事项

- 资源池容量正常不代表租户已获得授权。
- 跨地域资源池不要混用。
- 停用资源池前确认没有运行中部署。

# 场景概览 - 本地运行镜像与存储底座

本场景帮助运营方在本地集群接入后，配置镜像仓库、运行镜像和块/文件/对象存储，为在线 IDE、训练作业和推理服务建立可复用的运行底座。

## 适用角色

- 平台运营方、本地算力资源管理员、存储管理员

## 场景目标

- 镜像组件能够被平台和目标集群访问并正常同步。
- 运行镜像具备明确版本、硬件环境和用途标签。
- 块、文件和对象存储按工作负载需要接入正确地域。
- 测试工作负载能够拉取镜像并完成存储创建、挂载或访问。

## 场景流程

**主线：** 确认集群与网络 -> 注册镜像组件 -> 同步运行镜像 -> 注册存储组件 -> 绑定地域 -> 使用测试工作负载验证

| 阶段 | 核心结果 |
| --- | --- |
| 1. 确认依赖 | 集群、网络、仓库和存储服务可达 |
| 2. 准备镜像 | 镜像组件正常，目标镜像可见且版本明确 |
| 3. 准备存储 | 所需存储组件已注册并绑定到目标地域 |
| 4. 联合验证 | 作业可拉取镜像并按预期使用存储 |

## 开始前准备

- 已完成[本地算力纳管](../on-prem-compute-onboarding/)。
- 已准备镜像仓库、存储服务、网络和认证材料。
- 已明确目标地域、集群、租户隔离、容量和回收策略。
- 认证材料只在平台配置页填写，不写入文档或工单。

## 推荐阅读顺序

1. 先注册镜像组件并验证同步。
2. 再同步或上传带明确版本标签的运行镜像。
3. 按工作负载选择块、文件或对象存储组件。
4. 绑定地域后，通过测试工作负载做端到端验证。

## 完成检查

> **用途：** 以下检查是本场景的退出条件，用于确认运行环境和数据存储已经真正被目标集群使用，而不只是列表中出现了配置记录。

| 检查项 | 通过标准 |
| --- | --- |
| 1 | 镜像组件状态正常，并绑定到正确地域或集群。 |
| 2 | 目标运行镜像可见，版本、架构、硬件环境和用途清晰。 |
| 3 | 所需存储组件状态、容量、访问策略和地域绑定正确。 |
| 4 | 测试工作负载能拉取镜像并完成存储创建、挂载、读写或对象访问。 |
| 5 | 删除、回收、备份和凭据轮换责任人已明确。 |

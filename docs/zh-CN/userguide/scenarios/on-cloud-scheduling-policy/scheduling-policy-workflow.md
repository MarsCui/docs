# 配置并验证多云调度策略

本任务用于为一个租户或业务地域配置首选资源池和回退策略，并通过测试部署验证。

## 适用角色

- 平台运营方、多云资源管理员、部署策略管理员

## 开始前准备

1. 确认云平台、云账号、资源池状态正常。
2. 确认目标租户和业务地域已获得首选、回退资源池授权。
3. 明确优先级、容量阈值、回退条件及跨地域约束。

## 操作流程

### 1. 检查接入与授权基础

进入[接入总览](../../../usermanual/ai-infra-on-cloud/operator/access-workbench/access-overview/)，确认目标云账号和资源池状态；再核对租户云授权、业务地域授权和资源池可用容量。缺少授权时不要直接创建调度策略。

![检查多云接入和资源状态](../../../usermanual/ai-infra-on-cloud/operator/access-workbench/access-overview/images/access-overview-list.png)

### 2. 配置调度策略

进入[调度策略](../../../usermanual/ai-infra-on-cloud/operator/scheduling-governance/policies/)，设置策略名称、适用租户或业务地域、首选资源池、优先级、容量阈值、回退资源池和约束。跨地域回退前必须确认时延、成本和数据合规影响。

![配置策略范围、优先级和资源池](../../../usermanual/ai-infra-on-cloud/operator/scheduling-governance/policies/images/policies-list.png)

### 3. 执行测试部署

使用一个低风险模型执行测试部署，记录所选资源池、地域、部署状态和事件。先验证首选资源池；在可控条件下验证容量不足时的回退或明确失败行为。

### 4. 复核结果并建立巡检

确认部署结果与策略一致，并记录实际成本、启动时延和资源占用。定期复核资源池容量、授权关系和策略优先级，避免基础条件变化后策略失效。

## 完成检查

> **用途：** 以下检查确认策略已通过测试部署证明，而不是只完成页面保存。任一项失败时，应先恢复授权、容量或策略配置。

| 检查项 | 通过标准 |
| --- | --- |
| 基础授权 | 首选和回退资源池对目标范围均可用。 |
| 策略配置 | 适用范围、优先级、阈值、回退和约束正确。 |
| 首选验证 | 正常容量下，测试部署命中首选资源池。 |
| 回退验证 | 首选不可用时，按策略回退或返回明确原因。 |
| 运营复核 | 容量、成本、时延和合规影响已记录。 |

## 常见失败分支

| 现象 | 优先检查 |
| --- | --- |
| 部署未命中首选资源池 | 容量、启用状态、业务地域授权和策略优先级 |
| 首选不可用时没有回退 | 回退资源池配置、授权、容量和跨地域约束 |
| 策略保存后不影响已有实例 | 策略通常影响后续调度，已有实例需单独处理 |
| 回退后成本或时延异常 | 资源池价格、地域距离、网络和 SLA 约束 |

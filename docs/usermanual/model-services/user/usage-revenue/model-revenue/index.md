# Model Earnings

## Preface

| Item | Content |
|------|------|
| Applicable Role | Model provider (tenant that provides models and earns revenue) |
| Navigation Path | Usage & Earnings > Model Earnings |
| Feature Positioning | Entry for model providers to view earnings overview and settlement details (English UI title Model Earnings), including **2 sub-tabs**: **My Earnings Overview** (default) + **Earning Details** |

## Page Structure

### Top Toolbar

The fixed area at the top of the page provides filtering and query tools:
- **"My Earnings Overview" / "Earning Details"** Tab switch (My Earnings Overview by default)
- **"Data Granularity"** dropdown selector (**"Day"** by default)
- **"Time Range"** selector (default `2026-06-01 00:00:00 - 2026-06-23 15:44:00`)

### My Earnings Overview (Default Tab)

The page is divided into 2 major blocks: 4 core metric cards + 6 charts (2x3 grid layout).

- **4 core metric cards** (each with a mini line chart + comparison with the previous period):
  - **User Count** (for example, `2`, down 0% from previous period) - Number of independent users who called models during the statistical period
  - **Model Count** (for example, `5`, down 0% from previous period) - Number of models that generated earnings during the statistical period
  - **Revenue Tokens** (for example, `753,596`, down 0% from previous period) - Total Tokens generated by user calls during the statistical period
  - **Revenue Credit** (for example, `59.12`, down 0% from previous period) - Total revenue Credit during the statistical period
- **6 charts** (2x3 grid):
  - **Model Revenue Trend** (line chart, multi-line comparison) - Revenue time series by model type (Chat Model / Image Model / Audio Model / Video Model / Embedding Model / Rerank Model)
  - **Model Revenue Percentage** (donut chart) - Contribution percentage of different model types to total revenue
  - **User Activity** (bar chart) - Daily active user count
  - **Model Call Count Trend** (line chart, multi-line comparison) - Call volume time series by model instance (for example, `dsy002` / `audio-dsy-1.1` / `images-dsy-1.1` / `dsy-3.1` / `dsy001`)
  - **Credit Consumption Ranking (TOP10)** (table) - Credit consumption ranking by user
  - **Model Call Count Distribution** (donut chart) - Call percentage distribution of different models

### Earning Details Tab

The page is divided into 2 sections: 3 settlement data cards + a 9-column earning detail table.

- **3 settlement data cards** (top):
  - **Current Billing Cycle Earnings** (for example, `59.12`) - Total earnings in the current billing cycle
  - **Recorded** (for example, `59.03`) - Earnings that have completed settlement
  - **Pending Carryover (+)** (for example, `0.09`) - Earnings in the current billing cycle that have not yet been settled
- **Filter toolbar** (below the billing cycle):
  - **"Billing Cycle"** selector (default `2026-06`, YYYY-MM format)
  - Time range (locked to the start and end time of the billing cycle by default)
  - **"User Name"** / **"Model Name"** input boxes
  - **"Search"** / **"Reset"** / **"Expand"** buttons
- **9-column earning detail table** (sorted by time descending, each row shows earnings details of one call): Usage Time / Model / User Name / Latency / Metered Usage / Free Quota / Billing Mode / Billing Rule / Credit

## Operations

### View Earnings Overview (My Earnings Overview Tab, Default)

1. Enter the platform homepage, click **"Usage & Earnings > Model Earnings"** in the left navigation bar, and enter the **"My Earnings Overview"** Tab by default.
2. Use the top **"Data Granularity"** selector (**"Day"** by default) + **"Time Range"** selector (for example, `2026-06-01 00:00:00 - 2026-06-23 15:44:00`).
3. View **4 core metric cards** (each with a mini line chart + comparison with the previous period):
   - **User Count** (for example, `2`, down 0% from previous period) - Number of independent users who called models during the statistical period
   - **Model Count** (for example, `5`, down 0% from previous period) - Number of models that generated earnings during the statistical period
   - **Revenue Tokens** (for example, `753,596`, down 0% from previous period) - Total Tokens generated by user calls during the statistical period
   - **Revenue Credit** (for example, `59.12`, down 0% from previous period) - Total revenue Credit during the statistical period
4. View **6 charts** (2x3 grid layout):
   - **Model Revenue Trend** (line chart, multi-line comparison): revenue changes over time by model type (Chat Model / Image Model / Audio Model / Video Model / Embedding Model / Rerank Model); hover to show the specific revenue value of each model type on that day
   - **Model Revenue Percentage** (donut chart): contribution percentage of different model types to total revenue (for example, Chat Model 57.81 Credit 97.78% / Image Model 1.32 Credit 2.22% / Audio Model 0.01 Credit 0.01%)
   - **User Activity** (bar chart): daily active user count
   - **Model Call Count Trend** (line chart, multi-line comparison): call volume changes over time by model instance (for example, `dsy002` / `audio-dsy-1.1` / `images-dsy-1.1` / `dsy-3.1` / `dsy001`)
   - **Credit Consumption Ranking (TOP10)** (table): Credit consumption ranking by user (for example, 1. `user-dsy01` 59.12 Credit / 2. `dushuangyan001` 0.01 Credit)
   - **Model Call Count Distribution** (donut chart): call percentage distribution of different models

### View Earning Details (Earning Details Tab)

1. Click the **"Earning Details"** Tab to switch to the settlement detail view.
2. Use the top **"Billing Cycle"** selector (default `2026-06`, YYYY-MM format). After switching the billing cycle, the data below refreshes by billing cycle.
3. View **3 settlement data cards**:
   - **Current Billing Cycle Earnings** (for example, `59.12`) - Total earnings in the current billing cycle
   - **Recorded** (for example, `59.03`) - Earnings that have completed settlement
   - **Pending Carryover (+)** (for example, `0.09`) - Earnings in the current billing cycle that have not yet been settled
4. Use the time range (locked to the start and end time of the billing cycle by default, for example, `2026-06-01 00:00:00 - 2026-06-23 15:44:00`) + filter toolbar (**User Name** / **Model Name** input boxes + **Search** / **Reset** / **Expand** buttons).
5. View the **9-column earning detail table** (Usage Time / Model / User Name / Latency / Metered Usage / Free Quota / Billing Mode / Billing Rule / Credit), sorted by time descending to show earning details of each call:
   - **Usage Time**: Time when the call occurred (for example, `2026-06-18 19:20:53`)
   - **Model**: Model name + model type tag (for example, `dsy-3.1` + `Chat Model`)
   - **User Name**: User who initiated the call (for example, `user-dsy01`)
   - **Latency**: Time consumed by this call (ms, for example, `11,508`)
   - **Metered Usage**: 4 rows of Token information (**Input Tokens** + **Output Tokens** + **Cached Input Tokens** + **Context 0K-256K**)
   - **Free Quota**: Whether the call is within the free quota (`--` means used up; Token / Tiered and other quota types)
   - **Billing Mode**: Tiered / Cache / Token, etc.
   - **Billing Rule**: Brief rule (for example, `Output 80 Credits/1M tokens`) + detail link (for example, `View all 3 prices · 1 detail` / `View all 2 prices` / `View full details`)
   - **Credit**: Revenue Credit of this call (for example, `42.33 credits` / `0.33 credits` / `0.01 credits`)
6. The bottom of the table provides pagination controls.

## Other Operations

| Operation | Steps |
|----------|----------|
| Switch Data Granularity | Switch `Day / Hour / Minute` in the top **"Data Granularity"** selector -> All charts and cards refresh by the new granularity |
| Switch Time Range | Use the top time range selector (default `2026-06-01 ~ 2026-06-23`) -> After customizing the start and end time, charts and cards refresh by the new range |
| Switch Billing Cycle | Earning Details Tab -> Top **"Billing Cycle"** selector (default `2026-06`, YYYY-MM format) -> After switching the billing cycle, data below refreshes by billing cycle |
| Filter User | Earning Details Tab -> **"User Name"** input box -> Enter keywords and click **"Search"** to filter details of that user |
| Filter Model | Earning Details Tab -> **"Model Name"** input box -> Enter keywords and click **"Search"** to filter details of that model |
| Reset Filters | Earning Details Tab -> Click **"Reset"** -> Clear User Name / Model Name filter conditions |
| Expand / Collapse Details | Earning Details Tab -> Click **"Expand"** -> Expand / collapse folded areas such as "Metered Usage" and "Billing Rule" in the table |
| View TOP10 Ranking | Overview Tab -> **"Credit Consumption Ranking (TOP10)"** card -> View Credit consumption ranking by user |

## Notes

- **Role permission**: This module is visible only to **model providers**. Ordinary consuming users should be redirected to the **Model Usage** page.
- **Settlement cycle**: Earnings are usually settled monthly (billing cycle = YYYY-MM). The exact cycle follows platform rules; earnings in the current cycle can be "withdrawn" only in the next billing cycle.
- **Data latency**: Earnings data usually has a 1-2 day settlement delay. Earnings from calls that just occurred are not reflected immediately.
- **Billing modes**: Supports **Tiered Cache** (tiered by context window + cache hit discount) and **Token** (by Token count).
- **Cache hits**: **Cached Input Tokens** are input Tokens that hit cache (not charged or charged at cache price), visible in the "Metered Usage" column.
- **Current billing cycle vs Recorded vs Pending Carryover**:
  - **Current Billing Cycle Earnings** = Total earnings generated in the current billing cycle
  - **Recorded** = Earnings that have completed settlement (withdrawable)
  - **Pending Carryover** = Earnings in the current billing cycle that have not yet been settled (settled in the next billing cycle)
- **Multi-model earnings**: The same model can be deployed by multiple supplier instances, and each instance settles earnings independently.
- **Real data example**: User `user-dsy01` called `dsy-3.1` (Chat Model) once, consuming 64,215 Input + 515,880 Output + 12,843 Cached = 42.33 Credit revenue (`dsy-3.1`, `dsy002`, etc. are supplier instance names of the same model series).
- **Menu vs page title**: The menu shows **"Model Earnings"** (Chinese: 模型收益), and the page displays **"Model Earnings"** (Tab name "My Earnings Overview"); all refer to the same feature.

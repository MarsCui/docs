# 币种设置

::: info 文档信息
版本：v1.0
更新日期：2026-07-08
:::

## 功能概述

`币种设置` 用于维护平台计价单位、汇率、展示符号、结算口径和启用状态，确保模型用量、收益和客户账单使用同一计价口径。

| 项目 | 内容 |
| --- | --- |
| 适用角色 | 运营方 |
| 导航路径 | 系统设置 > 币种设置 |
| 页面路由 | /operator/settings/currency-settings |
| 管理对象 | 币种、汇率、展示符号、结算口径和启用状态 |
| 典型用途 | 维护模型市场计费展示与收益换算口径 |

### 新手理解

币种设置像平台里的计价尺，决定用量如何折算成点数、余额或收益。口径不清会导致用量、收益和客户账单对不上。

### 术语速查

| 术语 | 说明 |
| --- | --- |
| 币种 | 价格展示或结算使用的币种单位。 |
| 汇率 | 不同币种之间的换算比例。 |
| 结算口径 | 收益、用量或费用统计时采用的计算规则。 |
| 启用状态 | 控制币种是否在页面中可用。 |

## 前提条件

1. 当前账号具备币种或点数设置权限。
2. 计量单位、换算比例、展示精度和适用业务范围已确认。
3. 收益、用量和账单导出口径已与财务或运营口径对齐。
## 页面说明

页面用于维护计费币种、点数名称、换算比例、展示精度和启用状态。运营方应与计费、收益和用量统计口径保持一致，并避免在文档中暴露真实商业价格策略。

## 主要操作

### 管理币种设置

1. 进入 `系统设置 > 币种设置`。
2. 查看当前币种、点数名称和换算比例。
3. 新增或编辑时填写展示名称、单位、精度和启用状态。
4. 保存前与用量统计和收益页面口径核对。
5. 保存后在模型收益和模型用量页面验证展示结果。

## 参数说明

| 字段名称 | 是否必填 | 字段类型 | 示例 | 说明 |
| --- | --- | --- | --- | --- |
| 展示名称 | 是 | 文本 | `Credits` | 用户看到的计量单位名称。 |
| 换算比例 | 是 | 数字 | `1:1000` | 内部计量值到展示单位的换算规则。 |
| 精度 | 是 | 数字 | `2` | 页面展示的小数位。 |
| 适用范围 | 否 | 多选 | `模型调用` | 该币种配置影响的业务范围。 |
| 状态 | 是 | 枚举 | `启用` | 控制是否生效。 |

## 踩坑提示

- 不要把真实商业折扣、客户协议价或内部结算价写入截图。
- 换算比例变更会影响收益和用量展示，发布前要做口径说明。
- 同一页面不要混用多个单位名称。

## 结果校验

| 检查项 | 成功表现 | 异常时处理 |
| --- | --- | --- |
| 单位一致 | 用量、收益页面展示的单位与币种设置一致 | 回到币种设置核对展示名称和启用状态 |
| 明细可对账 | 明细导出中的原始计量值和展示值可对账 | 核对换算比例、精度和统计时间范围 |
| 生效周期明确 | 配置变更后新统计周期按新口径展示 | 确认生效时间并等待统计任务刷新 |
## 常见问题

### 收益金额与用量不一致

**问题现象：**

模型收益页和模型用量页折算结果对不上。

**可能原因：**

- 换算比例或精度配置不一致。
- 统计时间范围不同。
- 存在延迟结算或退款调整。

**处理方式：**

1. 核对币种设置和统计时间范围。
2. 查看导出明细中的原始计量值。
3. 等待结算任务完成后复核。

### 页面单位展示异常

**问题现象：**

页面显示旧单位或小数位不符合预期。

**可能原因：**

- 配置未启用。
- 页面缓存未刷新。
- 部分模块仍使用旧配置。

**处理方式：**

1. 确认新配置启用状态。
2. 刷新页面或等待缓存更新。
3. 检查相关模块是否支持该单位。

### 币种变更后历史数据会变化吗？

**问题现象：**

运营方调整币种展示名称、精度或换算比例后，不确定历史收益和用量是否会同步变化。

**可能原因：**

不同统计页面可能按生效时间、账期或实时配置展示，历史数据和新周期数据的口径不一定相同。

**处理方式：**

变更前记录生效时间和影响范围；变更后分别核对历史账期、新统计周期和导出明细，必要时在对外说明中标明口径变化。

- 对外材料优先使用脱敏样例金额。

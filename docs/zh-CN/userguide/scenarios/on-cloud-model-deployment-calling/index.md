# 场景概览 - 多云模型部署与调用

指导用户（租户成员）从云上模型广场快速部署模型实例，并获取 API 调用能力。

## 适用角色

- 模型提供方、平台用户

## 场景目标

- 接入或确认可用云账号。
- 选择目标模型、推理引擎和算力方案完成部署。
- 从我的部署获取状态、监控和 API 调用能力。

## 场景流程

**主线：** 准备用户接入账号 → 选择模型和算力方案 → 检查部署状态 → 调用并管理生命周期

| 阶段 | 核心结果 |
| --- | --- |
| 1. 准备账号 | 用户云账号校验成功，授权地域可见 |
| 2. 创建部署 | 模型、引擎、策略、规格和费用方案确认后提交 |
| 3. 检查运行 | 部署进入运行状态，事件和监控无持续错误 |
| 4. 调用管理 | API 请求成功，并能按需停止或释放云资源 |

## 开始前准备

- 运营管理员已完成云资源、业务地域和租户授权。
- 目标模型资产已上架。
- 用户已准备经过处理的云账号凭据样例。

## 推荐阅读顺序

1. 接入账号
2. 快速部署
3. 在我的部署检查状态、事件、监控和 API

## 完成检查

> **用途：** 以下检查是本场景的退出条件，用于判断是否已经取得可观察、可复核的结果，以及是否可以进入下一场景。它不是操作步骤的重复；任一项不满足时，请返回对应功能文档的“常见失败分支”排查。

| 检查项 | 通过标准 |
| --- | --- |
| 1 | 用户接入账号校验成功并能看到被授权地域。 |
| 2 | 快速部署能够选择目标模型、推理引擎和算力方案。 |
| 3 | 部署状态进入运行中，事件和监控没有持续错误。 |
| 4 | 使用脱敏后的 接口地址 和 API Key 完成受控调用。 |
| 5 | 停止或删除测试部署后，状态和资源释放符合预期。 |

## 常见失败分支

| 现象 | 优先检查 |
| --- | --- |
| 无可用账号或地域 | 云账号校验、资源授权和租户范围 |
| 智能推荐没有方案 | 模型资产、框架、算力方案和地域容量 |
| 部署长期创建中或失败 | 事件、云账号权限、镜像、配额和云侧资源状态 |
| 部署运行但调用失败 | 接口地址、认证、端口、协议和服务健康 |

## 功能概述

账务汇总 AGIOne 的账务、计费、结算、充值和 License 管理入口。它把用户侧的余额、充值、流水、月度账单，以及运营侧的客户账务、运营财务、巡检、调账和授权管理放在同一个子系统下，方便不同角色按自己的问题入口继续排查。

### 新手理解

账务像平台的财务服务台：普通用户先看自己的余额、充值和账单，运营人员再看客户档案、结算单、财务账户和巡检结果，License 管理员负责确认资源授权是否有效。新手不要一上来就查单据，先判断问题属于“用户余额”“客户充值”“账期结算”还是“资源授权”。

## 账务是什么

账务是平台内围绕余额、充值、消费、结算、对账、调账和 License 授权形成的统一入口。它不直接替代业务订单或外部支付系统，而是把用户侧账单、运营侧财务处理和授权状态串联起来。

## 使用前提

1. 当前账号已具备对应账务菜单权限。
2. 已明确要处理的是用户侧账务、客户账务、运营财务还是 License。
3. 涉及金额核对时，先统一账期、组织、客户、业务线和交易类型。
4. 涉及结算、调账、补偿或 License 激活时，先确认审批依据和影响范围。
5. 对外沟通或截图前，确认账号、组织、流水号、订单号、金额和授权信息已脱敏。

## 常见问题

### 不知道先看用户侧还是运营侧怎么办？

**问题现象：**

只知道账务数据有疑问，但不确定应该进入用户侧账务还是运营侧账务。

**可能原因：**

用户侧页面关注当前账号能看到的余额、充值和账单；运营侧页面关注客户、组织、结算、账户流水和异常处理。

**处理方式：**

如果问题来自某个用户自己看余额或充值，先看 [我的账务](../user/billing/)；如果要核对客户、组织、结算单、财务账户或巡检异常，进入 [运营财务](../operator/finance-operations/) 或 [客户账务](../operator/customer-billing/)。

### 余额、流水和月度账单为什么对不上？

**问题现象：**

账户概览、流水明细和月度账单看到的金额不一致。

**可能原因：**

页面统计口径不同，或账期、交易时间、账号、项目、业务线筛选条件不一致，也可能存在同步延迟。

**处理方式：**

先统一账期和筛选条件；再用流水明细定位单笔变化；最后用月度账单核对账期汇总。仍不一致时，由运营人员进入财务账户和巡检中心继续排查。

### License 正常但账务仍异常怎么办？

**问题现象：**

License 显示有效，但充值、余额、结算或流水仍存在异常。

**可能原因：**

License 只说明资源授权状态，不代表账务链路已经完成结算、入账或对账。

**处理方式：**

资源授权问题看 License；金额、充值、流水、结算问题进入客户账务或运营财务。不要用 License 状态替代账务核对结果。

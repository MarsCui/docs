# 场景概览-可观测与问题排查

本场景介绍如何从模型调用、组织范围和基础资源三个视角定位吞吐、时延、失败、排队和资源异常。

## 适用角色

- 运营管理员、模型提供方、最终用户

## 场景目标

- 先确认影响范围和时间范围，再选择正确监控入口。
- 将错误对应到请求、模型、作业、节点或设备。
- 形成可复核且不包含完整凭据的排障证据。

## 开始前准备

- 准备脱敏后的请求、实例或作业标识。
- 明确租户、地域、模型和时间范围。

## 推荐阅读路径

1. 确认影响范围
2. 检查调用或作业状态
3. 下钻到节点和设备
4. 记录证据并复测

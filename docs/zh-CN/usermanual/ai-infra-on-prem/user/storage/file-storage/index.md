## 功能概述

`文件存储` 用于查看和使用运营方在目标地域开放的共享文件存储能力。当能力开放后，页面会展示可用文件系统、共享目录、容量、状态和挂载入口；当能力未开放时，用户需要确认地域、权限、配额和运营方组件接入状态。

| 项目 | 内容 |
| --- | --- |
| 适用角色 | 普通用户 |
| 导航路径 | AI基础设施 > On-Prem > 存储服务 > 文件存储 |
| 页面路由 | `/powerone/storage-service/file` |
| 管理对象 | 共享文件系统、目录、容量、挂载关系和访问策略 |
| 典型途径 | 为多个实例或作业提供共享目录，适合数据集、模型仓库和输出目录 |

#### 新手理解

文件存储像团队共享文件夹，适合多个实例或作业读取同一批数据、模型文件或输出目录。它强调目录和文件语义，和块存储的独立硬盘语义不同。

#### 术语速查

| 术语 | 说明 |
| --- | --- |
| 共享目录 | 多个实例或作业可访问的文件系统目录。 |
| NFS | 常见共享文件系统协议。 |
| 挂载路径 | 容器内访问共享目录的路径。 |
| 访问策略 | 只读或读写权限控制。 |

## 前提条件

1. 运营方已在目标地域接入并开放文件存储组件。
2. 当前账号具备查看和使用文件存储的权限。
3. 租户配额、容量和实例规格满足使用要求。
4. 需要挂载到实例时，实例所在地域、集群和存储能力一致。

## 页面说明

页面用于展示所选地域的文件存储能力。能力开放时，通常会展示列表、容量、状态、创建入口、挂载入口和操作入口；能力未开放时，页面会给出能力不可用提示。

## 主要操作

### 创建共享目录

#### 功能可用时页面会展示

| 区域 | 说明 |
| --- | --- |
| 列表区 | 展示已有共享目录、容量、状态和更新时间。 |
| 创建入口 | 新增文件存储目录或声明。 |
| 挂载入口 | 将共享目录关联到实例或容器路径。 |
| 操作入口 | 编辑、扩容、卸载、删除或查看详情，具体以页面提供能力为准。 |

#### 操作步骤

1. 进入 `AI基础设施 > On-Prem > 存储服务 > 文件存储`。
2. 确认右上角地域。
3. 如页面提供创建入口，填写名称、容量、访问策略和描述。
4. 提交后返回列表查看状态。
5. 在创建实例或实例详情中选择该共享目录并设置容器内路径。

## 参数说明

| 字段名称 | 是否必填 | 字段类型 | 示例 | 说明 |
| --- | --- | --- | --- | --- |
| 共享目录名称 | 必填 | 文本 | `shared-dataset` | 文件存储资源展示名称。 |
| 容量 | 必填 | 数字 | `500GiB` | 共享目录可用空间。 |
| 访问策略 | 条件必填 | 枚举 | `读写` | 控制只读或读写访问。 |
| 挂载路径 | 条件必填 | 文本 | `/mnt/shared` | 实例或容器内访问路径。 |
| 共享状态 | 系统生成 | 枚举 | `可用` | 用于判断是否可挂载、扩容或删除。 |

#### 挂载、卸载和容量确认

#### 挂载

1. 打开实例创建页或存储挂载入口。
2. 选择目标文件存储资源。
3. 填写容器内路径，例如 `/mnt/data` 或 `/mnt/output`。
4. 提交后查看实例事件和日志，确认挂载成功。

#### 卸载

1. 确认没有运行中的进程正在读写该路径。
2. 在实例或存储操作入口执行卸载。
3. 刷新页面确认挂载关系解除。

#### 容量确认

1. 在文件存储列表查看容量和状态。
2. 在实例内执行 `df -h` 或应用侧容量检查。
3. 如容量不足，按页面能力扩容或联系运营方调整配额。

## 替代路径

- 需要保存模型文件、数据集或产物包时，优先考虑 [对象存储](../object-storage/)。
- 需要共享目录语义时，使用文件存储或运营方配置的集群共享存储。
- 需要独立卷能力时，使用块存储；如果页面未开放，联系运营方确认目标地域是否具备底层存储组件。

## 排障信息准备

文件存储异常时，先准备以下信息，便于判断是共享路径、权限、客户端节点还是多节点访问问题：

| 信息 | 示例 | 作用 |
| --- | --- | --- |
| 文件系统 ID | `fs-20260713001` | 定位目标文件存储。 |
| 挂载路径 | `/mnt/share` | 判断容器内路径是否冲突。 |
| 权限 | `读写 / 只读` | 判断写入失败是否由权限导致。 |
| 客户端节点 | `node-gpu-01` | 判断哪台节点挂载失败。 |
| 共享目录 | `/exports/models` | 核对 NFS 导出路径和租户目录。 |

## 踩坑提示

- 共享目录中的文件会被多个任务看到，写入前确认命名和覆盖风险。
- NFS 或共享存储路径截图前必须脱敏。
- 高并发小文件读写可能影响性能，必要时拆分目录或改用对象存储。

## 结果校验

| 检查项 | 成功表现 | 异常时处理 |
| --- | --- | --- |
| 页面可访问 | 能进入文件存储页面并看到共享目录列表或空状态提示。 | 检查当前账号权限、区域范围和文件存储组件开放状态。 |
| 目录创建入口可见 | 具备权限时能看到创建共享目录入口并打开表单。 | 确认文件存储能力是否对当前租户开放。 |
| 挂载信息清楚 | 能看到文件系统 ID、挂载路径、权限、客户端节点和状态。 | 核对 NFS 地址、导出目录、读写权限和客户端网络。 |
| 多节点访问可确认 | 共享目录在目标实例或节点上按预期读写。 | 检查导出策略、权限范围、挂载参数和节点连通性。 |



## 常见问题

#### 页面没有文件存储数据

**问题现象：**

进入页面后没有看到可用文件存储资源，或页面提示该能力未向所选地域开放。

**可能原因：**

- 运营方尚未在目标地域接入文件存储组件。
- 当前账号没有查看或使用权限。
- 租户配额或容量不足。
- 筛选地域与实例所在地域不一致。

**处理方式：**

1. 确认右上角地域。
2. 联系运营方确认文件存储组件、地域绑定和租户权限。
3. 检查资源配额和容量。
4. 短期可使用对象存储或实例内临时目录，但临时目录不适合保存重要结果。

#### 挂载后路径不可用

**问题现象：**

实例启动后，容器内无法访问文件存储挂载路径。

**可能原因：**

- 挂载路径填写错误或与应用目录冲突。
- 实例所在集群无法访问底层存储。
- 权限、访问策略或容量配置不正确。

**处理方式：**

1. 查看实例事件和日志。
2. 确认容器内路径、访问策略和实例地域。
3. 联系运营方检查底层存储组件和集群挂载能力。

#### 删除或卸载失败

**问题现象：**

尝试删除或卸载文件存储资源时失败。

**可能原因：**

- 仍有运行中实例正在使用该资源。
- 资源处于异常、扩容中或回收中状态。
- 当前账号权限不足。

**处理方式：**

1. 先停止或迁移依赖实例。
2. 刷新页面确认资源状态。
3. 联系运营方检查权限和底层存储回收状态。

## 后续操作

1. 在运行实例或在线 IDE 中验证挂载路径。
2. 将输入数据和输出结果写入持久化路径。
3. 定期清理无用数据，避免占满配额。

## 注意事项

- 删除或卸载前先确认没有运行中实例依赖。
- 挂载路径不要覆盖系统目录、应用目录或镜像内关键目录。

# 应用发布与审核

本任务用于核对应用发布信息、处理审核并验证客户可见性。

## 适用角色

- 平台运营方、应用发布管理人员、应用审核人员

## 开始前准备

- 确认具备应用发布管理或应用审核权限。
- 准备应用说明、绑定模型、调用入口、参数映射和客户范围。
- 确认绑定模型可用，目标客户已经获得授权。

## 操作流程

### 1. 核对应用发布信息

进入[应用列表](../../../usermanual/model-services/operator/publishing/apps/)，按应用名称或发布状态筛选，打开详情核对绑定模型、调用入口、参数映射、客户可见范围和发布说明。

![在应用列表中定位发布记录](../../../usermanual/model-services/operator/publishing/apps/images/apps-list.png)

### 2. 处理应用审核

进入[应用审核](../../../usermanual/model-services/operator/approvals/app-reviews/)，按状态、应用名称、申请人或提交时间筛选。核对应用说明、绑定模型、调用入口和客户范围后，选择通过、驳回或要求补充材料，并填写具体审核意见。

![在应用审核列表中定位申请](../../../usermanual/model-services/operator/approvals/app-reviews/images/app-reviews-list.png)

### 3. 验证发布状态和客户可见性

审核通过后返回应用列表确认发布状态，再使用目标客户视角验证应用是否可见、调用入口是否正确。非目标客户不应看到应用。

### 4. 验证调用并跟踪问题

执行一次受控调用，确认绑定模型、参数映射和客户额度正常。失败时结合模型状态、调用日志、错误码和配额继续排查。

## 完成检查

> **用途：** 以下检查用于确认应用审核已经形成预期发布边界和可用客户路径。任一项不满足时，不应宣布应用可用。

| 检查项 | 通过标准 |
| --- | --- |
| 发布信息 | 应用、模型、入口、参数和客户范围正确。 |
| 审核记录 | 状态、意见、处理人和时间完整。 |
| 可见范围 | 目标客户可见，非目标客户不可见。 |
| 调用验证 | 应用调用成功，或失败原因已定位。 |

## 常见失败分支

| 现象 | 优先检查 |
| --- | --- |
| 应用发布后客户不可见 | 发布状态、客户范围、绑定模型状态和同步时间 |
| 审核无法通过 | 必填材料、调用入口、模型状态和审核权限 |
| 审核通过后仍不可见 | 应用发布状态和客户授权范围 |
| 应用调用失败 | 模型状态、参数映射、客户额度和调用日志 |

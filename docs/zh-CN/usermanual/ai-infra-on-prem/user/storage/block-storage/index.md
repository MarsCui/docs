# 块存储

## 功能概述

`块存储` 用于查看和使用运营方在目标地域开放的块存储卷能力。当能力开放后，页面会展示可用卷、容量、状态和挂载入口；当能力未开放时，用户需要确认地域、权限、配额和运营方组件接入状态。

| 项目 | 内容 |
| --- | --- |
| 适用角色 | 普通用户 |
| 导航路径 | 存储服务 > 块存储 |
| 页面路由 | `/powerone/storage-service/block` |
| 管理对象 | 块存储卷、容量、挂载关系和卷状态 |
| 典型用途 | 为实例提供独立持久化卷，适合需要块设备或独立卷的任务 |

### 新手理解

块存储像给实例挂一块独立硬盘，适合需要持久化写入、独立卷或块设备语义的任务。它不是共享目录，多个实例同时读写前需要确认平台是否支持对应挂载方式。

### 术语速查

| 术语 | 说明 |
| --- | --- |
| 卷 | 可被实例挂载的持久化块设备。 |
| 容量 | 卷可使用的存储大小。 |
| 挂载 | 把卷连接到实例或容器内路径。 |
| 卸载 | 解除卷与实例的挂载关系。 |

## 前提条件

1. 运营方已在目标地域接入并开放块存储组件。
2. 当前账号具备查看和使用块存储的权限。
3. 租户配额、容量和实例规格满足使用要求。
4. 需要挂载到实例时，实例所在地域、集群和存储能力一致。

## 页面说明

页面用于展示所选地域的块存储能力。能力开放时，通常会展示列表、容量、状态、创建入口、挂载入口和操作入口；能力未开放时，页面会给出能力不可用提示。

## 创建卷

### 功能可用时页面会展示

| 区域 | 说明 |
| --- | --- |
| 列表区 | 展示已有卷、容量、状态和更新时间。 |
| 创建入口 | 新增块存储卷。 |
| 挂载入口 | 将卷关联到实例或容器路径。 |
| 操作入口 | 编辑、扩容、卸载、删除或查看详情，具体以页面提供能力为准。 |

### 操作步骤

1. 进入 `存储服务 > 块存储`。
2. 确认右上角地域。
3. 如页面提供创建入口，填写名称、容量、访问策略和描述。
4. 提交后返回列表查看状态。
5. 在创建实例或实例详情中选择该卷并设置容器内路径。

### 参数说明

| 字段名称 | 是否必填 | 字段类型 | 示例 | 说明 |
| --- | --- | --- | --- | --- |
| 卷名称 | 是 | 文本 | `train-data-volume` | 块存储卷展示名称。 |
| 容量 | 是 | 数字 | `100GiB` | 卷可使用空间。 |
| 访问模式 | 条件必填 | 枚举 | `ReadWriteOnce` | 控制卷是否允许单实例或多实例挂载。 |
| 挂载路径 | 条件必填 | 文本 | `/mnt/data` | 实例或容器内访问路径。 |
| 卷状态 | 系统生成 | 枚举 | `可用` | 用于判断是否可挂载、扩容或删除。 |

## 挂载、卸载和容量确认

### 挂载

1. 打开实例创建页或存储挂载入口。
2. 选择目标块存储资源。
3. 填写容器内路径，例如 `/mnt/data` 或 `/mnt/output`。
4. 提交后查看实例事件和日志，确认挂载成功。

### 卸载

1. 确认没有运行中的进程正在读写该路径。
2. 在实例或存储操作入口执行卸载。
3. 刷新页面确认挂载关系解除。

### 容量确认

1. 在块存储列表查看容量和状态。
2. 在实例内执行 `df -h` 或应用侧容量检查。
3. 如容量不足，按页面能力扩容或联系运营方调整配额。

## 替代路径

- 需要保存模型文件、数据集或产物包时，优先考虑 [对象存储](../object-storage/)。
- 需要共享目录语义时，使用文件存储或运营方配置的集群共享存储。
- 需要独立卷能力时，使用块存储；如果页面未开放，联系运营方确认目标地域是否具备底层存储组件。

## 联系运营方前准备

当页面能力未开放、数据为空或挂载失败时，联系运营方前建议准备以下信息：

| 信息 | 示例 | 作用 |
| --- | --- | --- |
| 当前地域 | `武汉` | 判断能力是否在该地域开放。 |
| 当前账号 / 租户 | `tenant-a` | 判断菜单、资源和监控权限。 |
| 目标实例或作业 | `train-job-001` | 便于定位日志、事件和计量记录。 |
| 目标规格或资源 | `gpu-a100-1-16c-64g` | 判断配额、规格和集群能力。 |
| 页面现象 | `无数据 / 挂载失败 / 图表为空` | 帮助运营方判断入口、采集或底层资源问题。 |

替代排障路径：

1. 先查看实例详情、日志和事件。
2. 查看资源用量和资源配额，确认是否存在配额或额度限制。
3. 存储能力不可用时，优先使用对象存储保存模型、数据集和输出产物。
4. 监控能力未开放时，以实例状态、日志、事件和用量作为短期排障依据。

### 踩坑提示

- 块存储通常不适合多个实例同时读写同一路径，使用前确认访问模式。
- 挂载路径不要覆盖系统目录、启动目录或镜像内关键目录。
- 删除卷前确认没有运行中实例、训练任务或输出产物依赖。

## 常见问题

### 页面没有块存储数据

**问题现象：**

进入页面后没有看到可用块存储资源，或页面提示该能力未向所选地域开放。

**可能原因：**

- 运营方尚未在目标地域接入块存储组件。
- 当前账号没有查看或使用权限。
- 租户配额或容量不足。
- 筛选地域与实例所在地域不一致。

**处理方式：**

1. 确认右上角地域。
2. 联系运营方确认块存储组件、地域绑定和租户权限。
3. 检查资源配额和容量。
4. 短期可使用对象存储或实例内临时目录，但临时目录不适合保存重要结果。

### 挂载后路径不可用

**问题现象：**

实例启动后，容器内无法访问块存储挂载路径。

**可能原因：**

- 挂载路径填写错误或与应用目录冲突。
- 实例所在集群无法访问底层存储。
- 权限、访问策略或容量配置不正确。

**处理方式：**

1. 查看实例事件和日志。
2. 确认容器内路径、访问策略和实例地域。
3. 联系运营方检查底层存储组件和集群挂载能力。

### 删除或卸载失败

**问题现象：**

尝试删除或卸载块存储资源时失败。

**可能原因：**

- 仍有运行中实例正在使用该资源。
- 资源处于异常、扩容中或回收中状态。
- 当前账号权限不足。

**处理方式：**

1. 先停止或迁移依赖实例。
2. 刷新页面确认资源状态。
3. 联系运营方检查权限和底层存储回收状态。

## 后续操作

1. 在运行实例或在线 IDE 中验证挂载路径。
2. 将输入数据和输出结果写入持久化路径。
3. 定期清理无用数据，避免占满配额。

## 注意事项

- 删除或卸载前先确认没有运行中实例依赖。
- 挂载路径不要覆盖系统目录、应用目录或镜像内关键目录。

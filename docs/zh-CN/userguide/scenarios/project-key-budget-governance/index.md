# 场景概览 - 项目、Key 与预算治理

本场景帮助服务商账号创建项目、设置预算和模型白名单，并为不同用途创建可控、可轮换的调用 Key。

## 适用角色

- 模型提供方、平台用户、服务商管理员

## 场景目标

- 项目具备明确用途、预算周期和超预算策略。
- 模型白名单与项目业务范围一致。
- Key 的用途、过期时间、限额和责任人清晰。
- Key 创建后能够完成受控调用，并可在活动或日志中追溯。

## 场景流程

**主线：** 创建项目 → 配置预算与模型范围 → 创建 Key → 设置限额 → 验证调用与审计

| 阶段 | 核心结果 |
| --- | --- |
| 1. 定义项目 | 项目用途、负责人和成员范围清晰 |
| 2. 治理预算 | 周期预算、预警阈值和超预算策略已保存 |
| 3. 限制访问 | 模型白名单和 Key 范围与项目一致 |
| 4. 验证与审计 | 调用遵守限制，变更记录可追溯 |

## 开始前准备

- 当前账号具备项目和 Key 管理权限。
- 已明确项目负责人、预算周期、模型范围和超预算策略。
- 已确认使用 Model API Key 还是 System API AK/SK。

## 推荐阅读顺序

1. 创建并检查项目。
2. 配置预算、成员和模型白名单。
3. 创建独立用途的 Key。
4. 设置 Key 限额并验证调用。

## 完成检查

> **用途：** 以下检查是本场景的退出条件，用于确认项目成本和 API 凭据已具备可执行边界及明确责任人，再进入生产使用。

| 检查项 | 通过标准 |
| --- | --- |
| 1 | 项目名称、负责人、预算周期和用途清晰。 |
| 2 | 项目预算、预警、超预算策略和模型白名单正确。 |
| 3 | Key 用途、有效期、限额和责任人可追溯。 |
| 4 | 测试调用符合预算、白名单和限额规则。 |

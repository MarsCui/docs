# 场景概览-身份授权

本场景介绍如何用组织、成员和角色建立访问边界，并验证用户能看到哪些菜单、按钮和业务资源。

## 适用角色

- 平台管理员、运营管理员
- 用于验证授权结果的模型提供方和最终用户

## 场景目标

- 用户归属正确组织并绑定正确角色。
- 菜单、按钮和资源范围符合最小权限原则。
- 权限变更能够通过重新登录和操作日志验证。

## 开始前准备

- 明确目标用户、组织和岗位职责。
- 列出必须允许和必须禁止的功能。
- 准备一个验证账号。

## 推荐阅读路径

1. 确定组织和角色边界
2. 配置成员与角色
3. 使用目标账号验证权限

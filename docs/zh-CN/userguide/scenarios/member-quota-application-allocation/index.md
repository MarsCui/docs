# 场景概览 - 成员额度申请与分配

本场景帮助服务商管理员按成员业务需要申请、调整和限制额度，并核对成员、项目和 Key 的多层限制。

## 适用角色

- 服务商管理员、模型提供方、需要申请额度的平台用户

## 场景目标

- 额度申请具备数量、原因和唯一待审批记录。
- 额度调整对象、方向和原因正确。
- 成员限额、模型白名单、项目预算和 Key 限额不互相矛盾。
- 调整后能够从成员详情和实际调用中验证。

## 场景流程

**主线：** 定位成员 → 发起额度申请 → 跟踪状态 → 调整成员额度 → 设置限额 → 验证调用

## 开始前准备

- 明确目标成员、追加额度和业务原因。
- 确认当前没有重复待审批申请。
- 设置限额前明确重置周期、触限策略和模型范围。

## 推荐阅读顺序

1. 查看成员额度与现有用量。
2. 发起并跟踪额度申请。
3. 审批后调整成员额度和限额。
4. 使用成员账号或 Key 验证结果。

## 完成检查

> **用途：** 以下检查是本场景的退出条件，用于确认审批额度已到达目标成员，并与项目和 Key 限制保持一致。

| 检查项 | 通过标准 |
| --- | --- |
| 1 | 申请对象、额度数量和业务原因正确。 |
| 2 | 申请状态、审批结果和调整记录可追溯。 |
| 3 | 成员额度、模型范围、项目预算和 Key 限额一致。 |
| 4 | 调整后的剩余额度和实际调用结果符合预期。 |

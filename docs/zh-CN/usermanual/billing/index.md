## 子系统定位

账务汇总 AGIOne 的计费与账务入口，覆盖用户账务、客户账务、运营财务和 License 管理。平台运营、账务运营、财务对账人员、Provider 和普通用户可以在这里判断入口边界，再进入余额、充值订单、月度账单、客户概览、月结进度、结算单、财务账户、账务巡检和 License 授权页面。

#### 新手理解

账务像 AGIOne 的财务总控室：用户账务负责普通用户自查余额、流水、充值和账单；客户账务负责识别客户、标签、充值来源和业务线；运营财务负责核对结算、账户和资金状态；License 负责核对平台资源授权、有效期和额度。新手不要从某个单据直接开始，应先判断自己要处理用户余额、客户充值、账期结算还是资源授权。

## 核心术语速查

| 术语 | 含义 | 常见位置 |
| --- | --- | --- |
| 账期 | 账务统计和结算使用的时间周期，通常按月份查看。 | 月度账单、月结总览、结算单列表、巡检中心 |
| Credits | 平台内用于计费或额度展示的单位。 | 账户概览、客户概览、充值订单、用户账单 |
| 充值订单 | 用户或客户发起充值后的处理记录。 | 充值订单、客户充值单 |
| 交易流水 | 余额、收入、支出或调整变化记录。 | 流水明细、财务账户 |
| 结算单 | 某个组织在某个账期下的结算记录。 | 结算单列表、月度结算单 |
| 清算账户 | 用于归集或核对交易资金的账户。 | 财务账户 |
| 收益账户 | 平台收益或 Provider 收益对应账户。 | 财务账户、收益总览 |
| 补偿队列 | 需要重试、补偿或人工处理的账务任务队列。 | 巡检中心 |
| 调账 | 对异常账务记录进行审批后的资金修正动作。 | 调账处理 |
| License | 控制平台模块、资源额度和有效期的授权。 | License |

## 推荐阅读路径

1. 第一次接触账务：先读本页，理解用户账务、客户账务、运营财务、Provider 收益和 License 的边界。
2. 需要快速建立全局理解：继续阅读 [账务快速入门](./getting-started/)。
3. 处理用户侧余额、充值或账单：进入 [我的账务](./user/billing/overview/)。
4. 处理 Provider 收益：进入 [收益总览](./user/earnings/revenue/)，再查看月度结算单或客户总览。
5. 处理客户相关问题：进入 [客户账务](./operator/customer-billing/customer-overview/)，再按需查看客户概览、客户充值单或业务线。
6. 处理结算和对账问题：进入 [运营财务](./operator/finance-operations/today-tasks/)，先看今日任务和月结总览，再进入结算单、财务账户或巡检中心。
7. 处理完整账期闭环：阅读 [完成一次账期对账与结算](./end-to-end/reconcile-billing-cycle/)。
8. 处理授权和额度问题：进入 [License](./operator/license/license/)，确认 License 类型、剩余额度和有效期。
9. 写操作记录或工单时，只记录脱敏后的账期、单据状态、处理结论和下一步动作。

## 使用前提

1. 当前账号已获得账务相关菜单权限。
2. 已明确本次处理对象属于用户账务、Provider 收益、客户账务、运营财务还是 License。
3. 涉及金额核对时，先统一账期、组织、客户、业务线、账号和交易类型。
4. 处理结算、调账、补偿、补建、License 激活等高风险事项前，已准备审批、账期、组织和影响范围说明。
5. 对外沟通前已完成金额、组织、账号、邮箱、流水号、订单号和 License 信息脱敏。

## 踩坑提示

- 不要在入口页直接判断金额是否正确，金额问题应进入月度账单、结算单、财务账户或交易记录核对。
- 用户账务、Provider 收益、客户账务、运营财务和 License 是不同视角，不要混用统计口径。
- 账期、组织、客户、业务线范围不一致时，看到的金额和状态会不同。
- 不记录真实组织、客户名、账号、邮箱、金额、订单号、流水号、License 注册码、激活码、Token 或 Key。

## 常见问题

#### 不知道从哪个账务入口开始看怎么办？

**问题现象：**

只知道“账务有问题”，但不确定应该进入用户账务、客户账务、运营财务还是 License。

**可能原因：**

问题对象没有先分类，用户余额、客户充值、Provider 收益、结算对账和资源授权被混在一起处理。

**处理方式：**

先判断问题关键词：余额、流水、充值订单、月度账单进入用户账务；收益、结算单、客户收益进入 Provider 收益；客户、标签、充值、业务线进入客户账务；账期、结算单、账户流水、巡检、调账进入运营财务；额度、授权、到期、激活进入 License。

#### 余额、结算单、月结金额不一致怎么办？

**问题现象：**

客户余额、结算单金额、月结总览或财务账户余额之间存在差异。

**可能原因：**

不同页面统计口径不同，或账期、组织、交易类型选择不一致，也可能存在未完成结算、退款、调账或巡检异常。

**处理方式：**

先统一账期和组织；再查看结算单列表确认单据状态；进入财务账户核对账户流水；最后进入巡检中心检查未配对划转、缺失收益明细或补偿队列。

#### License 和计费账务是什么关系？

**问题现象：**

平台资源不可用时，不确定应该查账务余额还是 License 状态。

**可能原因：**

账务余额用于处理客户或平台资金问题，License 用于处理平台资源授权和额度问题，两者影响方向不同。

**处理方式：**

如果问题表现为充值、扣费、结算或余额异常，进入账务的用户账务、客户账务或运营财务；如果问题表现为资源额度不足、模块受限、授权过期，进入 License 查看授权状态和有效期。

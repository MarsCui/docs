## 功能概述

`策略管理` 用于维护调度规则、资源优先级、地域偏好、限额和回退策略，用于约束模型部署时的地域选择、优先级和回退行为。

| 项目 | 内容 |
| --- | --- |
| 适用角色 | 运营方 |
| 导航路径 | 调度治理 > 策略管理 |
| 页面路由 | /operator/scheduling-governance/policies |
| 管理对象 | 调度规则、资源优先级、地域偏好、限额和回退策略 |
| 典型用途 | 治理跨云模型部署的资源选择和调度规则 |

### 新手理解

策略管理像多云部署的路线规则，决定首选哪个资源池、容量不足时怎么回退，以及不同租户或业务地域能不能跨云调度。

### 术语速查

| 术语 | 说明 |
| --- | --- |
| 调度策略 | 决定部署请求如何选择云资源的规则。 |
| 优先级 | 多个资源可用时的排序依据。 |
| 回退策略 | 首选资源不可用时的替代选择。 |
| 限额 | 控制租户或业务使用资源的上限。 |

## 前提条件

1. 资源池、租户授权和业务地域授权已完成。
2. 首选资源池、回退资源池和容量阈值已确认。
3. 跨地域调度策略已评估成本和合规影响。
## 页面说明

页面用于维护多云部署调度规则，包括资源优先级、地域偏好、容量阈值、回退策略和适用范围。运营方应结合资源池水位、业务地域授权和 SLA 要求配置策略。

## 主要操作

### 管理调度策略

1. 进入 `调度治理 > 策略管理`。
2. 按租户、业务地域、资源池或状态筛选策略。
3. 新增策略时设置适用范围、首选资源池和优先级。
4. 配置容量阈值、失败回退和限制条件。
5. 保存后通过测试部署查看实际调度结果。

## 参数说明

| 字段名称 | 是否必填 | 字段类型 | 示例 | 说明 |
| --- | --- | --- | --- | --- |
| 策略名称 | 必填 | 文本 | `prod-gpu-fallback` | 调度策略展示名称。 |
| 适用范围 | 必填 | 多选 | `tenant-a / 华东生产` | 策略作用的租户或业务地域。 |
| 首选资源池 | 必填 | 下拉选择 | `gpu-cn-shanghai-prod` | 优先调度的资源池。 |
| 回退资源池 | 否 | 多选 | `gpu-cn-hangzhou-backup` | 首选资源不可用时的替代资源。 |
| 容量阈值 | 否 | 数字 | `80%` | 超过阈值后触发降级或回退。 |

## 踩坑提示

- 回退资源池必须已完成租户和业务地域授权。
- 调度优先级变更只影响后续部署，已运行实例需按平台能力单独处理。
- 跨地域回退可能影响延迟、成本和数据合规边界。

## 排障路径

| 问题类型 | 先检查 | 下一步 |
| --- | --- | --- |
| 调度不到资源 | 策略优先级、回退资源池和库存 | 进入资源池确认可用容量 |
| 跨地域调度异常 | 业务地域授权、网络延迟和费用口径 | 调整授权或关闭跨地域回退 |
| 策略变更未生效 | 发布状态和新部署时间 | 使用测试部署验证后再通知用户 |

## 常见问题

### 部署没有命中首选资源池

**问题现象：**

调度结果落到回退资源池或其他云地域。

**可能原因：**

- 首选资源池容量不足。
- 业务地域授权不包含首选资源池。
- 策略优先级低于其他规则。

**处理方式：**

1. 检查资源池容量和启用状态。
2. 核对租户及业务地域授权。
3. 查看调度策略优先级和部署事件。

### 回退策略没有生效

**问题现象：**

首选资源池不可用时部署直接失败。

**可能原因：**

- 未配置回退资源池。
- 回退资源池未授权或未启用。
- 限制条件禁止跨地域或跨云回退。

**处理方式：**

1. 补充回退资源池配置。
2. 确认回退资源池授权和容量。
3. 检查策略限制条件。

## 后续操作

1. 监控资源池水位。
2. 复核业务地域授权。
3. 定期用测试部署验证调度策略。

## 注意事项

- 调度策略主要影响后续新部署，已运行服务是否变化取决于平台的重调度机制。
- 优先级、回退和跨地域规则会同时影响成本、延迟和资源隔离，应在发布前确认影响租户。
- 策略变更建议记录变更前后规则、时间和影响范围，便于回滚和审计。

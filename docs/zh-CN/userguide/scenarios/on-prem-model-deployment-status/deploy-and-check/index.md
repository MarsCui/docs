# 部署模型并检查状态

## 功能入口

- **角色**：模型提供方 / 最终用户
- **菜单**：AI 基础设施（On-Prem） > 模型部署 > 部署模板 / 模型实例
- **实例路由**：`/powerone/quickstart/model-service`

## 操作步骤

1. 在部署模板页面选择已发布的 NPU 推理模板。
2. 选择 1 卡、2 卡或 4 卡规格，并填写实例名称和模型参数。
3. 提交部署后进入 **模型实例**，按名称定位目标实例。
4. 实例长时间创建中时，查看作业事件、镜像拉取和资源调度结果。
5. 实例运行后，核对服务地址、健康状态和实际 NPU 占用。

![模型实例列表](./images/model-instances.png)

## 状态判断

| 状态 | 处理建议 |
| --- | --- |
| 创建中 | 等待镜像拉取和调度，持续查看作业事件 |
| 运行中 | 检查健康状态、访问入口和设备占用 |
| 排队中 | 检查租户配额、资源规格和剩余 NPU 数量 |
| 失败 | 检查镜像、启动参数、驱动、存储和多卡通信 |

## 4 张 NPU 卡的验证重点

- 4 卡实例应实际占用 4 张卡，不应只看模板名称。
- 如果 4 卡规格无法调度，先验证 1 卡，再验证 2 卡，最后验证 4 卡。
- 节点上总卡数足够不代表单个调度拓扑满足 4 卡需求。

## 完成检查

- 实例状态为运行中。
- 设备监控显示对应卡被该作业占用。
- 服务健康检查和一次最小调用成功。

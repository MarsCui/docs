# 部署模板

::: info 文档信息
版本：v1.0
更新日期：2026-07-08
:::

## 功能概述

`部署模板` 用于按向导选择模型、加速卡、业务参数和推荐规格，并生成模型服务部署配置。普通用户通过该页面把已发布模板转化为可运行的模型实例。

| 项目 | 内容 |
| --- | --- |
| 适用角色 | 普通用户 |
| 导航路径 | 模型部署 > 部署模板 |
| 页面路由 | `/powerone/quickstart/inference-template` |
| 管理对象 | 模型库、加速卡、业务参数、推荐规格、部署设置和预览信息 |
| 典型用途 | 基于运营方发布的模板选择模型、加速卡和规格，快速创建模型服务实例 |

### 新手理解

部署模板像模型服务的点单页：先选模型，再选加速卡，再确认规格和启动参数，最后提交创建模型实例。页面已经把运营方维护的模型、硬件和参数组合整理成向导，减少手动填写成本。

### 首次使用流程

1. 进入 `模型部署 > 部署模板`。
2. 在 `模型库（Model Library）` 中选择目标模型。
3. 在 `加速卡（Accelerators）` 中选择可用加速卡。
4. 检查 `业务参数（Business Parameters）` 和 `推荐规格（Recommended Specifications）`。
5. 进入 `部署设置（Deployment Settings）` 并确认实例名称、地域、规格等信息。
6. 在 `预览（Preview）` 中核对配置后提交创建模型实例。

### 术语速查

| 术语 | 说明 |
| --- | --- |
| 镜像 | 运行作业所需的容器环境，通常来自平台镜像服务或自定义镜像项目。 |
| 规格 | 作业可申请的资源套餐，例如 CPU、内存、GPU 型号和卡数。 |
| 业务参数 | 模型部署时需要确认的上下文长度、能力、启动参数或运行选项。 |
| 推荐规格 | 平台根据模型和加速卡组合推荐的资源套餐。 |

## 前提条件

1. 运营方已发布至少一个可见的推理模板。
2. 当前租户在目标地域下有可用配额。
3. 目标模型所需镜像和加速卡已完成适配。
4. 如部署会暴露服务，请提前确认访问范围和安全策略。

页面按向导展示模型库、加速卡、业务参数、推荐规格、部署设置和预览信息。截图中展示了模型库和加速卡选择区域。

| 字段/区域 | 说明 |
| --- | --- |
| 模型库 | 按模型厂商和模型名称展示可部署模型。 |

## 主要操作

### 创建模型实例

#### 适用场景

当需要把某个已发布模型部署成在线服务时，使用部署模板创建模型实例。

#### 操作前确认

1. 已确认模型在模板列表中显示为可部署。
2. 已确认目标加速卡和规格有可用配额。
3. 已确认部署后的服务访问方式和生命周期。

#### 操作步骤

1. 进入 `模型部署 > 部署模板`。
2. 在模型库中选择目标模型。
3. 在加速卡区域选择目标厂商和型号。
4. 检查业务参数和推荐规格，选择适合的实例类型。
5. 在部署设置中填写实例名称、地域、规格和启动参数。
6. 在预览页核对配置，确认无误后提交。

#### 参数说明

| 字段名称 | 是否必填 | 字段类型 | 示例 | 说明 |
| --- | --- | --- | --- | --- |
| 模板名称 | 系统生成 | 文本 | `qwen-vllm-template` | 可选推理模板名称。 |
| 推荐规格 | 系统生成 | 文本 | `1GPU-16C-64G` | 模板建议使用规格。 |
| 框架 | 系统生成 | 文本 | `vLLM` | 模板使用的运行框架。 |
| 启动参数 | 系统生成 | 文本 | `--max-model-len 8192` | 模板默认启动参数。 |
| 状态 | 系统生成 | 枚举 | `可用` | 模板是否可创建实例。 |
#### 踩坑提示

- 加速卡显示未适配（Unadapted）或未纳管（Unmanaged）时，可能无法直接部署。
- 推荐规格为空通常表示该模型和硬件组合没有可用资源或未完成配置。
- 不要随意修改启动参数，错误参数可能导致模型服务启动失败。

#### 结果校验

| 检查项 | 成功表现 | 异常时处理 |
| --- | --- | --- |
| 提交后进入 模型部署 > 模型实 | 提交后进入 `模型部署 > 模型实例`。 | 未达到时检查模板、规格、镜像、端口、环境变量和实例事件 |
| 确认新实例出现在列表中 | 确认新实例出现在列表中。 | 未达到时检查模板、规格、镜像、端口、环境变量和实例事件 |
| 实例状态进入创建中、运行中或符合 | 实例状态进入创建中、运行中或符合预期状态。 | 未达到时检查模板、规格、镜像、端口、环境变量和实例事件 |


## 配置规则与影响

- 模型、加速卡和规格之间存在适配关系，不能只按显存大小选择。
- 模板参数由运营方维护，普通用户只应调整业务明确需要的字段。
- 提交前必须核对地域和规格，避免实例创建到错误资源池。

## 常见问题

### 继续按钮不可用

**问题现象：**选择模型后无法进入下一步或推荐规格为空。

**可能原因：**

- 未选择加速卡。
- 模型与加速卡未适配。
- 当前租户没有对应规格配额。

**处理方式：**

1. 确认模型和加速卡都已选中。
2. 切换其他加速卡或实例类型。
3. 到 `资源配额` 查看是否有可用额度。

### 提交后模型实例启动失败

**问题现象：**实例创建后状态异常或无法提供服务。

**可能原因：**

- 镜像拉取失败。
- 启动参数不正确。
- 目标集群资源不足。

**处理方式：**

1. 进入模型实例详情查看状态、日志和事件。
2. 使用模板默认参数重新创建测试实例。
3. 联系运营方检查镜像、规格和集群资源。

## 后续操作

1. 进入 `模型部署 > 模型实例` 查看实例状态。
2. 根据服务地址进行联调或压测。
3. 持续关注用量和配额消耗。

## 注意事项

- 提交部署会创建资源并产生用量，提交前确认实例名称、规格和运行周期。
- 截图或工单中不要包含内部服务地址、访问密钥或敏感启动参数。

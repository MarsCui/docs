# 业务地域授权

## 功能概述

`业务地域授权` 用于维护业务地域、云资源池、租户范围和授权状态，支撑多云调度、资源授权和模型部署流程。

| 项目 | 内容 |
| --- | --- |
| 适用角色 | 运营方 |
| 导航路径 | 授权管理 > 业务地域授权 |
| 页面路由 | /operator/auth-management/business-region-auth |
| 管理对象 | 业务地域、云资源池、租户范围和授权状态 |
| 典型用途 | 把资源池能力开放给指定业务地域 |

### 新手理解

业务地域授权像把租户的资源使用权落到具体业务区域。租户有权限只是第一层，业务地域匹配后，部署请求才能被调度到正确云资源。

### 术语速查

| 术语 | 说明 |
| --- | --- |
| 业务地域 | 面向业务展示的资源区域。 |
| 授权范围 | 允许使用资源池的地域、租户或用户集合。 |
| 资源池映射 | 业务地域与云端资源池的关联关系。 |
| 生效状态 | 授权配置是否已被下游部署流程识别。 |

## 前提条件

1. 目标业务地域已创建。
2. 租户云授权和资源池已准备。
3. 业务地域与云地域映射关系已确认。
## 页面说明

页面用于将云资源池授权给指定业务地域，控制不同地域、业务线或环境可使用的资源范围。运营方应保证业务地域、云地域和调度策略之间口径一致。

## 主要操作

### 操作步骤

1. 进入 `授权管理 > 业务地域授权`。
2. 选择业务地域、租户或业务线。
3. 勾选可用云平台、云地域和资源池。
4. 设置启用状态、优先级或备注。
5. 保存后在部署页按业务地域验证资源可见性。

关键步骤截图：

![授权地域选择](./images/auth-regions.png)

授权前确认业务地域和资源范围。

### 参数说明

| 字段名称 | 是否必填 | 字段类型 | 示例 | 说明 |
| --- | --- | --- | --- | --- |
| 业务地域 | 是 | 下拉选择 | `华东生产` | 用户部署时选择的业务地域。 |
| 租户 | 条件必填 | 下拉选择 | `tenant-a` | 授权适用的租户范围。 |
| 云地域 | 是 | 下拉选择 | `cn-shanghai` | 实际承载部署的云厂商地域。 |
| 资源池 | 是 | 多选 | `gpu-cn-shanghai-prod` | 业务地域可调度资源。 |
| 优先级 | 否 | 数字 | `10` | 多个资源池可用时的选择顺序。 |

### 踩坑提示

- 业务地域名称相近时要核对编码，避免授权到错误环境。
- 云地域和业务地域不是同一个概念，不能直接混写。
- 资源池优先级变更会影响后续部署的调度结果。

### 结果校验

1. 业务地域授权记录已启用。
2. 用户选择该业务地域时能看到目标资源。
3. 调度策略按优先级选择预期资源池。

## 常见问题

### 业务地域下没有可用规格

**问题现象：**

用户选择业务地域后，规格或资源池为空。

**可能原因：**

- 该业务地域未绑定资源池。
- 资源池容量不足或停用。
- 租户云授权缺失。

**处理方式：**

1. 检查业务地域授权记录。
2. 确认资源池状态和容量。
3. 回到租户云授权页面核对上层授权。

### 部署被调度到非预期云地域

**问题现象：**

用户选择某业务地域后，部署落到了其他云地域。

**可能原因：**

- 业务地域绑定了多个云地域。
- 调度策略设置了回退资源池。
- 首选资源池容量不足。

**处理方式：**

1. 检查业务地域授权和资源池优先级。
2. 查看调度策略的回退配置。
3. 核对部署事件中的调度原因。

## 后续操作

1. 维护调度策略。
2. 验证用户快速部署流程。
3. 定期复核业务地域与云地域映射。

## 注意事项

- 业务地域和云地域不是同一概念。
- 资源池优先级会影响后续调度。
- 跨地域调度需评估延迟、成本和合规。

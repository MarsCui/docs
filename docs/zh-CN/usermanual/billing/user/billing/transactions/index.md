# 流水明细

::: info 文档信息
版本：v1.0
更新日期：2026-07-23
:::

## 功能概述

`流水明细` 页面标题显示为 `收支明细`，用于查询当前账号的账务收支记录。用户可按交易类型、收支类型和时间范围筛选，查看每一笔流水的发生时间、交易类型、上下文、额度变化、变更后额度、交易单号、备注和详情入口。

| 项目 | 内容 |
| --- | --- |
| 适用角色 | 用户侧账号、业务管理员、账务查看人员 |
| 导航路径 | 账务 > 用户账务 > 流水明细 |
| 页面路由 | `/billing/my/account/transactions/records` |
| 管理对象 | 收入流水、支出流水、交易上下文、交易单号 |
| 典型途径 | 追踪余额变化、核对充值到账、排查消费来源 |

#### 新手理解

流水明细像账户的小票列表：余额为什么增加、为什么减少，都应能在这里找到对应记录。排查余额问题时，不要只看账户概览，应进入流水明细按时间、收支方向和交易类型核对来源。

#### 术语速查

| 术语 | 含义 | 处理建议 |
| --- | --- | --- |
| 收支类型 | 表示本次流水是收入还是支出。 | 先看方向，再看交易类型。 |
| 交易类型 | 转账、消费、退款、调整等业务分类。 | 用于缩小排查范围。 |
| 变更额度 | 本次流水导致的 Credits 增减。 | 与变更后额度一起看。 |
| 交易单号 | 关联业务交易的单号。 | 不等同于支付流水号。 |
| 变更后额度 | 本次流水发生后的账户额度。 | 用于解释余额变化。 |

## 前提条件

1. 当前账号具备用户侧账务查看权限。
2. 已进入 `我的账务 > 流水明细`。
3. 已确认需要查询的交易时间范围。
4. 对外沟通时应使用脱敏后的流水号或交易单号。

## 页面说明

页面包含筛选区和流水列表。筛选区支持 `交易类型`、`收支类型` 和时间范围筛选，并提供 `搜索`、`重置` 等按钮。表格展示 `流水号`、`发生时间`、`收支类型`、`交易类型`、`上下文`、`变更额度`、`变更后额度`、`交易单号`、`备注` 和 `操作`。

下图展示流水明细列表，截图中的流水号、时间、上下文和金额必须脱敏处理。

| 区域 | 说明 |
| --- | --- |
| 交易类型 | 按业务交易类型筛选流水。 |
| 收支类型 | 按收入或支出方向筛选流水。 |
| 时间范围 | 按流水发生时间筛选。 |

| 详情 | 查看单笔流水的更多信息。 |

## 主要操作

### 查询流水明细

1. 进入 `账务 > 用户账务 > 流水明细`。
2. 在筛选区选择 `交易类型`。
3. 选择 `收支类型`。
4. 按需要选择时间范围。
5. 点击 `搜索` 查看结果。
6. 如需重新查询，点击 `重置` 清空条件。
7. 如仅学习或截图，只查看筛选条件和列表字段，不导出真实流水数据。

### 查看单笔流水

1. 进入 `账务 > 用户账务 > 流水明细`。
2. 在列表中定位目标流水。
3. 核对流水号、发生时间、收支类型、交易类型和上下文。
4. 查看变更额度、变更后额度、交易单号和备注。
5. 如存在行内 `详情` 入口，点击后查看更完整信息。
6. 对外沟通时隐藏真实流水号、交易单号、账号、金额和业务上下文。

## 踩坑提示

- 交易单号不等于支付流水号，也不一定等同于充值订单号。
- 单条流水只能解释一次额度变化，月度对账仍需进入月度账单核对汇总。
- 收入、支出、退款、冲正等方向容易混淆，先看交易类型再判断金额变化。
- 查询不到流水时先放宽时间范围，不要直接判断为数据丢失。
- 不记录真实账号、邮箱、订单号、流水号、金额、客户名、组织名、Token 或 Key。
- 截图、导出、工单和评论必须脱敏。

## 常见问题

#### 找不到目标流水

**问题现象：**

按交易类型、收支类型或时间范围查询后没有目标记录。

**可能原因：**

时间范围过窄、筛选条件不匹配，或相关交易尚未完成入账。

**处理方式：**

先清空筛选条件，再扩大时间范围；如仍找不到，使用脱敏后的订单号或业务记录联系运营方核对。

#### 流水方向与预期不同

**问题现象：**

某条记录显示为收入或支出，但与预期方向不一致。

**可能原因：**

交易类型、退款、冲正、转账或调整会影响收支方向。

**处理方式：**

查看交易类型和上下文；再结合交易单号、月度账单和账户概览判断口径。

#### 流水汇总和月度账单不一致

**问题现象：**

流水明细加总结果与月度账单汇总不同。

**可能原因：**

两边账期、时间范围、交易类型或统计口径不同。

**处理方式：**

统一账期和时间范围；刷新月度账单；必要时按交易类型分组核对。

## 注意事项

- 流水号、交易单号、上下文和金额属于敏感账务信息，对外提供前应脱敏。
- 不要仅凭单条流水判断整月费用，应结合月度账单核对。
- 发现异常时，先记录筛选条件和脱敏后的流水标识，再联系运营方处理。
- 学习或截图时只查看筛选条件、列表字段和详情入口，不导出真实流水数据。

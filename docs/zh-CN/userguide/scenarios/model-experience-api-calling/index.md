# 场景概览-模型的体验与调用

指导用户发现模型、体验模型并完成 API 调用。

## 适用角色

- 最终用户
- 查看客户调用结果的模型提供方

## 场景目标

- 在模型市场找到目标模型并领取或确认可用配额。
- 在 Playground 完成一次体验。
- 使用个人调用 Key 完成 API 调用并查看调用记录。

## 开始前准备

- 已完成登录并拥有目标模型的访问权限。
- 已准备个人调用 Key，且账户额度和模型配额可用。

## 推荐阅读顺序

1. 查看当前场景概览和准备条件
2. 按用户快速指引发现并体验模型
3. 完成 API 调用
4. 查看调用记录并处理失败分支

## 完成检查

- [ ] 用户能够在模型市场找到目标模型并查看能力、供应方和调用说明。
- [ ] Playground 请求成功，输入输出符合模型模态。
- [ ] API 请求使用个人授权密钥并获得成功响应。
- [ ] 我的调用中能找到对应时间和模型的调用记录。
- [ ] 提供方能在客户调用中看到脱敏后的同一调用统计。

## 常见失败分支

| 现象 | 优先检查 |
| --- | --- |
| 找不到模型 | 模型状态、公有/私有范围、租户授权和筛选条件 |
| Playground 失败 | 输入模态、参数、模型状态和账户额度 |
| API 返回认证错误 | Personal Key、Header 格式和模型授权 |
| API 成功但调用记录为空 | 时间范围、统计延迟、模型和账号筛选 |

# 我的部署

本任务用于检查云上模型服务的运行状态、事件、监控和 API，并在不用时停止资源。

## 场景目标

部署达到健康状态，受控调用成功，并且可以持续监控和正确释放资源。

## 适用角色

- 平台用户（End User）
- 需要把已验证部署发布为模型服务的模型提供方（Provider）

## 开始前准备

- 记录部署名称、云平台、地域和预期创建时间。
- 准备无副作用的测试请求；运行中的云资源可能持续产生费用。

## 操作步骤

### 发布模型（扩展操作）

1. 在「我的部署」列表中，找到目标部署任务，点击卡片上的 **"..."**（更多）按钮，选择 **"发布"**。
2. 选择发布区域：
   - **私有区**：仅团队 / 租户内部可见，适合业务与安全敏感场景；
   - **公有区**：上架公开目录，可独立定价与限流，供所有租户调用。
3. 进入发布配置页，完成以下配置：
   - 填写模型元数据：模型类型、模型源、地域、请求 URL、API 密钥、模型 ID 等；
   - 配置请求头（如 Authorization 认证信息）；
   - 设置模型参数：输入 / 输出模态，功能配置（函数调用 / 思考模式）、Token 限制，支持协议；
   - 补充基本信息（个性化标识、描述），选择发布方式（立即 / 定时）。
4. 确认配置无误后提交，模型将发布至对应区域，供其他用户调用。

进入**我的部署**后，先按名称、云平台和地域定位目标服务，再继续核对状态、事件和监控。

## 其他操作

| 操作名称 | 操作步骤 |
|----------|----------|
| 切换云平台 Tab | 点击页面顶部的云平台 Tab（华为云 / aws 亚马逊 / 阿里云 / 谷歌云 / AGIOne-powerone / 全部）→ 查看对应云平台下的部署任务 |
| 重置筛选 | 点击 **"重置"** 按钮 → 清空名称 / 状态 / 模型名称 三个筛选条件 |
| 查看部署详情 | 点击部署任务的 **"详情"** 按钮 → 查看部署基本信息、API 调用、监控信息、事件记录 |
| 查看 API 调用信息 | 在详情页切换至 **"API 调用"** 标签 → 查看请求 URL、请求方法、请求头及参数规范 |
| 启动部署 | 在部署卡片上，点击 **"启动"** 按钮 → 启动已停止的部署任务 |
| 停止部署 | 在部署卡片上，点击 **"停止"** 按钮 → 停止运行中的部署任务（节省费用） |
| 发布模型 | 点击部署卡片的 **"..."**（更多）按钮 → 选择 **"发布"** → 选择发布到私有区或公有区 |
| 删除部署 | 点击部署卡片的 **"..."**（更多）按钮 → 选择 **"删除"** → 删除操作不可逆，请谨慎操作 |

## 完成检查

> **用途：** 以下检查是当前功能任务的退出条件，用于判断操作结果是否可观察、可复核，以及是否可以继续当前场景的下一步。它不是操作步骤的重复；任一项不满足时，请按下方“常见失败分支”继续排查。

| 检查项 | 通过标准 |
| --- | --- |
| 1 | 运行状态和事件中没有重复失败。 |
| 2 | 监控时间范围与服务运行时段一致。 |
| 3 | 使用页面给出的 API 信息完成一次受控调用。 |
| 4 | 停止或删除后资源状态和费用占用按预期变化。 |

## 常见失败分支

| 现象 | 优先检查 |
| --- | --- |
| 部署长时间创建中或失败 | 事件记录、云账号、配额、容量、镜像拉取和启动命令 |
| 服务运行但调用失败 | API 地址、请求头、模型参数、服务端口和限流 |

## 功能概述

`充值记录` 用于查看当前租户的额度充值历史，包括授予金额、价值金额、来源、获取方式、第三方流水号和充值时间。

| 项目 | 内容 |
| --- | --- |
| 适用角色 | 普通用户 |
| 导航路径 | 配额&用量 > 充值记录 |
| 页面路由 | `/powerone/quota-usage/top-up-history` |
| 管理对象 | 额度充值记录、来源、获取方式、第三方流水号和充值时间 |
| 典型用途 | 查看租户额度或余额的充值历史，核对额度来源和到账情况 |

### 新手理解

充值记录像额度钱包的流水账，用来查看每次充值、扣减或调整的时间、数量和状态。

### 首次使用流程

1. 进入 `配额&用量 > 充值记录`。
2. 按条件搜索目标记录。
3. 查看授予金额、来源和时间。
4. 如没有数据，确认当前租户是否发生过充值或授予。
5. 对账时结合外部付款或审批记录。

### 术语速查

| 术语 | 说明 |
| --- | --- |
| 授予金额 | 平台授予到账户的额度数量。 |
| 价值金额 | 额度对应的价值金额。 |
| 来源 | 额度来源，例如人工授予、活动或外部系统。 |
| 第三方流水号 | 外部支付或业务系统中的流水标识。 |

## 前提条件

1. 当前账号具备查看充值记录的权限。
2. 当前租户发生过额度充值或授予。
3. 如用于对账，外部付款或审批记录已准备。

## 查询充值记录

### 适用场景

当需要确认额度是否到账、对账或追踪额度来源时，查询充值记录。

### 操作前确认

1. 明确要查询的时间范围或流水信息。
2. 确认当前账号查看的是目标租户。

### 操作步骤

1. 进入 `配额&用量 > 充值记录`。
2. 输入查询条件。
3. 点击 `搜索（Search）`。
4. 在表格中查看记录。
5. 如需恢复默认列表，点击 `重置（Reset）`。

### 参数说明

| 字段名称 | 是否必填 | 字段类型 | 示例 | 说明 |
| --- | --- | --- | --- | --- |
| 记录编号 | 系统生成 | 文本 | `topup-20260706-001` | 定位单笔充值或调整记录。 |
| 变更类型 | 系统生成 | 枚举 | `充值` | 展示充值、扣减、退款或人工调整类型。 |
| 变更数量 | 系统生成 | 数字 | `2000 Credits` | 本次额度变更的数量。 |
| 操作时间 | 系统生成 | 日期时间 | `2026-07-06 10:00` | 额度变更发生的时间。 |
| 生效状态 | 系统生成 | 状态 | `已生效` | 记录是否已经影响可用额度。 |
| 备注 | 否 | 文本 | `项目扩容` | 说明本次变更的业务背景或来源。 |

### 踩坑提示

- 无数据时先点击 `重置（Reset）` 排除筛选影响。
- 充值记录用于对账，不代表资源配额已经调整。

### 结果校验

1. 记录中的金额、来源和时间符合预期。
2. 第三方流水号能与外部系统匹配。

## 配置规则与影响

- 充值记录只反映额度变动，不反映 GPU、CPU 等资源配额。
- 额度到账和资源配额调整是不同概念。
- 对账时应保留外部流水号和时间。

## 常见问题

### 充值记录为空

**问题现象：**页面显示无数据（No Data）。

**可能原因：**

- 当前租户没有充值记录。
- 筛选条件过窄。
- 账号无权查看历史记录。

**处理方式：**

1. 点击 `重置（Reset）`。
2. 确认当前租户。
3. 联系运营方核对额度授予记录。

### 充值到账但资源仍不可创建

**问题现象：**有充值记录，但创建实例仍失败。

**可能原因：**

- 额度和资源配额不是同一项。
- 底层集群资源不足。
- 目标规格未开放。

**处理方式：**

1. 查看 `资源配额`。
2. 确认实例创建错误。
3. 联系运营方调整资源配额或规格。

## 后续操作

1. 充值后额度未变化时，核对记录状态和生效时间。
2. 发现异常扣减时，结合用量记录和运营方计量明细排查。
3. 需要对账时，按时间范围导出或记录充值流水。
4. 联系运营方时提供记录编号、时间、变更数量和当前额度截图。

## 注意事项

- 充值记录涉及额度和结算信息，不要在公开渠道展示完整截图。
- 记录已生效后仍可能因计量扣减导致余额变化，应同时查看用量。
- 对账时使用记录编号和时间范围，不要泄露内部账号或业务合同信息。

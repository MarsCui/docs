## 功能概述

`模型及AI服务` 汇总 AGIOne 面向模型发布、发现、体验、调用、审核和收益管理的能力。它把元模型、模型来源、模板、标签、模型市场、创作空间、体验中心、调用日志、收益明细和审核流程串成一条完整链路：运营管理员先维护基础数据和审核规则，模型提供方发布模型，模型调用方在市场和体验中心使用模型，平台再记录调用、收益和客户调用数据。

| 推荐入口 | 本文档、[模型及AI服务概览](../)、[从发布到调用模型](../end-to-end/publish-and-call-model/) |
| 关键对象 | 元模型、模型来源、模板、标签、BYOK、聚合模型、模型市场、体验中心、调用日志、收益、审核 |
| 典型用途 | 建立模型服务理解框架，明确发布、审核、调用、计费和收益的操作边界 |

## 模型及AI服务是什么

模型及AI服务作为模型资产从配置、发布、审核、上架、体验、调用到统计结算的一体化服务层，不直接替代上游模型服务或第三方 Endpoint，而通过标准化元模型、来源、协议、请求头、计费和限流配置，把不同来源的模型统一纳入模型市场和调用体系。

在模型及AI服务场景中，平台负责：

1. 维护模型基础信息，例如模型作者、元模型、模型类型、模态、协议和能力标签。
2. 管理模型接入方式，例如模型来源、Endpoint、请求头、BYOK 和协议连通性测试。
3. 承载模型部署、发布与审核，例如我的部署、我的模型、聚合模型、模型审核和应用审核。
4. 提供模型体验与调用，例如模型市场、体验中心、快速开始、调用统计和调用日志。
5. 记录用量、收益和客户调用数据。

## 模型发布与调用层级

| 层级 | 说明 | 影响范围 |
| --- | --- | --- |
| 元模型 | 描述基础模型能力、模态、协议、上下文和高级能力。 | 影响发布时可选择的模型基础信息。 |
| 模型来源 | 描述模型供应方、区域、请求 URL、认证请求头和来源渠道。 | 影响 BYOK 或第三方 Endpoint 接入。 |
| 模板 | 将供应方、元模型、协议、能力和默认参数组合成可复用配置。 | 影响发布流程的预置项和一致性。 |
| 标签 | 用于模型分类、场景筛选和模型市场展示。 | 影响用户发现和筛选模型。 |
| 发布模型 | 模型提供方创建的单模型服务配置。 | 影响模型市场、体验中心和 API 调用。 |
| 聚合模型 | 将多个已发布模型组合成统一入口，并按策略路由。 | 影响成本、成功率、体验和可用性。 |
| 审核 | 运营方对模型或应用发布进行确认。 | 影响模型是否可上架、可调用。 |
| 调用与收益 | 调用方使用模型后产生的日志、统计、用量和收益。 | 影响计费、结算、运营分析和排障。 |

推荐配置顺序是：先由运营管理员维护元模型、模型来源、模板和标签，再由模型提供方发布模型或聚合模型，之后提交审核，上架后由调用方在模型市场和体验中心使用。

## 使用前提

1. 已确认当前账号是模型提供方、调用方还是运营管理员。
2. 发布模型前已准备模型来源、协议、价格和安全说明。
3. 调用模型前已确认 Personal Key、Endpoint、额度和模型可见范围。
4. 通过 On-Cloud 快速部署提交后，可在 `模型及AI服务 > 创作空间 > 我的部署` 的 `On-Cloud` 列表查看部署记录。

#### 理解自检清单

- 能区分元模型、模型来源、模型模板和具体模型。
- 知道模型发布需要审核，调用需要有效凭据和可见范围。
- 知道调用失败时先查请求 ID、错误码、限流和来源健康。

## 踩坑提示

- 不要在模型来源未通过连通性测试前提交发布，否则审核或体验中心调用容易失败。
- 不要把调用方 Key、供应方 API Key 和模型来源请求头混用，它们属于不同鉴权层。
- 审核通过只代表允许发布，不代表调用方一定有额度、可见范围和生产接入条件。
- 入门学习只用于理解入口和流程，不执行真实发布、提交审核、上架、删除、禁用等高风险动作。
- 不在文档中写真实模型 ID、API Key、Endpoint、租户信息、用户信息、模型仓库地址或测试参数。

## 常见问题

#### 发布模型时找不到目标元模型或模型来源

**问题现象：**

模型提供方进入发布流程后，目标元模型、模型来源或模板不在下拉列表中。

**可能原因：**

- 运营管理员尚未创建或启用对应元模型。
- 模型来源配置不完整或与当前区域、供应方不匹配。
- 模板或标签未维护。
- 当前账号缺少发布或查看权限。

**处理方式：**

1. 请运营管理员检查 [元模型](../operator/settings/meta-models/) 和 [模型来源](../operator/settings/model-source/)。
2. 确认模板、标签和供应方信息已配置。
3. 刷新发布流程或重新进入创作空间。
4. 检查当前账号的菜单和发布权限。

#### 模型发布后无法上架或调用

**问题现象：**

模型已保存或已提交审核，但无法在模型市场展示，或调用时失败。

**可能原因：**

- 模型尚未通过审核或未执行上架操作。
- 协议连通性测试未通过。
- Endpoint、请求头、API Key 或模型源 ID 配置错误。
- 计费、限流或发布区域配置不符合预期。

**处理方式：**

1. 检查 [模型审核](../operator/approvals/model-reviews/) 状态。
2. 回到 [我的模型](../user/studio/my-models/) 查看发布、上架和详情信息。
3. 重新执行协议连通性测试。
4. 检查 Endpoint、请求头、模型源 ID、计费和限流配置。

# 场景概览 - 多云调度策略

本场景帮助运营方在云资源、租户和业务地域完成授权后，配置首选资源池、容量阈值和故障回退，让后续模型部署按明确的成本、容量和合规边界选择云资源。

## 适用角色

- 平台运营方、多云资源管理员、部署策略管理员

## 场景目标

- 调度策略具备明确适用范围、优先级和首选资源池。
- 回退资源池已经授权并具备可用容量。
- 跨地域或跨云回退的时延、成本和合规影响已评估。
- 测试部署能够命中预期资源池或给出明确失败原因。

## 场景流程

**主线：** 检查接入与授权 -> 设计首选和回退 -> 保存并启用策略 -> 执行测试部署 -> 核对调度事件与容量

| 阶段 | 核心结果 |
| --- | --- |
| 1. 检查基础 | 云账号、资源池和授权关系完整 |
| 2. 设计策略 | 适用范围、优先级、阈值和回退清晰 |
| 3. 验证调度 | 测试部署命中预期资源池或明确失败 |
| 4. 持续治理 | 容量、成本、时延和授权变化被持续复核 |

## 开始前准备

- 已完成[多云资源接入](../on-cloud-resource-access/)。
- 首选和回退资源池已完成租户及业务地域授权。
- 已明确容量阈值、SLA、成本和数据合规边界。
- 已准备一个低风险测试部署和验证时段。

## 推荐阅读顺序

1. 先检查接入总览、资源池和授权关系。
2. 再设计策略适用范围、优先级和回退条件。
3. 保存启用后执行测试部署。
4. 核对实际资源池、部署事件和回退结果。

## 完成检查

> **用途：** 以下检查是本场景的退出条件，用于确认策略已经影响真实调度结果，而不只是保存了一条配置记录。

| 检查项 | 通过标准 |
| --- | --- |
| 1 | 策略适用租户、业务地域和资源池范围正确。 |
| 2 | 首选和回退资源池均已授权、启用且容量可用。 |
| 3 | 优先级、容量阈值和约束符合 SLA、成本与合规要求。 |
| 4 | 测试部署命中预期资源池；回退或失败行为符合策略。 |
| 5 | 后续容量和策略复核责任人已明确。 |

# 场景概览-On Cloud 云资源接入

指导运营管理员将外部云平台资源（云厂商、云账号、地域）接入 AGIOne 平台，并完成面向业务与租户的授权分配，为后续的部署、调用与计量提供底层资源。

## 适用角色

- 运营管理员

## 场景目标

- 云平台、云账号和目标地域处于可用状态。
- 业务类型和租户获得正确的云资源授权。
- 普通用户能够看到被授权的云平台和地域。

## 开始前准备

- 已准备云平台类型、云账号和经过处理的访问凭据样例。
- 已明确目标地域、业务类型和租户范围。

## 推荐阅读路径

1. 接入云平台
2. 接入云账号和资源池
3. 配置业务地域授权
4. 配置租户云平台授权

## 完成检查

- [ ] 云平台、云账号和目标地域均处于可用状态。
- [ ] 业务类型已获得目标地域授权。
- [ ] 目标租户已获得云平台授权。
- [ ] 使用普通用户账号能看到被授权的云平台、地域和接入账号入口。

## 常见失败分支

| 现象 | 优先检查 |
| --- | --- |
| 云账号校验失败 | 云平台类型、账号权限、网络和凭据有效性 |
| 用户看不到地域 | 资源池状态、业务地域授权和租户云授权 |
| 授权保存后不生效 | 租户、业务类型、账号角色和重新登录 |

# 场景概览 - 服务商收益与结算

本场景帮助服务商从收益总览出发，按客户和模型解释收益来源，并核对月度结算单与到账状态。

## 适用角色

- 模型提供方、服务商财务查看人员、收益运营人员

## 场景目标

- 收益趋势能够对应到客户、模型和账期。
- 应结算、实际到账和转入状态能够核对。
- 余额转入前完成账期、金额和目标账户复核。

## 场景流程

**主线：** 查看收益总览 → 分析客户与模型贡献 → 核对结算单 → 确认到账或转入

| 阶段 | 核心结果 |
| --- | --- |
| 1. 查看汇总 | 账户余额、预估收益和账期趋势清晰 |
| 2. 解释来源 | Top 客户、Top 模型和客户明细可核对 |
| 3. 核对结算 | 应结算、实际到账和状态一致 |
| 4. 处理转入 | 金额、账期、账户和权限已复核 |

## 开始前准备

- 具备 Provider 收益查看权限。
- 明确目标账期和需要核对的客户或模型。
- 涉及余额转入时，已确认操作权限和复核人。

## 推荐阅读顺序

1. 收益总览。
2. 客户总览。
3. 月度结算单。
4. 必要时核对出账流水和余额转入。

## 完成检查

> **用途：** 以下检查是本场景的退出条件，用于确认收益来源和月度结算能够解释、能够复核，再执行余额转入或关闭账期。

| 检查项 | 通过标准 |
| --- | --- |
| 1 | 收益总览中的账期、余额和趋势已确认。 |
| 2 | 主要收益能够按客户和模型解释。 |
| 3 | 结算单的应结算、实际到账和状态正确。 |
| 4 | 转入操作前已核对目标账户、金额和权限。 |

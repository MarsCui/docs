# 场景概览-On Cloud 模型资产上架

指导运营管理员在完成云资源接入后，准备运行时资产（推理镜像、推理框架）并组装为可部署的模型资产，为后续的部署与调用提供业务侧资产。

## 适用角色

- 运营管理员

## 场景目标

- 推理镜像能够在目标云环境中拉取。
- 推理框架包含正确的端口和启动命令。
- 模型资产能够被普通用户用于快速部署。

## 开始前准备

- 已完成云资源接入和授权。
- 已准备镜像地址、框架类型、模型信息和输出 API 规则。

## 推荐阅读路径

1. 维护推理镜像
2. 维护推理框架
3. 在模型库组装可部署模型资产

## 完成检查

- [ ] 运行镜像可从目标云环境拉取，框架类型和版本正确。
- [ ] 推理框架的启动命令、端口和输出协议与模型匹配。
- [ ] 模型库条目已关联元模型、云侧部署点、算力方案和输出配置。
- [ ] 普通用户在快速部署中能看到该模型及可用方案。

## 常见失败分支

| 现象 | 优先检查 |
| --- | --- |
| 模型库无法选择镜像或框架 | 资产状态、框架类型、地域和关联关系 |
| 用户部署页看不到模型 | 模型发布状态、部署范围、授权和租户可见性 |
| 部署后接口不可用 | 启动命令、端口、输出配置和健康状态 |

# 快速部署

::: info 文档信息
版本：v1.0
更新日期：2026-07-21
:::

## 功能概述

`快速部署` 用于在模型库中选择可部署模型，并通过智能推荐部署流程选择推理引擎、业务策略和算力匹配方案，最终发起云端模型服务部署任务。

| 项目 | 内容 |
| --- | --- |
| 适用角色 | 普通用户 |
| 导航路径 | AI基础设施 > On-Cloud > 模型服务 > 快速部署 |
| 页面路由 | `/infrahub/user/model/store` |
| 管理对象 | 可部署范围、匹配模型、部署模式、推理引擎、业务策略、算力匹配方案、云账号和部署任务 |
| 典型途径 | 按已授权云账号和地域快速选择模型并发起模型部署 |

#### 新手理解

快速部署像从模型库里挑选一个可用模型，再让平台根据当前云账号、地域、策略和资源库存推荐部署方案。用户只需要在提交前核对模型、算力、费用和云账号，避免误发起真实部署任务。

#### 术语速查

| 术语 | 说明 |
| --- | --- |
| 可部署范围 | 当前账号可使用的云平台和地域范围。 |
| 匹配模型 | 在当前可部署范围和筛选条件下可部署的模型。 |
| 部署模式 | 部署形态，截图中可见 `单节点`、`验证、低并发、成本` 和 `高可用`。 |
| 推理引擎 | 用于承载模型服务的推理框架，截图中可见 `vLLM` 和 `SGLang`。 |
| 业务策略 | 推荐部署方案的排序偏好，截图中可见 `经济适用`、`卓越性能`、`现货速达` 和 `显卡数量`。 |
| 算力匹配方案 | 平台基于模型、地域、引擎和策略推荐的云平台、规格和费用组合。 |

## 前提条件

1. 当前账号具备 `快速部署` 和模型部署权限。
2. 已有可用云账号、云平台、地域、资源池和模型授权。
3. 目标模型、推理引擎、部署模式和算力方案已完成业务评估。
4. 部署前已确认费用、资源占用、任务名称和云账号信息均为脱敏内容。

## 页面说明

页面用于从模型库发起快速部署。用户可在 `可部署范围` 中选择云平台和地域，在 `匹配模型` 中按模型类型、系列、场景、关键词和排序方式筛选模型，并点击模型行的 `部署模型` 进入智能推荐部署页面。

页面截图：

![可部署范围](./images/deployable-scope.png)

智能推荐部署页面展示已选模型、部署模式、推理引擎、业务策略和模型算力匹配方案。用户确认推荐方案后，可打开 `发布部署` 弹窗填写云账号、任务名称和任务描述。

## 主要操作

### 部署模型

1. 进入 `AI Infra > On-Cloud > 模型服务 > 快速部署`。
2. 在 `可部署范围` 中选择当前可用云平台和地域，可使用 `全部云`、`全部地域` 或页面真实筛选项缩小范围。
3. 在 `匹配模型` 中按 `全模型`、`全系列`、`全场景`、搜索框或 `默认排序` 筛选模型。
4. 找到目标模型后，可先点击 `详情` 查看模型信息，确认无误后点击 `部署模型`。

![可部署范围](./images/deployable-scope.png)

5. 在 `智能推荐部署` 页面核对 `已选模型` 和 `部署模式`。
6. 选择 `推理引擎`，截图中可见 `vLLM` 和 `SGLang`。
7. 选择 `业务策略`，截图中可见 `经济适用`、`卓越性能`、`现货速达` 和 `显卡数量`。
8. 在 `模型算力匹配方案` 中查看推荐云平台、地域、GPU、CPU/内存和费用估算。
9. 确认方案后点击 `确认部署`，打开 `发布部署` 弹窗。

![智能推荐部署](./images/smart-deployment-recommendation.png)

10. 在 `发布部署` 弹窗中选择 `云账号`，填写 `任务名称` 和 `任务描述`。
11. 点击最终 `发起部署` 前，再次核对模型、云账号、地域、资源规格和费用影响。
12. 如仅学习或验证页面，请点击 `返回推荐` 或关闭弹窗，不提交真实部署任务。
13. 如执行真实提交，提交后可在 `我的部署` 列表中查看新建部署及其状态，也会在 `模型及AI服务 > 创作空间 > 我的部署` 的 `On-Cloud` 列表中展示。

![发布部署](./images/publish-deployment.png)

## 踩坑提示

- 截图未展示环境变量、启动参数、副本数、推理镜像或运行框架字段，本文不将这些内容写成已确认 UI 字段。
- 当前可部署范围由云账号、地域授权、资源池和模型授权共同决定，可选模型为空时不要直接判断为页面故障。
- 费用估算会随云平台、地域、资源规格和计费策略变化，提交前必须复核。
- `发起部署` 会创建真实部署任务，可能占用资源池容量并产生费用。
- 不要在文档、截图或工单中写入真实账号、密钥、Token、AK/SK、内网地址、镜像仓库凭据、模型存储路径、云资源 ID 或内部测试参数。

## 排障路径

| 问题类型 | 先检查 | 下一步 |
| --- | --- | --- |
| 没有可部署范围 | 云账号授权、地域授权和当前范围筛选条件。 | 点击 `刷新资源` 后重试，仍为空时联系运营方。 |
| 模型不可见 | 模型类型、系列、场景、关键词和模型授权。 | 点击 `默认排序` 或清空筛选后重新搜索。 |

## 常见问题

#### 模型或规格不可选

**问题现象：**

进入快速部署后，模型列表为空、部署模型入口不可见，或推荐方案为空。

**可能原因：**

- 当前云账号或地域未授权。
- 模型资产未授权给当前用户。
- 筛选条件过窄。
- 资源池容量不足，无法生成匹配方案。

**处理方式：**

1. 点击 `刷新资源` 并清空筛选条件。
2. 切换云平台、地域、模型类型、系列或场景后重新查看。
3. 联系运营方核对模型资产、云账号授权、资源池和配额。

#### 部署创建失败

**问题现象：**

点击 `发起部署` 后部署失败，或我的部署中显示创建失败。

**可能原因：**

- 推荐资源容量不足或已发生变化。
- 模型、推理引擎或算力方案不匹配。
- 云账号不可用或授权范围不足。
- 费用、配额或调度策略不满足部署要求。

**处理方式：**

1. 进入我的部署查看事件和错误信息。
2. 返回快速部署，调整云平台、地域、推理引擎或业务策略后重试。
3. 携带脱敏后的任务名称、时间、地域、模型和错误摘要联系运营方。

## 后续操作

1. 进入 `我的部署` 查看部署状态、事件和监控。
2. 部署成功后获取脱敏调用信息，并按权限进行测试调用。
3. 持续关注费用估算、资源占用、服务状态和业务访问结果。

## 注意事项

- 部署模型可能创建真实推理服务、占用资源池容量并产生费用。
- 错误的模型、推理引擎、云账号、地域、资源规格或任务配置可能导致部署失败或服务异常。
- `发起部署`、`部署`、`提交`、`确定` 属于高风险最终动作，文档只描述字段查看和提交前核对，不引导测试学习时提交。
- 截图或文档中不得展示真实账号、密钥、Token、AK/SK、内网地址、镜像仓库凭据、模型存储路径、云资源 ID、Endpoint、API Key 或内部测试参数。

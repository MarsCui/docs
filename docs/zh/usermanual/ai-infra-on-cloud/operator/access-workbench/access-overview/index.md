# InfraHub 接入总览

## 前言

| 项目   | 内容                                               |
| ---- | ------------------------------------------------ |
| 适用角色 | 运营方                                          |
| 导航路径 | 接入工作台 > 接入总览                                    |
| 功能定位 | 运营方的接入工作台首页，分 3 大区块汇总云资源接入、运营资源与授权状态的总体情况，并提供各资源模块的"打开"入口 |

## 页面结构

页面由 **3 个区块** 自上而下排列：① 接入底座（统计卡片）→ ② 运营资源清单（资源表）→ ③ 授权状态（卡片）。

### 接入底座

3 张统计卡片横向排列，每张卡片展示对应模块的总数、已接入项的标签云，以及指向明细列表的入口链接：

- **已接入云平台**：统计已接入的公有云 / 私有云厂商总数（如图 5），卡片下方按厂商显示图标（AGIOne-powerone / 阿里云 / 亚马逊 / 谷歌云 / 华为云 等），底部提供 **"接入云平台 →"** 入口。
- **已接入资源池**：统计已启用的地域总数（如图 7），卡片下方按云平台分组显示地域数量（如 华为云 1 / 谷歌云 2 / 亚马逊 1 / 阿里云 3），底部提供 **"接入资源池 →"** 入口。
- **已接入账号**：统计已接入的云账号总数（如图 4），卡片下方列出账号名称（如 `HuaweiOperatorAccount` / `google-prod-account` / `aws` / `aliyun-wh-dev`）及其所属云厂商，底部提供 **"接入账号 →"** 入口。

### 运营资源清单

一个 5 列资源清单表（资源 / 数量 / 前置条件 / 支撑 / 入口），按 **依赖链自下而上** 展示平台核心运营资源，每行提供 **"打开"** 入口跳转到对应模块：

| 资源 | 数量 | 前置条件 | 支撑 | 入口 |
| --- | --- | --- | --- | --- |
| 推理镜像 | 7788 | 云平台 > 资源池 | 登记云平台资源池下的运行镜像，供推理框架版本选择和后续推理运行使用。 | 打开 |
| 推理框架 | 47 | 云平台 > 资源池 > 接入账号 > 推理镜像 | 绑定云平台、资源池、接入账号和运行镜像，沉淀可模型配置选择的框架版本。 | 打开 |
| 模型/库模型 | 40 | 元模型 > 云平台 > 资源池 > 推理框架 | 关联元模型、云配置、资源规格和框架版本，作为策略配置与推理部署的模型入口。 | 打开 |
| 策略 | 1 | 模型/库模型 > 推理框架版本 | 基于模型云配置与框架版本编排主备路由，支撑高可用推理部署调度。 | 打开 |

> 注：每行的"前置条件"使用 `>` 分隔符表达**逐层依赖**，例如要创建"模型/库模型"，需先完成"元模型 + 云平台 + 资源池 + 推理框架"的配置。

### 授权状态

2 张授权卡片横向排列，分别展示两类授权的统计与明细，并提供 **"管理授权 →"** 入口：

- **租户-云平台授权**：副标题"统计租户是否已配置云平台授权"；展示 **租户总数 / 已授权租户数 / 未授权租户数** 3 项指标。
- **业务-资源池授权**：副标题"定义租户各业务类型可用的地域范围"；按业务类型展示已授权资源池数量与按云平台分布的地域数（如 推理部署 7：华为云 1 / 亚马逊 1 / …）。

## 操作步骤

### 查看接入总览

1. 进入平台首页，点击左侧导航栏的 **"接入工作台 > 接入总览"** 菜单，进入 InfraHub 接入总览页面。
2. 页面默认展示 3 大区块的当前数据：
   - 接入底座（云平台 / 资源池 / 账号的总数与已接入项）
   - 运营资源清单（推理镜像 / 推理框架 / 模型库 / 策略的数量与依赖链）
   - 授权状态（租户授权统计 + 业务-资源池授权分布）

![InfraHub 接入总览](./images/access-overview.png)


### 跳转到明细模块

接入底座 / 授权状态区块提供 **"→"** 入口跳转：

- 点击 **"接入云平台 →"** → 跳转到 [接入云平台](../../access-management/access-cloudtype/)
- 点击 **"接入资源池 →"** → 跳转到 [接入资源池](../../access-management/resource-pools/)
- 点击 **"接入账号 →"** → 跳转到 [接入账号](../../access-management/cloud-accounts/)
- 点击 **"管理授权 →"**（租户-云平台授权卡片）→ 跳转到 [租户-云平台授权](../../auth-management/tenant-cloud-auth/)
- 点击 **"管理授权 →"**（业务-资源池授权卡片）→ 跳转到 [业务-资源池授权](../../auth-management/business-region-auth/)

### 跳转到运营资源

运营资源清单中每行末尾的 **"打开"** 按钮跳转到对应模块：

- 点击 **推理镜像行的"打开"** → 跳转到 [推理镜像](../../deploy-assets/runtime-images/)
- 点击 **推理框架行的"打开"** → 跳转到 [推理框架](../../deploy-assets/frameworks/)
- 点击 **模型/库模型行的"打开"** → 跳转到 [模型库](../../deploy-assets/models/)
- 点击 **策略行的"打开"** → 跳转到 [策略管理](../../scheduling-governance/policies/)

## 其他操作

| 操作名称 | 操作步骤 |
|----------|----------|
| 快速接入 | 点击左侧导航栏的 **"快速接入"** 菜单（位于"接入总览"下方）→ 进入 5 步引导流程（接入云平台 → 租户-云平台授权 → 接入账号 → 接入资源池 → 业务-资源池授权）|
| 下钻明细 | 点击任一区块的 **"→"** 或资源行的 **"打开"** 按钮 → 跳转到对应模块的明细列表 |

## 注意事项

- **数据来源**：接入总览是聚合统计视图，数据来源于接入管理、授权管理与部署资产等子模块；任一子模块的变更会实时（或近实时）同步反映到总览视图
- **依赖链**：运营资源清单中每行的"前置条件"链是从下到上的强依赖关系；若上游资源缺失，对应行的入口跳转后可能因无可选项而无法继续配置
- **授权覆盖**：授权状态卡片仅展示统计概览，明细（已授权租户列表 / 已授权地域列表）需进入对应管理页查看
- **接入总览仅供运营方内部使用**，不直接面向租户
- 接入总览的"策略"资源来自 **调度治理** 分区，与"运营资源"中的"模型/库模型 / 推理框架"同属部署资源链，但由不同菜单路径管理

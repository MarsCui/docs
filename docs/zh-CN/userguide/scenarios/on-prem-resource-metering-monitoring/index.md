# 场景概览 - On-Prem 资源计量与监控

本场景介绍如何为租户分配 NPU 配额，并从设备、节点和作业维度检查资源使用。对于 4 张 NPU 卡，本场景负责回答“谁能用几张卡、当前哪张卡被谁占用、是否存在异常”。

## 适用角色

- 运营管理员
- 模型提供方和最终用户（查看自身用量）

## 场景目标

- 租户具备符合业务需要的 NPU 配额。
- 4 张 NPU 卡都能在设备监控中识别。
- 能将异常卡、节点和作业关联起来排查。

## 场景流程

**主线：** 配置租户配额 → 运行工作负载 → 关联设备节点与作业 → 核对计量并调整

| 阶段 | 核心结果 |
| --- | --- |
| 1. 配置配额 | 租户获得符合规划的 NPU 使用上限 |
| 2. 运行负载 | 1/2/4 卡任务按规格申请并占用资源 |
| 3. 关联监控 | 设备、节点和作业在同一时间范围内可对应 |
| 4. 核对调整 | 配额、实际占用、利用率和计量明细一致并可优化 |

## 开始前准备

- NPU 已完成纳管，监控采集组件正常上报。
- 已明确租户可用卡数和资源隔离策略。
- 已有至少一个测试作业用于验证计量与监控。

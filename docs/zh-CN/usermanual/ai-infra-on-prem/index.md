## 子系统定位

AI Infra On-Prem 面向本地或私有化算力资源，帮助运营方管理地域、可用区、集群、节点、存储、镜像、模板、配额和监控，帮助普通用户创建实例、作业并查看用量。

### 新手理解

On-Prem 像一座自有算力园区：运营方负责规划园区、接入机器和制定规则，普通用户在已开放的区域里申请资源、运行模型和查看状态。

## 核心术语速查

| 术语 | 说明 |
| --- | --- |
| 地域 / 可用区 | 资源按物理位置或逻辑隔离划分的层级。 |
| 集群 | 承载计算、存储和调度能力的 Kubernetes 集群。 |
| 节点 | 集群中实际运行实例或作业的服务器。 |
| 资源规格 | 用户创建实例或作业时选择的 CPU、内存、GPU/NPU 组合。 |
| 模板 | 把模型、框架、镜像、启动参数和显存规则组合成可部署方案。 |
| 配额 / 用量 | 控制可用资源上限，并记录实际消耗。 |

## 推荐阅读路径

1. 新手先读 [平台入门](./getting-started/) 理解资源层级和角色边界。
2. 运营方按地域、集群、规格、模板、配额顺序准备资源。
3. 普通用户按模板、镜像、存储、实例、监控顺序完成模型服务创建。
4. 需要完整流程时阅读 [从零开始部署一个模型服务](./end-to-end/deploy-model-service/)。

## 使用前提

1. 已明确当前账号是运营方还是普通用户。
2. 已选择目标地域、可用区或资源池。
3. 涉及实例创建前，已确认规格、镜像、存储、配额和集群能力。
4. 涉及故障排查时，准备脱敏后的时间范围、资源编号、错误信息和日志摘要。

## 常见问题

### 用户侧看不到资源

**问题现象：**

普通用户进入创建页或监控页后看不到目标地域、规格、镜像或集群数据。

**可能原因：**

- 运营方未开放资源可见范围。
- 地域、规格、镜像或模板没有完成关联。
- 当前账号缺少对应权限。

**处理方式：**

1. 先确认当前地域和账号角色。
2. 对照前提条件检查规格、模板、镜像和配额。
3. 联系运营方核对资源开放范围和账号权限。

### 实例创建失败

**问题现象：**

用户提交实例、作业或模型服务后，任务排队、失败或无法启动。

**可能原因：**

- 配额不足、集群容量不足或规格不可调度。
- 镜像拉取、启动命令、环境变量或存储挂载异常。
- 模板参数与模型能力不匹配。

**处理方式：**

1. 查看实例事件、日志和用户侧监控。
2. 核对镜像地址、启动命令、存储路径和资源规格。
3. 仍无法定位时，提供脱敏资源编号和失败时间给运营方。

## 后续操作

1. 运营方持续维护资源池、模板、配额和监控。
2. 普通用户创建资源后及时查看用量、日志和监控。
3. 发布前定期检查文档截图、示例命令和敏感信息脱敏情况。

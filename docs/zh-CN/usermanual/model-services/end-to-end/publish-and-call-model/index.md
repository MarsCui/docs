# 从发布到调用模型

::: info 文档信息
版本：v1.0
更新日期：2026-07-08
:::

## 功能概述

本文把运营管理员、模型提供方和模型调用方的模型及AI服务操作串成一条端到端路径：运营管理员先维护元模型、模型来源、模板和标签，模型提供方发布单模型或聚合模型并提交审核，运营管理员完成审核后，模型调用方在模型市场和体验中心查看、体验和调用模型，最后各角色查看调用、用量和收益数据。

| 项目 | 内容 |
| --- | --- |
| 适用角色 | 运营管理员、模型提供方、模型调用方 |
| 推荐前置阅读 | [模型及AI服务快速入门](../../getting-started/)、[元模型](../../operator/settings/meta-models/)、[我的模型](../../user/studio/my-models/)、[我的部署](../../user/studio/my-deployments/) |
| 输出结果 | 一个已配置、已审核、可在模型市场查看并可体验调用的模型 |
| 典型用途 | 新模型上架、第三方 Endpoint 接入、聚合模型发布、审核验收和调用排障 |

#### 新手理解

从发布到调用模型像把模型放上货架再完成试用购买：运营方先准备元模型、来源和模板，提供方发布模型，调用方在市场、体验中心和 API 中验证。

## 端到端流程图

| 阶段 | 操作方 | 目标 |
| --- | --- | --- |
| 基础数据准备 | 运营管理员 | 维护元模型、模型来源、模板和标签。 |
| 单模型发布 | 模型提供方 | 配置元模型、来源、Endpoint、协议、计费和限流。 |
| On-Cloud 快速部署分支 | 模型提供方 | 先通过 On-Cloud 快速部署创建部署记录，再从我的部署选择发布区域并进入发布模型页面。 |
| 聚合模型发布 | 模型提供方 | 选择成员模型，配置路由策略、计费和发布方式。 |
| 审核处理 | 运营管理员 | 审核模型或应用发布申请。 |
| 市场展示 | 模型调用方 | 在模型市场查看详情、供应方、快速开始和性能。 |
| 体验调用 | 模型调用方 | 在体验中心选择模型并完成文本、图像、视频或语音体验。 |
| 数据查看 | 各角色 | 查看我的调用、调用日志、用量明细、模型收益和客户调用。 |
| 排障闭环 | 各角色 | 根据失败路径检查基础配置、审核状态、Endpoint、计费、限流和调用日志。 |

#### 术语速查

| 术语 | 说明 |
| --- | --- |
| 元模型 | 定义协议、模态、Token 限制和默认参数的能力抽象。 |

## 前提条件

1. 运营管理员具备模型服务设置、应用列表、模型审核和应用审核权限。
2. 模型提供方具备创作空间、发布模型、创建聚合模型和提交审核权限。
3. 模型调用方具备模型市场、体验中心和我的调用权限。
4. 发布模型所需的 Endpoint、API Key、模型源 ID、协议和默认参数已准备并可连通。
5. 如通过 On-Cloud 快速部署进入链路，当前账号具备 `AI Infra > On-Cloud > 模型服务 > 快速部署` 和 `模型及AI服务 > 创作空间 > 我的部署` 权限。
6. 计费、限流、发布区域和审核流程已明确。
7. 文档和截图中所有密钥、Endpoint 内部地址、请求 ID 和敏感调用参数均已脱敏。

## 参数说明

| 字段名称 | 是否必填 | 字段类型 | 示例 | 说明 |
| --- | --- | --- | --- | --- |
| 元模型 | 是 | 文本 | 示例元模型 A | 定义模型协议、模态、上下文和能力边界。 |
| 模型来源 | 是 | 文本 | 示例来源 A | 记录 Endpoint、认证、请求头和连通性配置。 |
| 发布模型 | 是 | 文本 | 示例模型 A | 模型提供方提交审核和上架的目标模型。 |

| 调用凭据 | 是 | 文本 | PERSONAL-KEY-001 | 调用方体验和 API 接入使用的凭据，占位符示例不代表真实密钥。 |
| 调用日志 | 系统生成 | 文本 | REQ-202607130001 | 排查 401、429、5xx、超时和输出异常的关键线索。 |

## 踩坑提示

- 元模型、模型来源和模型模板是发布前置配置，缺一项都可能导致发布表单不可选。
- 审核通过只代表允许上架，不代表调用方一定具备额度、可见范围和有效 Key。
- 体验中心调通不等于生产接入完成，正式调用前还要核对限流、计费、错误日志和调用凭据。
- Endpoint、API Key、请求头和内部错误日志属于敏感信息，写入文档前必须脱敏。
- On-Cloud 快速部署、选择发布区域和模型发布可能改变真实服务暴露范围；学习或验证页面时不要执行最终 `发布`、`提交`、`保存` 或 `创建`。
- `发布`、`提交审核`、`审核通过`、`驳回`、`上架`、`创建聚合模型`、`删除` 属于高风险动作，执行前必须确认影响范围和回退方案。
- 学习或截图场景只查看页面、字段、状态和跳转，不执行最终发布、提交审核、审核通过、驳回、上架、创建聚合模型或删除动作。
- 不记录真实模型 ID、API Key、Endpoint、模型来源凭据、租户信息、用户信息、调用日志或测试参数。

## 步骤 4：模型提供方发布单模型

1. 进入 `创作空间 > 我的模型`。
2. 切换到 `我的发布`，点击 `发布模型`。
3. 选择发布区域，例如发布到私有区或发布到公有区。
4. 在基本信息中选择元模型和模型来源，填写请求 URL、API Key、模型源 ID、请求头、输入输出模态、高级能力和 Token 限制。
5. 选择支持协议，并完成协议连通性测试。
6. 填写个性化标识、描述和发布方式。
7. 进入计费配置，选择免费或按 Token 计费，配置阶梯价格、缓存命中、联网搜索和免费额度。
8. 进入限流配置，设置 RPM、TPM 或选择不限制。
9. 点击 `仅保存` 或 `提交审核`。

On-Cloud 快速部署分支：

1. 进入 `AI Infra > On-Cloud > 模型服务 > 快速部署`。
2. 按页面流程选择模型、模型版本、资源池、地域、实例规格和副本数，提交前核对资源占用、费用和服务暴露范围。
3. 提交后，部署记录展示在 `模型及AI服务 > 创作空间 > 我的部署` 的 `On-Cloud` 列表。
4. 在 [我的部署](../../user/studio/my-deployments/) 中找到目标部署，点击发布入口并选择发布区域。
5. 选择发布区域后，页面跳转到 [我的模型](../../user/studio/my-models/) 的发布模型页面，继续核对基础信息、计费配置、调用配置和可见范围。
6. 如仅学习或验证页面，只确认部署记录、发布区域弹窗和跳转结果，不执行最终 `发布`、`提交` 或 `保存`。

## 失败分支与排查路径

#### 失败分支：模型审核被驳回

下一跳：[模型审核](../../operator/approvals/model-reviews/)

**问题现象：**

模型提供方提交后无法发布，审核记录显示驳回。

**可能原因：**

模型说明、价格、样例、使用边界或授权材料不完整，或模型来源、协议、请求头和安全策略不符合审核要求。

**处理方式：**

1. 阅读审核意见，补齐模型说明、价格、样例、使用边界和授权材料。
2. 核对模型来源、协议、请求头和安全策略。
3. 重新提交前先在受控环境验证来源连通性和输出质量。

#### 失败分支：模型来源连通失败

下一跳：[模型来源](../../operator/settings/model-source/)

**问题现象：**

模型来源测试失败，或发布后体验中心无法调用。

**可能原因：**

Endpoint、认证请求头、API Key 或返回格式配置错误，也可能被上游限流、白名单或网络访问策略拦截。

**处理方式：**

1. 检查 Endpoint、认证请求头、API Key 和返回格式。
2. 确认上游服务没有限流、白名单或网络访问限制。
3. 用脱敏请求样例验证通过后，再关联模板或发布模型。

#### 失败分支：调用返回 401/429/5xx

下一跳：[我的调用日志](../../user/my-calls/call-logs/)

**问题现象：**

调用方通过 API 或体验中心调用失败，返回认证、限流或上游错误。

**可能原因：**

Personal Key、API Key、请求头或模型可见范围不匹配，或触发 QPM、Token、额度、供应方限流和上游服务异常。

**处理方式：**

1. 401 先检查 Personal Key、API Key、请求头和模型可见范围。
2. 429 先检查 QPM、Token 限制、客户额度和供应方限流。
3. 5xx 先查看调用日志中的请求 ID、延迟和模型来源健康状态。

## 结果校验

| 检查项 | 成功表现 | 异常时处理 |
| --- | --- | --- |
| 模型可见 | 模型市场能搜索到目标模型 | 进入详情和快速开始 |
| 状态正常 | 模型审核通过，体验中心能返回结果 | 进入正式调用或收益跟踪 |
| 排障入口可用 | 审核记录、调用日志或来源测试能定位错误 | 按失败分支继续排查 |

## 常见问题

#### 模型审核被驳回后怎么处理？

**问题现象：**

模型提交后审核未通过，调用方无法在模型市场使用该模型。

**可能原因：**

模型说明、标签、来源、模板、计费配置、测试结果或安全材料不完整。

**处理方式：**

查看审核意见；补齐模型说明、示例、来源、模板和测试材料；重新提交前先确认敏感信息已脱敏。

#### 模型发布成功但调用失败怎么办？

**问题现象：**

模型已经可见，但体验中心或 API 调用返回 401、429、5xx 或超时。

**可能原因：**

API Key、Token、Endpoint、限流策略、模型服务状态或上游模型来源异常。

**处理方式：**

先核对调用凭据、Endpoint 和请求参数；再查看调用日志、限流配置、模型部署状态和模型来源连通性。

#### 用量和收益数据为什么对不上？

**问题现象：**

调用记录、用量明细、模型收益或客户调用统计中的数量、时间或金额不一致。

**可能原因：**

统计时间范围、模型版本、客户范围、计费规则或数据同步时间不同。

**处理方式：**

统一时间范围、模型版本和客户范围；先看调用日志，再看用量明细和收益统计；仍不一致时保留脱敏线索联系运营。

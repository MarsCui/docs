# 模型发布审批

本场景指导运营管理员审核模型提供方提交的发布申请，并验证审批结果、上架范围和后续可见性。

## 适用角色

- 处理模型审核的平台运营方（Operator）
- 跟踪审核结果的模型提供方（Provider）

## 场景目标

- 审核人能识别申请对应的模型、提供方、发布区域和版本。
- 模型信息、协议连通性、计费、限流和展示范围经过检查。
- 通过或驳回结果附带清晰理由并可被提供方查看。
- 通过后的模型在预期范围内可见、可体验或可调用。

## 开始前准备

1. 当前账号具备 Model Services 模型审核权限。
2. 已确认平台的发布、计费、限流和内容治理规则。
3. 申请中的连接验证已完成，测试材料不包含真实凭据。

## 操作流程

| 步骤 | 操作 | 参考手册 | 完成标志 |
| --- | --- | --- | --- |
| 1 | 进入 `审批 > 模型审核` 并筛选待处理申请 | [模型审核](../../../usermanual/model-services/operator/approvals/model-reviews/) | 找到目标申请且状态为待处理 |
| 2 | 核对模型、提供方、来源和发布区域 | [模型审核](../../../usermanual/model-services/operator/approvals/model-reviews/) | 基础信息与申请目标一致 |
| 3 | 核对协议测试、模态、Token、计费和限流 | [从发布到调用模型](../../../usermanual/model-services/end-to-end/publish-and-call-model/) | 关键配置完整且符合规则 |
| 4 | 通过申请或填写原因后驳回 | [模型审核](../../../usermanual/model-services/operator/approvals/model-reviews/) | 状态和处理意见保存成功 |
| 5 | 使用提供方账号查看申请状态 | [我的模型](../../../usermanual/model-services/user/studio/my-models/) | 提供方能看到结果和原因 |
| 6 | 对通过的公有模型进行可见性验证 | [模型市场](../../../usermanual/model-services/user/discover/models/) | 目标用户能在预期范围看到模型 |

开始审核时，先在**模型审核**列表中按提供方、版本、发布范围、提交时间和状态定位申请。

![在模型审核列表中定位申请](./images/model-reviews-list.png)

## 完成检查

> **用途：** 以下检查是当前功能任务的退出条件，用于判断操作结果是否可观察、可复核，以及是否可以继续当前场景的下一步。它不是操作步骤的重复；任一项不满足时，请按下方“常见失败分支”继续排查。

| 检查项 | 通过标准 |
| --- | --- |
| 1 | 审批对象、版本、提供方和发布范围正确。 |
| 2 | 协议、计费和限流配置具备可验证依据。 |
| 3 | 审批状态、意见、处理人和时间可追溯。 |
| 4 | 驳回原因能指导提供方修改后重新提交。 |
| 5 | 通过后的模型只在预期范围内可见。 |

## 常见失败分支

| 现象 | 优先检查 |
| --- | --- |
| 找不到申请 | 状态筛选、提供方、提交时间和账号权限 |
| 无法通过审批 | 必填配置、协议测试、发布区域和审批权限 |
| 通过后模型不可见 | 发布时间、模型状态、发布区域和用户可见范围 |
| 驳回后提供方不知道如何修改 | 审批意见是否指出具体字段、规则和验证方式 |

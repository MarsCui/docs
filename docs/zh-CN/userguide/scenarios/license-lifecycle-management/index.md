# 场景概览 - License 生命周期管理

本场景帮助运营方从首次激活开始，持续检查 License 状态、有效期、授权额度和纳管对象，并在到期、扩容或重新授权前完成影响评估和复核。

## 适用角色

- 平台运营方、License 管理员、资源运营人员

## 场景目标

- License 与目标环境、实例和授权类型一致。
- 激活后状态、有效期、总额度、已用和剩余额度可验证。
- 纳管对象与授权构成能够解释额度占用。
- 到期或容量不足前有明确续期、扩容和业务处理计划。

## 场景流程

**主线：** 确认环境与当前状态 -> 选择激活方式 -> 执行激活 -> 核对授权构成 -> 监控有效期与额度 -> 续期或扩容

| 阶段 | 核心结果 |
| --- | --- |
| 1. 识别当前状态 | 环境、注册码、授权类型和现有状态明确 |
| 2. 完成激活 | 激活方式和授权结果与目标实例匹配 |
| 3. 核对使用 | 授权构成、纳管对象和额度占用可解释 |
| 4. 持续治理 | 到期、剩余额度和扩容计划持续可见 |

## 开始前准备

- 具备 `账务 > License > License` 页面权限。
- 已确认当前页面属于目标环境和目标实例。
- 已选择在线支付激活或注册码/激活码方式。
- 激活、续期和扩容前已确认授权范围及业务影响。

## 推荐阅读顺序

1. 先查看 License 管理中的当前状态、有效期和额度。
2. 再按实际情况选择在线支付或激活码流程。
3. 激活后核对详情、授权构成和纳管对象。
4. 建立到期与剩余额度巡检，提前安排续期或扩容。

## 完成检查

> **用途：** 以下检查是本场景的退出条件，用于确认 License 已在正确实例生效且可支撑预期资源规模，不以“激活按钮返回成功”作为唯一标准。

| 检查项 | 通过标准 |
| --- | --- |
| 1 | 环境、实例、授权类型和激活材料彼此匹配。 |
| 2 | License 状态有效，生效和到期时间正确。 |
| 3 | 总额度、已用、剩余额度和授权构成能够解释。 |
| 4 | 纳管对象与实际资源一致，无未知占用。 |
| 5 | 到期、扩容和剩余额度预警责任人及处理计划明确。 |

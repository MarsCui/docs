## 功能概述

设置汇总 AGIOne 的账号、成员、角色、组织、登录安全、操作日志和 API 流控配置入口。运营管理员通过它维护平台级权限和系统策略，普通用户通过它查看个人账号、项目、Key、额度申请和组织规则。

| 项目 | 内容 |
| --- | --- |
| 阅读对象 | 普通用户、平台运营、安全管理员和流控管理员 |
| 阅读目标 | 理解设置子系统的账号、成员、角色、组织、登录安全、操作日志和 API 流控入口 |
| 适用入口 | 设置概览、用户侧设置、运营管理员设置、登录配置和 API 流控概览 |
| 使用前提 | 当前账号已具备设置菜单权限，并已明确要处理的配置对象和风险范围 |
| 推荐路径 | 先读设置概览，再按个人账号、成员权限、组织规则、登录安全或接口流控进入对应功能页 |

### 新手理解

设置像平台的控制面板：先确认自己是谁，再确认团队里有哪些人、每个人有什么角色，最后再配置组织规则、登录安全和 API 流控。新手不要直接改配置，应先判断问题属于个人账号、成员权限、组织规则、登录安全还是接口流控。

## 设置是什么

设置是平台账号、权限、组织、安全策略和 API 流控的统一管理入口。它不替代具体业务功能页，而是决定用户能否登录、能看到哪些菜单、能操作哪些对象，以及系统策略如何生效。

## 使用前提

1. 当前账号已具备设置菜单权限。
2. 已明确本次要处理的是个人账号、成员角色、组织、日志、系统配置还是 API 流控。
3. 涉及删除、重置、授权、发布、导出或登录策略变更前，已确认影响范围和回退方式。
4. 对外沟通前，确认账号、邮箱、手机号、Key、AK/SK、token 和内部地址已脱敏。

## 常见问题

### 不知道先看用户侧还是运营侧怎么办？

**问题现象：**

进入设置后，不确定该看用户侧设置还是运营管理员设置。

**可能原因：**

用户侧更关注当前账号和组织内可见对象，运营侧更关注平台级成员、组织、系统策略和 API 流控。

**处理方式：**

处理自己的账号、Key、项目和额度，先看用户侧；处理成员、角色、组织、登录配置或流控规则，进入运营管理员侧。

### 看不到某个设置菜单怎么办？

**问题现象：**

左侧导航没有目标菜单，或页面打开后没有操作按钮。

**可能原因：**

当前账号缺少对应角色权限，或当前组织上下文不包含该对象。

**处理方式：**

先确认当前账号和组织；再让管理员核对角色授权；必要时重新登录后刷新菜单。

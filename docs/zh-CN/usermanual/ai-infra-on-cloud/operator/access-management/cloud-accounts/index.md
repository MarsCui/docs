# 接入账号

## 功能概述

`接入账号` 用于维护阿里云、AWS、华为云、Google Cloud 或私有云等云厂商账号凭据。

| 项目 | 内容 |
| --- | --- |
| 适用角色 | 运营方 |
| 导航路径 | 接入管理 > 接入账号 |
| 页面路由 | /infrahub/op/access/account |
| 管理对象 | 云账号、云平台、Access Key、授权状态和资源访问权限 |
| 典型用途 | 接入云厂商账号，为资源池发现、模型部署和云端资源管理提供认证 |

### 新手理解

云账号像平台访问云厂商资源的钥匙串。钥匙要能开门，但权限不能过大；文档只说明字段含义，真实 AK/SK、账号 ID 和授权策略只能在安全输入框或密钥系统中维护。

### 术语速查

| 术语 | 说明 |
| --- | --- |
| Access Key ID | 云厂商访问密钥 ID，用于标识访问身份。 |
| Secret Key | 与 Access Key ID 配套的敏感密钥，只能在平台安全输入框中保存。 |
| 云平台 | 账号所属云厂商或私有云平台类型。 |
| 最小权限 | 仅授予当前业务需要的资源访问和部署权限。 |

## 前提条件

1. 当前账号具备云账号接入权限。
2. 云厂商账号、AK/SK 或角色授权已准备并符合最小权限。
3. 资源同步地域和授权策略已确认。
## 页面说明

页面用于接入和维护云厂商账号、凭据校验状态、资源同步范围和授权策略。运营方需要按云平台、账号用途和地域范围管理账号，避免把生产账号、测试账号和客户账号混用。

## 主要操作

### 操作步骤

1. 进入 `接入管理 > 云账号`。
2. 按云平台、账号状态或账号名称筛选目标账号。
3. 新增账号时选择云平台并填写账号名称、授权方式和资源范围。
4. 在安全输入框录入访问密钥或选择已有密钥引用。
5. 保存后执行连通性校验，并检查资源同步状态。

关键步骤截图：

![添加云账号](./images/add-account.png)

添加时使用脱敏示例，不在截图中暴露 AK/SK 或账号 ID。

### 参数说明

| 字段名称 | 是否必填 | 字段类型 | 示例 | 说明 |
| --- | --- | --- | --- | --- |
| 账号名称 | 是 | 文本 | `prod-cloud-account` | 平台内展示名称，建议体现用途和云平台。 |
| 云平台 | 是 | 下拉选择 | `阿里云` | 账号所属云厂商或私有云平台。 |
| 授权方式 | 是 | 枚举 | `AK/SK` | 访问密钥、角色授权或平台支持的其他方式。 |
| 资源范围 | 条件必填 | 多选 | `cn-shanghai` | 允许同步和使用的地域或资源范围。 |
| 校验状态 | 系统生成 | 枚举 | `校验通过` | 由平台根据凭据和云厂商接口返回。 |

### 踩坑提示

- 不要把 AK/SK、Secret Key 或账号 ID 写入文档、截图和工单评论。
- 生产账号建议使用最小权限策略，不要直接使用全局管理员权限。
- 账号停用前先确认资源池、授权和部署实例是否仍在引用。

### 结果校验

1. 账号列表展示新账号且校验状态正常。
2. 资源同步任务能获取目标地域资源。
3. 资源池和授权页面可以选择该账号下的资源。

## 常见问题

### 账号校验失败

**问题现象：**

保存云账号后，校验状态显示失败或资源无法同步。

**可能原因：**

- 访问密钥无效或已过期。
- 授权策略缺少查询资源的权限。
- 云厂商接口、代理或私有云 Endpoint 不可达。

**处理方式：**

1. 在密钥系统中更新凭据。
2. 核对云厂商授权策略和最小权限清单。
3. 检查网络、代理、证书和云平台连通性。

### 同步不到目标地域资源

**问题现象：**

账号校验通过，但资源池页面没有目标地域或资源。

**可能原因：**

- 资源范围没有包含目标地域。
- 云厂商资源标签或类型不符合同步规则。
- 同步任务尚未完成或失败。

**处理方式：**

1. 检查账号资源范围配置。
2. 确认云厂商侧资源已创建且状态可用。
3. 查看同步任务状态并按错误信息处理。

## 后续操作

1. 创建或同步云资源池。
2. 配置租户云授权和业务地域授权。
3. 在部署资产页面维护可部署模型和运行环境。

## 注意事项

- AK/SK、Secret Key 和账号 ID 不得出现在截图或工单正文中。
- 云账号校验通过后仍需确认资源同步结果。
- 生产账号变更前先评估资源池和部署影响。

## 子系统定位

Model Services 面向模型发布、审核、体验、调用、日志和收益分析，帮助模型提供方发布模型，帮助调用方发现和使用模型，帮助运营方治理模型能力和审核流程。

### 新手理解

Model Services 像模型能力市场和运营后台：模型提供方把模型整理上架，调用方在模型市场和 Playground 试用，运营方维护元模型、来源、模板和审核规则。

## 核心术语速查

| 术语 | 说明 |
| --- | --- |
| 元模型 | 描述模型协议、模态、Token 限制和默认参数的能力抽象。 |
| 模型来源 | 实际调用模型的 Endpoint、认证方式和请求头配置。 |
| 供应方 | 提供模型能力、价格和服务质量的主体。 |
| Personal Key | 用户用于调用模型服务的个人访问凭据。 |
| 审核单 | 模型或应用发布前的审核记录。 |
| Playground | 用于文本、图像、音频或视频体验模型效果的页面。 |
| 调用日志 | 记录请求 ID、错误码、延迟、Token 和脱敏摘要的排障入口。 |
| 收益统计 | 按模型、客户和时间统计调用收益。 |

## 推荐阅读路径

1. 新手先读 [平台入门](./getting-started/) 理解模型发布、审核和调用关系。
2. 运营管理员先维护元模型、来源、模板和标签。
3. 模型提供方发布模型并查看审核结果。
4. 调用方从模型市场进入 Playground，再根据调用日志和用量收益持续优化。
5. 需要完整流程时阅读 [从发布到调用模型](./end-to-end/publish-and-call-model/)。

## 使用前提

1. 已确认当前账号是模型提供方、调用方还是运营管理员。
2. 发布模型前已准备模型来源、协议、价格、限流和安全说明。
3. 调用模型前已确认 Personal Key、Endpoint、额度和模型可见范围。
4. 查看收益或客户调用前，已确认当前账号具备对应数据权限。

## 常见问题

### 模型发布后不可见

**问题现象：**

模型提交后，在模型市场或我的模型列表中找不到。

**可能原因：**

- 模型仍在审核中或发布任务未完成。
- 可见范围、供应方或标签配置不正确。
- 模型来源、模板或元模型配置不完整。

**处理方式：**

1. 查看我的模型中的审核记录和发布状态。
2. 核对可见范围、来源和模板配置。
3. 审核通过但仍不可见时，联系运营方检查发布索引。

### 模型调用失败

**问题现象：**

Playground 或 API 调用返回认证失败、限流、超时或上游错误。

**可能原因：**

- Personal Key、API Key 或请求头无效。
- 模型来源 Endpoint 不可用。
- 触发限流、Token 上限或计费额度限制。

**处理方式：**

1. 确认调用示例使用占位符替换为有效凭据。
2. 查看调用日志中的请求 ID、错误码和延迟。
3. 联系供应方或运营方核对来源健康、限流和额度。

## 后续操作

1. 运营管理员定期检查元模型、来源和模板的一致性。
2. 模型提供方发布后跟踪调用质量、收益和客户反馈。
3. 调用方正式接入前在 Playground 和调用日志中完成验证。

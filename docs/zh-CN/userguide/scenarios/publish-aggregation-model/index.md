# 场景概览-发布聚合模型

本场景介绍如何把多个已发布模型组合成统一调用入口，并配置路由策略、计费方式和发布范围。

## 适用角色

- 模型提供方

## 场景目标

- 选择可参与聚合的基础模型。
- 配置成功率、成本、均衡、随机或轮询等路由策略。
- 提交聚合模型并进入审核或发布流程。

## 开始前准备

- 至少两个状态可用、协议兼容的已发布模型。
- 已明确路由策略、计费方式和发布范围。

## 推荐阅读路径

1. 确认基础模型和协议
2. 配置聚合策略
3. 配置计费并提交

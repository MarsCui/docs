# 场景概览 - 模型的体验与调用

指导用户发现模型、体验模型并完成 API 调用。

## 适用角色

- 平台用户
- 查看客户调用结果的模型提供方

## 场景目标

- 在模型市场找到目标模型并领取或确认可用配额。
- 在与模型模态对应的文本、图像、视频或语音 Playground 完成一次体验。
- 使用个人调用 Key 完成 API 调用并查看调用记录。

## 场景流程

**主线：** 发现模型并取得配额 → 在线体验 → API 接入 → 回看调用记录

| 阶段 | 核心结果 |
| --- | --- |
| 1. 发现模型 | 找到符合任务的模型、供应方和可用配额 |
| 2. 在线体验 | 使用无副作用输入验证模型能力和输出模态 |
| 3. API 接入 | 使用个人授权密钥完成一次受控请求 |
| 4. 回看记录 | 用户调用日志和提供方客户统计能定位同一请求 |

## 开始前准备

- 已完成登录并拥有目标模型的访问权限。
- 已准备个人调用 Key，且账户额度和模型配额可用。

## 推荐阅读顺序

1. 查看当前场景概览和准备条件
2. 按用户快速指引发现模型，并选择与模型模态一致的体验中心
3. 完成 API 调用
4. 查看调用记录并处理失败分支

## 完成检查

> **用途：** 以下检查是本场景的退出条件，用于判断是否已经取得可观察、可复核的结果，以及是否可以进入下一场景。它不是操作步骤的重复；任一项不满足时，请返回对应功能文档的“常见失败分支”排查。

| 检查项 | 通过标准 |
| --- | --- |
| 1 | 用户能够在模型市场找到目标模型并查看能力、供应方和调用说明。 |
| 2 | Playground 请求成功，输入输出符合模型模态。 |
| 3 | API 请求使用个人授权密钥并获得成功响应。 |
| 4 | 我的调用中能找到对应时间和模型的调用记录。 |
| 5 | 提供方能在客户调用中看到脱敏后的同一调用统计。 |

## 常见失败分支

| 现象 | 优先检查 |
| --- | --- |
| 找不到模型 | 模型状态、公有/私有范围、租户授权和筛选条件 |
| Playground 失败 | 输入模态、参数、模型状态和账户额度 |
| API 返回认证错误 | Personal Key、Header 格式和模型授权 |
| API 成功但调用记录为空 | 时间范围、统计延迟、模型和账号筛选 |

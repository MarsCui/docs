## 功能概述

`在线IDE` 用于创建和管理交互式开发环境。普通用户可选择单节点或集群形态，基于平台镜像和规格启动开发环境，用于代码调试、数据处理和实验验证。

| 项目 | 内容 |
| --- | --- |
| 适用角色 | 普通用户 |
| 导航路径 | 开发资源 > 在线IDE |
| 页面路由 | `/powerone/dev-training/dev-ide` |
| 管理对象 | 在线开发环境、单节点 IDE、集群 IDE、镜像、规格和运行状态 |
| 典型用途 | 创建交互式开发环境，在线调试代码、运行 Notebook 或准备训练和推理脚本 |

### 新手理解

在线 IDE 像运行在资源池里的云端开发机。你不需要本地安装 GPU 环境，只要选择镜像和规格，平台会创建一个可访问的开发环境。

### 首次使用流程

1. 进入 `开发资源 > 在线IDE`。
2. 查看现有 IDE 列表和状态。
3. 点击 `创建 IDE（Create IDE）`。
4. 选择 `单节点（Single Node）` 或 `集群（Cluster）` 部署类型。
5. 继续填写镜像、规格、存储和启动参数。
6. 创建后返回列表查看状态并进入 IDE。

### 术语速查

| 术语 | 说明 |
| --- | --- |
| 镜像 | 运行作业所需的容器环境，通常来自平台镜像服务或自定义镜像项目。 |
| 规格 | 作业可申请的资源套餐，例如 CPU、内存、GPU 型号和卡数。 |
| 配额 | 租户可使用的资源上限，常见维度包括 GPU、CPU、内存和规格。 |
| 单节点（Single Node） | 单节点开发环境，适合常规调试和小规模实验。 |
| 集群（Cluster） | 集群开发环境，适合需要主从节点或分布式资源的场景。 |

## 前提条件

1. 当前账号具备创建在线 IDE 的权限。
2. 目标地域有可用规格和配额。
3. 已有可用镜像，且镜像中包含所需开发工具。
4. 如需挂载数据或代码目录，相关存储已准备。

## 主要操作

### 创建在线 IDE

#### 适用场景

当需要一个可交互的开发、调试或 Notebook 环境时，创建在线 IDE。

#### 操作前确认

1. 已确认目标规格和配额可用。
2. 已确认镜像包含 Python、CUDA、框架或其他开发依赖。
3. 已确认运行周期，避免长时间空转消耗资源。

#### 操作步骤

1. 进入 `开发资源 > 在线IDE`。
2. 点击 `创建 IDE（Create IDE）`。
3. 在创建页选择 `单节点（Single Node）` 或 `集群（Cluster）`。
4. 点击 `填写参数（Fill Parameters）` 进入参数填写。
5. 选择地域、镜像、规格、存储等配置。
6. 提交后返回列表等待 IDE 进入可用状态。

下图展示在线 IDE 创建页，用户先选择部署类型，再进入参数填写。

![创建在线 IDE](./images/create-ide.png)

#### 参数说明

| 字段名称 | 是否必填 | 字段类型 | 示例 | 说明 |
| --- | --- | --- | --- | --- |
| 环境名称 | 必填 | 文本 | `ide-pytorch` | 在线 IDE 展示名称。 |
| 镜像 | 必填 | 下拉选择 | `pytorch:2.3` | 开发环境镜像。 |
| 资源规格 | 必填 | 下拉选择 | `1GPU-16C-64G` | IDE 使用的算力规格。 |
| 存储挂载 | 否 | 路径 | `/mnt/workspace` | 工作目录挂载路径。 |
| 启动命令 | 否 | 文本 | `jupyter lab` | IDE 启动命令。 |
#### 踩坑提示

- 集群模式可能需要更多配额，普通调试优先使用单节点。
- 镜像缺少依赖时，IDE 能启动但代码可能运行失败。
- 长时间不用的 IDE 应及时停止或释放。

#### 结果校验

| 检查项 | 成功表现 | 异常时处理 |
| --- | --- | --- |
| 创建后列表中出现新 IDE | 创建后列表中出现新 IDE。 | 未达到时检查镜像、规格、启动命令、挂载路径和实例事件 |
| 状态进入运行中或可访问 | 状态进入运行中或可访问。 | 未达到时检查镜像、规格、启动命令、挂载路径和实例事件 |
| 能够打开 Web IDE、Jup | 能够打开 Web IDE、JupyterLab 或对应开发入口。 | 未达到时检查镜像、规格、启动命令、挂载路径和实例事件 |

### 搜索和刷新 IDE

#### 适用场景

当 IDE 较多或状态变化后，需要快速定位目标环境。

#### 操作步骤

1. 在筛选区选择地域或状态。
2. 输入关键字后点击 `搜索（Search）`。
3. 如筛选后无结果，点击 `重置（Reset）`。
4. 点击 `刷新（Refresh）` 获取最新状态。

#### 踩坑提示

- 刚提交创建后状态刷新可能有延迟。
- 筛选条件过窄时，已存在的 IDE 也可能看不到。


## 配置规则与影响

- 在线 IDE 会占用配额和集群资源，运行时间越长消耗越多。
- 镜像、规格和存储由运营方配置，普通用户只能选择可见项。
- 停止、删除或释放 IDE 前，应确认代码和输出已保存到持久化目录。

## 常见问题

### 创建 IDE 按钮不可用或看不到

**问题现象：**列表页没有创建入口。

**可能原因：**

- 当前账号没有创建权限。
- 所选地域未开放在线 IDE。
- 页面加载或权限状态异常。

**处理方式：**

1. 刷新页面。
2. 确认当前地域。
3. 联系运营方检查账号权限和资源池配置。

### IDE 创建后无法打开

**问题现象：**列表有 IDE，但访问入口打不开或页面一直加载。

**可能原因：**

- 实例仍在启动。
- 镜像启动脚本异常。
- 网络入口或代理不可用。

**处理方式：**

1. 等待状态进入运行中。
2. 查看实例日志或事件。
3. 联系运营方检查集群入口和镜像配置。

## 后续操作

1. 进入 IDE 编写和调试代码。
2. 将重要代码、数据和输出保存到持久化存储。
3. 不用时停止或释放 IDE。

## 注意事项

- 不要把账号密码、token 或私钥写入 Notebook、代码仓库或截图。
- 创建前确认存储挂载路径，避免输出只保存在临时容器目录。

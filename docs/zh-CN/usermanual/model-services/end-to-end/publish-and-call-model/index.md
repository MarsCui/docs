# 从发布到调用模型

:::: info 文档信息
版本：v1.0
更新日期：2026-07-06
::::

## 功能概述

本文把运营管理员、模型提供方和模型调用方的 Model Services 操作串成一条端到端路径：运营管理员先维护元模型、模型来源、模板和标签，模型提供方发布单模型或聚合模型并提交审核，运营管理员完成审核后，模型调用方在模型市场和体验中心查看、体验和调用模型，最后各角色查看调用、用量和收益数据。

| 项目 | 内容 |
| --- | --- |
| 适用角色 | 运营管理员、模型提供方、模型调用方 |
| 推荐前置阅读 | [平台入门](../../getting-started/)、[元模型](../../operator/settings/meta-models/)、[我的模型](../../user/studio/my-models/) |
| 输出结果 | 一个已配置、已审核、可在模型市场查看并可体验调用的模型 |
| 典型用途 | 新模型上架、第三方 Endpoint 接入、聚合模型发布、审核验收和调用排障 |

## 新手理解

从发布到调用模型像把模型放上货架再完成试用购买：运营方先准备元模型、来源和模板，提供方发布模型，调用方在市场、Playground 和 API 中验证。

## 端到端流程图

| 阶段 | 操作方 | 目标 |
| --- | --- | --- |
| 基础数据准备 | 运营管理员 | 维护元模型、模型来源、模板和标签。 |
| 单模型发布 | 模型提供方 | 配置元模型、来源、Endpoint、协议、计费和限流。 |
| 聚合模型发布 | 模型提供方 | 选择成员模型，配置路由策略、计费和发布方式。 |
| 审核处理 | 运营管理员 | 审核模型或应用发布申请。 |
| 市场展示 | 模型调用方 | 在模型市场查看详情、供应方、快速开始和性能。 |
| 体验调用 | 模型调用方 | 在体验中心选择模型并完成文本、图像、视频或语音体验。 |
| 数据查看 | 各角色 | 查看我的调用、调用日志、用量明细、模型收益和客户调用。 |
| 排障闭环 | 各角色 | 根据失败路径检查基础配置、审核状态、Endpoint、计费、限流和调用日志。 |

## 术语速查

| 术语 | 说明 |
| --- | --- |
| 元模型 | 定义协议、模态、Token 限制和默认参数的能力抽象。 |
| 模型来源 | 实际访问模型能力的 Endpoint、认证和请求头配置。 |
| 审核单 | 模型或应用发布前的审核记录和处理意见。 |
| 调用日志 | 排查 401、429、5xx、超时和输出异常的请求记录。 |

## 前提条件

1. 运营管理员具备模型服务设置、应用列表、模型审核和应用审核权限。
2. 模型提供方具备创作空间、发布模型、创建聚合模型和提交审核权限。
3. 模型调用方具备模型市场、体验中心和我的调用权限。
4. 发布模型所需的 Endpoint、API Key、模型源 ID、协议和默认参数已准备并可连通。
5. 计费、限流、发布区域和审核流程已明确。
6. 文档和截图中所有密钥、Endpoint 内部地址和敏感调用参数均已脱敏。

## 步骤 1：运营管理员维护元模型

1. 进入 `设置 > 元模型`。
2. 如缺少模型作者，先添加模型作者。
3. 点击添加元模型，配置模型名称、模型类型、输入模态、输出模态、高级能力、Token 限制、协议和元模型详情。
4. 提交后确认元模型在列表中可见。

结果校验：

1. [元模型](../../operator/settings/meta-models/) 中目标元模型可见。
2. 模型类型、输入输出模态、协议和上下文上限符合发布需求。
3. 模型提供方发布时能选择该元模型。

## 步骤 2：运营管理员维护模型来源

1. 进入 `设置 > 模型来源`。
2. 点击添加模型来源。
3. 填写供应方、区域、请求 URL、请求头、认证信息和来源说明。
4. 提交后确认模型来源在列表中可见。

结果校验：

1. [模型来源](../../operator/settings/model-source/) 中目标来源可见。
2. 来源区域、请求 URL 和请求头配置准确。
3. 发布模型时能选择或复用该来源。

## 步骤 3：运营管理员维护模板和标签

1. 进入 `设置 > 模板`。
2. 添加模板，选择模型供应方、作者、元模型、协议、能力和默认参数。
3. 进入 `设置 > 标签`，维护模型市场和发布流程中需要使用的分类标签。
4. 确认模板和标签均处于可用状态。

结果校验：

1. [模板](../../operator/settings/model-templates/) 中目标模板可见。
2. [标签](../../operator/settings/tags/) 中目标标签可见。
3. 发布模型或聚合模型时可选择对应模板和标签。

## 步骤 4：模型提供方发布单模型

1. 进入 `创作空间 > 我的模型`。
2. 切换到 `我的发布`，点击 `发布模型`。
3. 选择发布区域，例如发布到私有区或发布到公有区。
4. 在基本信息中选择元模型和模型来源，填写请求 URL、API Key、模型源 ID、请求头、输入输出模态、高级能力和 Token 限制。
5. 选择支持协议，并完成协议连通性测试。
6. 填写个性化标识、描述和发布方式。
7. 进入计费配置，选择免费或按 Token 计费，配置阶梯价格、缓存命中、联网搜索和免费额度。
8. 进入限流配置，设置 RPM、TPM 或选择不限制。
9. 点击 `仅保存` 或 `提交审核`。

结果校验：

1. [我的模型](../../user/studio/my-models/) 中目标模型可见。
2. 协议连通性测试通过。
3. 模型保存或提交审核成功。

## 步骤 5：模型提供方创建聚合模型

1. 进入 `创作空间 > 我的模型`。
2. 切换到 `我的聚合`，点击 `创建聚合模型`。
3. 选择发布区域。
4. 选择模型类型和模型子类型。
5. 添加 2 个及以上已发布成员模型。
6. 为成员模型配置启用状态、最低成功率、最高并发率、上下文最大长度、输入 Token 成本和输出 Token 成本。
7. 填写个性化标识，选择匹配策略、标签、描述和发布方式。
8. 配置计费方式、计费模式和价格。
9. 点击 `仅保存` 或 `提交审核`。

结果校验：

1. [我的模型](../../user/studio/my-models/) 的我的聚合列表中目标聚合模型可见。
2. 成员模型数量不少于 2 个。
3. 匹配策略、计费配置和发布方式符合预期。

## 步骤 6：运营管理员处理模型审核

1. 进入 `审批 > 模型审核`。
2. 查看待审核模型列表，按模型名称、状态或提交信息筛选。
3. 打开目标模型详情，检查元模型、来源、Endpoint、协议、计费、限流、标签和发布区域。
4. 根据审核结果执行通过或拒绝。

结果校验：

1. [模型审核](../../operator/approvals/model-reviews/) 中目标审核记录状态更新。
2. 审核通过后模型可进入上架或展示流程。
3. 审核拒绝时原因明确，模型提供方可据此修改。

## 步骤 7：模型调用方在模型市场查看模型

1. 进入 `发现 > 模型市场`。
2. 使用模型名称、模型类型、供应方、标签或搜索框筛选模型。
3. 打开目标模型详情，查看供应方、快速开始、性能和概览。
4. 记录调用前需要确认的协议、价格、上下文、能力和限制。

结果校验：

1. [模型市场](../../user/discover/models/) 中目标模型可见。
2. 模型详情页的供应方、快速开始、性能和概览信息完整。
3. 调用方能根据快速开始信息准备接入。

## 步骤 8：模型调用方在体验中心试用模型

1. 进入 `体验中心 > 文本对话`，或根据模型类型进入图像生成、视频生成、语音生成。
2. 点击模型选择，选择目标模型。
3. 配置参数，例如 Temperature、Top-P、Max Tokens、Stream 等。
4. 输入测试内容并发起生成。
5. 检查响应内容、耗时、错误信息和模型效果。

结果校验：

1. [文本对话](../../user/playground/text/) 可选择目标模型。
2. 模型能返回符合预期的响应。
3. 如果调用失败，错误信息可用于定位 Endpoint、协议、限流或计费问题。

## 步骤 9：查看调用、用量和收益

1. 调用方进入 `我的调用 > 概览` 查看整体调用趋势。
2. 调用方进入 `我的调用 > 调用统计` 和 `调用日志` 查看模型维度和单次请求详情。
3. 模型提供方进入 `用量与收益 > 模型收益` 查看收益概览和收益详情。
4. 模型提供方进入 `用量与收益 > 用量明细` 查看消耗明细。
5. 模型提供方进入 `客户调用` 查看客户维度的概览、统计和日志。

结果校验：

1. [我的调用概览](../../user/my-calls/overview/) 中能看到调用数据。
2. [调用日志](../../user/my-calls/call-logs/) 中能定位单次请求。
3. [模型收益](../../user/usage-revenue/model-revenue/) 和 [用量明细](../../user/usage-revenue/model-usage/) 中能查看收益与消耗。

## 常见失败路径排查

### 失败分支：模型审核被驳回

下一跳：[模型审核](../../operator/approvals/model-reviews/)

**问题现象：**模型提供方提交后无法发布，审核记录显示驳回。

**排查路径：**

1. 阅读审核意见，补齐模型说明、价格、样例、使用边界和授权材料。
2. 核对模型来源、协议、请求头和安全策略是否符合要求。
3. 重新提交前先在受控环境验证来源连通性和输出质量。

### 失败分支：模型来源连通失败

下一跳：[模型来源](../../operator/settings/model-source/)

**问题现象：**模型来源测试失败，或发布后 Playground 无法调用。

**排查路径：**

1. 检查 Endpoint、认证请求头、API Key 和返回格式。
2. 确认上游服务没有限流、白名单或网络访问限制。
3. 用脱敏请求样例验证通过后，再关联模板或发布模型。

### 失败分支：调用返回 401/429/5xx

下一跳：[我的调用日志](../../user/my-calls/call-logs/)

**问题现象：**调用方通过 API 或 Playground 调用失败，返回认证、限流或上游错误。

**排查路径：**

1. 401 先检查 Personal Key、API Key、请求头和模型可见范围。
2. 429 先检查 QPM、Token 限制、客户额度和供应方限流。
3. 5xx 先查看调用日志中的请求 ID、延迟和模型来源健康状态。

## 完成后检查

| 检查项 | 成功表现 | 下一步 |
| --- | --- | --- |
| 模型可见 | 模型市场能搜索到目标模型 | 进入详情和快速开始 |
| 状态正常 | 模型审核通过，Playground 能返回结果 | 进入正式调用或收益跟踪 |
| 排障入口可用 | 审核记录、调用日志或来源测试能定位错误 | 按失败分支继续排查 |

## 后续操作

1. 将已验证的元模型、来源、模板、Endpoint、协议和默认参数整理成团队规范。
2. 为高频模型沉淀稳定的标签、计费、限流和快速开始说明。
3. 定期检查调用日志、收益、客户调用和失败率。
4. 对废弃模型执行下架或删除前，先确认调用方影响范围和替代方案。

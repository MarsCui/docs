# Meta-Models

Meta-models define reusable model identity and capability metadata before providers publish concrete services.

## Procedure

1. Open `Settings > Meta-Models` and create or select the model author.
2. Maintain model family, type, variants, modalities, capabilities, and token limits.
3. Configure supported native protocols and descriptive details.
4. Save and confirm that the meta-model is available to templates and provider publishing.

See [Meta-Models in the User Manual](../../../../usermanual/model-services/operator/settings/meta-models/).

## Completion Checklist

- [ ] Author, model family, and variants use stable naming.
- [ ] Modalities, token limits, capabilities, and protocols match the real model.
- [ ] Providers and templates can select the meta-model.

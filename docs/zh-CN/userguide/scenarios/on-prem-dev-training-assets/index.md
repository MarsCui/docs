# 场景概览 - On-Prem 开发训练与资产沉淀

本场景介绍用户如何创建开发环境、提交训练任务，并把模型、镜像、数据集和输出结果沉淀为可复用资产。

## 适用角色

- 平台用户（End User）
- 负责镜像、存储和资源配置的运营管理员

## 场景目标

- 开发环境和训练任务能够使用已授权算力。
- 数据和输出写入持久存储。
- 训练产物能够进入后续模型部署流程。

## 场景流程

**主线：** 准备存储与镜像 → 创建开发环境 → 执行训练或批处理 → 沉淀并复用资产

| 阶段 | 核心结果 |
| --- | --- |
| 1. 准备依赖 | 数据、输出目录、镜像、算力和配额可用 |
| 2. 开发调试 | 在线 IDE 能访问代码、数据和持久化工作目录 |
| 3. 执行任务 | 运行实例正常执行并持续输出日志和监控数据 |
| 4. 沉淀资产 | 模型、镜像、数据和结果保存到可复用位置 |

## 开始前准备

- 已有可用镜像、资源规格和租户配额。
- 已准备代码、数据集和持久存储路径。

## 推荐阅读顺序

1. 创建开发环境
2. 挂载代码、数据和存储
3. 提交训练任务
4. 保存模型和日志等产物

## 完成检查

> **用途：** 以下检查是本场景的退出条件，用于判断是否已经取得可观察、可复核的结果，以及是否可以进入下一场景。它不是操作步骤的重复；任一项不满足时，请返回对应功能文档的“常见失败分支”排查。

| 检查项 | 通过标准 |
| --- | --- |
| 1 | 在线 IDE 或训练实例进入运行状态，日志中没有持续启动错误。 |
| 2 | 数据、模型和输出结果已写入持久化存储，并能由后续任务重新读取。 |
| 3 | 作业用量和监控记录可定位，闲置实例已经停止或释放。 |

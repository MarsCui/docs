# 流水明细

::: info 文档信息
版本：v1.0
更新日期：2026-07-10
:::

## 功能概述

`流水明细` 页面标题显示为 `收支明细`，用于查询当前账号的账务收支记录。用户可按交易类型、收支类型和时间范围筛选，查看每一笔流水的发生时间、交易类型、上下文、额度变化、变更后额度、交易单号和备注。

| 项目 | 内容 |
| --- | --- |
| 适用角色 | 用户侧账号、业务管理员、账务查看人员 |
| 导航路径 | 我的账务 > 流水明细 |
| 页面标题 | 收支明细 |
| 管理对象 | 收入流水、支出流水、交易上下文、交易单号 |
| 典型用途 | 追踪余额变化、核对充值到账、排查消费来源 |

### 新手理解

流水明细像账户的小票列表：余额为什么增加、为什么减少，都应能在这里找到对应记录。排查余额问题时，不要只看账户概览，应进入流水明细按时间和类型核对来源。

### 术语速查

| 术语 | 含义 | 处理建议 |
| --- | --- | --- |
| 收支类型 | 表示本次流水是收入还是支出。 | 先看方向，再看交易类型。 |
| 交易类型 | 转账、消费、退款、调整等业务分类。 | 用于缩小排查范围。 |
| 变更额度 | 本次流水导致的 Credits 增减。 | 与变更后额度一起看。 |
| 交易单号 | 关联业务交易的单号。 | 不等同于支付流水号。 |
| 变更后额度 | 本次流水发生后的账户额度。 | 用于解释余额变化。 |

## 前提条件

1. 当前账号具备用户侧账务查看权限。
2. 已进入 `我的账务 > 流水明细`。
3. 已确认需要查询的交易时间范围。
4. 对外沟通时应使用脱敏后的流水号或交易单号。

## 页面说明

页面包含筛选区和流水列表。筛选区支持 `交易类型`、`收支类型` 和时间范围筛选，并提供 `搜索`、`重置` 等按钮。表格展示 `流水号`、`发生时间`、`收支类型`、`交易类型`、`上下文`、`变更额度`、`变更后额度`、`交易单号`、`备注` 和 `操作`。

## 主要操作

### 查询流水

1. 进入 `我的账务 > 流水明细`。
2. 在筛选区选择 `交易类型`。
3. 选择 `收支类型`。
4. 按需要选择时间范围。
5. 点击 `搜索` 查看结果。
6. 如需重新查询，点击 `重置` 清空条件。

### 查看单笔流水

1. 在列表中定位目标流水。
2. 核对流水号、发生时间、收支类型、交易类型和上下文。
3. 查看变更额度、变更后额度和交易单号。
4. 如存在行内详情入口，点击 `Details` 查看更完整信息。

## 参数说明

| 字段名称 | 是否必填 | 字段类型 | 示例 | 说明 |
| --- | --- | --- | --- | --- |
| 交易类型 | 否 | 枚举 | 转账 | 用于按业务交易类型筛选流水。 |
| 收支类型 | 否 | 枚举 | 收入 | 用于筛选收入或支出记录。 |
| 时间范围 | 否 | 日期范围 | 账期内日期 | 用于限定流水发生时间。 |
| 流水号 | 系统生成 | 文本 | `TXN-202607080001` | 单笔流水的唯一标识。 |
| 上下文 | 系统生成 | 文本 | `示例模型调用` | 交易关联的业务上下文。 |
| 变更额度 | 系统生成 | Credits | `-120 Credits` | 本次交易导致的额度变化。 |
| 变更后额度 | 系统生成 | Credits | `9,880 Credits` | 本次交易后的账户额度。 |
| 交易单号 | 系统生成 | 文本 | `BIZ-202607080001` | 与业务交易关联的单号。 |

## 踩坑提示

- 交易单号不等于支付流水号，也不一定等同于充值订单号。
- 单条流水只能解释一次额度变化，月度对账仍需进入月度账单核对汇总。
- 收入、支出、退款、冲正等方向容易混淆，先看交易类型再判断金额变化。
- 查询不到流水时先放宽时间范围，不要直接判断为数据丢失。

## 结果校验

| 检查项 | 成功表现 | 异常时处理 |
| --- | --- | --- |
| 筛选可用 | 选择条件后可点击搜索 | 刷新页面或重置条件 |
| 流水可见 | 列表展示符合条件的记录 | 放宽时间范围或清空筛选 |
| 余额变化可解释 | 变更额度与变更后额度能解释账户变化 | 与账户概览和月度账单交叉核对 |
| 详情可追踪 | 行内详情入口可打开或跳转 | 检查权限或记录脱敏流水号咨询运营方 |

## 常见问题

### 找不到目标流水

**问题现象：**

按交易类型、收支类型或时间范围查询后没有目标记录。

**可能原因：**

时间范围过窄、筛选条件不匹配，或相关交易尚未完成入账。

**处理方式：**

先清空筛选条件，再扩大时间范围；如仍找不到，使用脱敏后的订单号或业务记录联系运营方核对。

### 流水方向与预期不同

**问题现象：**

某条记录显示为收入或支出，但与预期方向不一致。

**可能原因：**

交易类型、退款、冲正、转账或调整会影响收支方向。

**处理方式：**

查看交易类型和上下文；再结合交易单号、月度账单和账户概览判断口径。

### 流水汇总和月度账单不一致

**问题现象：**

流水明细加总结果与月度账单汇总不同。

**可能原因：**

两边账期、时间范围、交易类型或统计口径不同。

**处理方式：**

统一账期和时间范围；刷新月度账单；必要时按交易类型分组核对。

## 注意事项

- 流水号、交易单号、上下文和金额属于敏感账务信息，对外提供前应脱敏。
- 不要仅凭单条流水判断整月费用，应结合月度账单核对。
- 发现异常时，先记录筛选条件和脱敏后的流水标识，再联系运营方处理。

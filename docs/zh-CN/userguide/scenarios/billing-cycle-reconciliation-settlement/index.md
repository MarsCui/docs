# 场景概览 - 运营账期对账与结算

本场景帮助账务运营人员按固定顺序核对账期状态、财务账户、异常巡检和结算单，避免在数据未完整或差异未处理时提前结算。

## 适用角色

- 平台运营方、账务运营人员、财务对账人员

## 场景目标

- 账期、组织和业务线范围保持一致。
- 月结汇总、账户流水和巡检结果能够互相解释。
- 结算单只在阻塞异常处理完成后生成或确认。
- 调账具备明确依据，并能在完成后复核。

## 场景流程

**主线：** 确认账期 → 检查今日任务 → 核对账户 → 处理巡检异常 → 生成结算单 → 必要时调账复核

| 阶段 | 核心结果 |
| --- | --- |
| 1. 确认范围 | 账期、组织、客户和业务线一致 |
| 2. 核对数据 | 月结汇总与财务账户流水能够对应 |
| 3. 处理异常 | 未配对流水、缺失收益和补偿任务有结论 |
| 4. 完成结算 | 结算单状态、金额和入账信息可复核 |

## 开始前准备

- 具备运营财务查看权限。
- 明确目标账期、组织、客户或业务线。
- 准备脱敏后的结算单号、流水号和异常记录。
- 生成结算单、补建、补偿或调账前已确认审批要求。

## 推荐阅读顺序

1. 先查看月结总览和今日任务。
2. 再核对财务账户与巡检中心。
3. 确认无阻塞异常后查看或生成结算单。
4. 仅在差异来源明确时执行调账并回到前述页面复核。

## 完成检查

> **用途：** 以下检查是本场景的退出条件，用于确认账期已经形成可观察、可复核的结论。任一项不满足时，不应宣布账期结算完成。

| 检查项 | 通过标准 |
| --- | --- |
| 1 | 月结总览、财务账户和结算单使用相同账期与组织范围。 |
| 2 | 今日任务中影响结算的阻塞事项已处理或有明确责任人。 |
| 3 | 巡检中心异常已处理、豁免或形成后续计划。 |
| 4 | 结算单金额、状态和入账信息可由流水与汇总解释。 |
| 5 | 调账操作具备依据，完成后相关页面数据已复核。 |

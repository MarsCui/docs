## 功能概述

`业务-资源池授权` 用于为不同业务场景分配云平台下具体地域的使用权限。授权完成后，租户只能访问和使用已授权地域的资源与服务，从而控制业务在云平台地域范围内的可用资源。

| 项目 | 内容 |
| --- | --- |
| 适用角色 | 运营方 |
| 导航路径 | AI基础设施 > On-Cloud > 授权管理 > 业务-资源池授权 |
| 页面路由 | `/infrahub/op/auth/region-auth` |
| 管理对象 | 业务类型、云平台、授权地域和操作入口 |
| 典型途径 | 为指定业务类型配置可使用的云平台地域 |

#### 新手理解

业务-资源池授权像给某类业务划定可部署的地域边界。租户已获得云平台权限后，还需要业务地域授权，部署或调度流程才会在允许的地域范围内选择资源。

#### 术语速查

| 术语 | 说明 |
| --- | --- |
| 业务类型 | 页面中被授权的业务场景，例如推理部署。 |
| 授权地域 | 允许当前业务使用的云平台地域。 |
| 云平台 | 承载授权地域的云平台，例如华为云、阿里云或本地虚拟云。 |
| 当前授权业务 | 授权弹窗中正在配置地域权限的业务类型。 |
| 地域数量 | 列表卡片中展示的已授权地域数量。 |

## 前提条件

1. 目标业务类型已在平台中配置。
2. 需要授权的云平台和地域已接入并可用。
3. 已确认业务可用地域、资源调度范围和变更影响。

## 页面说明

页面用于查看和维护业务类型与云平台地域之间的授权关系。列表按 `业务类型` 展示授权数据，在每个业务类型下显示不同云平台及其已授权地域数量，并提供 `授权地域`、`导出`、`导入` 等入口。

## 主要操作

### 授权地域

1. 进入 `AI基础设施 > On-Cloud > 授权管理 > 业务-资源池授权`。
2. 在业务类型列表中找到目标业务，例如 `推理部署`。
3. 点击目标业务右侧的 `授权地域`。
4. 在弹窗中确认 `当前授权业务`，展开目标云平台。
5. 勾选或取消勾选需要授权的地域。
6. 点击最终 `确定` 前，再次核对业务类型、云平台和地域授权范围。
7. 如仅学习或验证页面，请点击 `取消` 或关闭弹窗，不提交真实授权配置。

| 字段名称 | 是否必填 | 字段类型 | 示例 | 说明 |

| 地域数量 | 否 | 数字 | `2` | 列表卡片中显示的已授权地域数量。 |
| 授权地域入口 | 否 | 按钮 | `授权地域` | 打开地域授权弹窗。 |
| 导出 | 否 | 按钮 | `导出` | 导出授权数据，可能包含敏感运营信息。 |
| 导入 | 否 | 按钮 | `导入` | 批量导入授权数据，可能改变多条授权配置。 |

## 踩坑提示

- 业务地域授权会影响部署或调度可用范围，不能只按地域名称相似度判断。
- `导入` 可能批量改变授权关系，学习或验证页面时不要执行真实导入。
- 截图未展示资源池、可用区或启用状态字段，本文不将这些内容写成已确认 UI 字段。

## 常见问题

#### 业务地域下没有可用规格

**问题现象：**

用户选择业务地域后，规格或可用资源为空。

**可能原因：**

- 当前业务未授权对应云平台地域。
- 云平台地域未接入、未同步或状态异常。
- 上层租户-云平台授权不完整。

**处理方式：**

1. 检查业务类型下的授权地域。
2. 确认云平台和地域接入状态。
3. 回到租户-云平台授权页面核对上层授权。

#### 部署被调度到非预期地域

**问题现象：**

用户选择某业务场景后，部署落到了非预期云平台地域。

**可能原因：**

- 当前业务授权了多个地域。
- 下游调度策略配置了回退地域或资源。
- 首选地域资源不可用或容量不足。

**处理方式：**

1. 检查目标业务的授权地域范围。
2. 查看调度策略和资源可用状态。
3. 核对部署事件中的调度原因。

## 后续操作

1. 使用业务视角验证部署或资源选择流程。
2. 结合调度策略复核地域选择顺序和回退范围。
3. 定期检查授权地域数量，避免业务可用范围过大或过窄。

## 注意事项

- 授权地域可能改变业务可用地域、资源池调度范围和部署可用性。
- 授权变更可能影响真实部署、资源调度、容量使用、费用归属和业务连续性。
- `确定`、`保存`、`提交` 属于高风险最终动作，文档只描述字段查看和提交前核对，不引导测试学习时提交。
- `导出` 可能包含敏感运营数据，`导入` 可能批量修改授权关系，操作前需确认影响范围。
- 不写入真实业务名称、租户信息、账号、密钥、Token、AK/SK、接口地址、云资源 ID 或内部测试参数。

# 用户手册

## 子系统简介

三个子系统像一条模型服务交付流水线：AI Infra On-Prem 管本地算力和资源池，AI Infra On-Cloud 管云上账号、授权和部署资源，Model Services 管模型上架、体验、调用和收益分析。

| 子系统 | 适用场景 | 主要对象 |
| --- | --- | --- |
| AI Infra On-Prem | 管理私有化或本地集群资源 | 地域、可用区、集群、节点、存储、模板、配额、监控 |
| AI Infra On-Cloud | 管理云账号接入和云上模型服务部署 | 云平台、云账号、资源池、授权、部署资产、调度策略 |
| Model Services | 管理模型发布、体验、调用和收益 | 模型市场、我的模型、Playground、调用日志、用量收益 |

## 按角色阅读

| 角色 | 推荐入口 | 重点任务 |
| --- | --- | --- |
| 运营方 | On-Prem 运营方、On-Cloud 运营方、Model Services 运营方 | 准备资源、配置模板、审核模型、维护配额和监控 |
| 普通用户 | On-Prem 用户、On-Cloud 用户、Model Services 用户 | 创建实例或作业、部署模型服务、查看额度用量和调用结果 |
| 模型提供方 | Model Services 的 Studio 和收益相关页面 | 发布模型、维护版本、验证体验效果、查看调用和收益 |

## 我该从哪里开始

| 你的目标 | 先看哪个入口 | 下一步 |
| --- | --- | --- |
| 管本地集群和资源池 | AI Infra On-Prem | 地域 / 集群 / 规格 / 模板 |
| 管云账号和云上部署 | AI Infra On-Cloud | 云账号 / 资源池 / 授权 / 快速部署 |
| 发布或调用模型 | Model Services | 模型市场 / 我的模型 / Playground |

## 推荐阅读路径

1. 新手先读对应子系统的 `getting-started`，理解角色边界、核心术语和资源层级。
2. 需要完整交付流程时，再读 `end-to-end`，按步骤串联运营方准备和用户侧使用。
3. 已经知道目标功能时，直接进入功能页查看前提条件、参数说明、结果校验和 FAQ。
4. 遇到创建失败、调用失败或用量异常时，优先查看配额、监控、调用日志和计量明细。

## 前提条件

1. 已确认当前账号角色和所属租户。
2. 已确认目标子系统、地域、资源池、模型或云账号对当前账号可见。
3. 涉及部署、调用、充值或授权操作前，应先确认费用、额度、权限和安全边界。
4. 对外沟通问题时，准备脱敏后的页面路径、时间范围、资源编号、错误信息和操作步骤。

## 常见问题

### 不确定应该读哪个子系统

**问题现象：**

想完成模型部署或调用，但不确定从 On-Prem、On-Cloud 还是 Model Services 开始。

**可能原因：**

- 不清楚目标资源在本地集群还是云上资源池。
- 不清楚自己是运营方、普通用户还是模型提供方。
- 任务同时涉及资源准备、模型发布和调用验证。

**处理方式：**

1. 本地集群资源管理先看 AI Infra On-Prem。
2. 云账号和云上部署先看 AI Infra On-Cloud。
3. 模型市场、模型发布、Playground 和调用收益先看 Model Services。

### 页面入口不可见

**问题现象：**

文档中提到的菜单、按钮或页面在当前账号下不可见。

**可能原因：**

- 当前账号角色或租户权限不包含该功能。
- 所选地域、资源池、云账号或模型未对当前账号开放。
- 功能依赖的前置资源尚未由运营方配置完成。

**处理方式：**

1. 先确认账号角色、租户和当前子系统。
2. 对照功能页的前提条件检查资源、授权和可见范围。
3. 联系运营方时提供脱敏账号、页面路径、时间范围和缺失入口说明。

### 操作失败但文档步骤已完成

**问题现象：**

按文档完成操作后，实例、部署、作业或调用仍然失败。

**可能原因：**

- 配额、容量、规格、镜像、存储或模型来源不满足运行条件。
- Endpoint、API Key、请求头或云账号授权已失效。
- 监控、计量或状态同步存在延迟。

**处理方式：**

1. 查看当前功能页的结果校验和踩坑提示。
2. 结合监控、事件、日志、调用记录和用量明细交叉定位。
3. 提交工单时只提供脱敏信息，不附带真实密钥、Token 或内部 Endpoint。

## 使用建议

- 先确认角色，再确认子系统，最后进入具体功能页。
- 涉及费用、凭据、部署和生产资源的操作，应先在受控范围内验证。
- 文档中的示例名称、路径和地址只用于说明，实际操作以当前环境的可见资源为准。

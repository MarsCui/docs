# 场景概览 - 发布模型（预设置）

指导运营管理员在 Provider 发布公有模型前，完成平台侧的基础数据与审批配置。

## 适用角色

- 运营管理员

## 场景目标

- 准备模型作者、元模型、模型来源和模型模板。
- 准备标签、币种和模型审核能力。
- 让模型提供方能够基于统一配置发布模型。

## 场景流程

**主线：** 定义标准模型 → 接入模型来源 → 组合发布模板 → 验证提供方可用性

| 阶段 | 核心结果 |
| --- | --- |
| 1. 定义标准模型 | 作者、元模型、模态、能力和协议形成统一口径 |
| 2. 接入模型来源 | 厂商、地域、接口和认证模板可复用 |
| 3. 组合发布模板 | 来源、元模型、标签、币种和默认参数关联完整 |
| 4. 验证可用性 | 模型提供方发布时能看到正确选项，审核入口可用 |

## 开始前准备

- 已明确模型类型、输入输出模态和协议。
- 已准备上游模型 Endpoint、地域和请求头规则。
- 已明确计费币种和审核责任人。

## 推荐阅读顺序

1. 元模型
2. 模型来源
3. 模型模板
4. 标签、币种和模型审核

## 完成检查

> **用途：** 以下检查是本场景的退出条件，用于判断是否已经取得可观察、可复核的结果，以及是否可以进入下一场景。它不是操作步骤的重复；任一项不满足时，请返回对应功能文档的“常见失败分支”排查。

| 检查项 | 通过标准 |
| --- | --- |
| 1 | Provider 能选择目标作者、元模型、模型来源和模板。 |
| 2 | 模态、协议、Token 限制和高级能力配置一致。 |
| 3 | 标签和币种满足发布与计费需要。 |
| 4 | 模型审核入口和审批角色可用。 |

## 常见失败分支

| 现象 | 优先检查 |
| --- | --- |
| Provider 看不到模板 | 模板状态、发布范围、元模型和模型来源关联 |
| 模型来源测试失败 | 地域节点、请求头、协议地址和连通性 |
| 计费配置无法完成 | 币种、计费单位、模型类型和价格规则 |

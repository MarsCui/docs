# 从零开始部署云上模型服务

:::: info 文档信息
版本：v1.0
更新日期：2026-07-08
::::

## 功能概述

本文把运营管理员和普通用户的 On-Cloud 操作串成一条端到端路径：运营管理员先接入云平台、云账号、资源池、授权关系、模型资产和调度策略，普通用户再选择云账号与模型，通过智能推荐发起部署，并在部署后查看 API 调用、监控和事件记录。

| 项目 | 内容 |
| --- | --- |
| 适用角色 | 运营管理员、普通用户 |
| 推荐前置阅读 | [平台入门](../../getting-started/)、[快速接入](../../operator/access-workbench/quick-start/)、[快速部署](../../user/model-services/quick-deployment/) |
| 输出结果 | 一个可查看状态、API、监控和事件的云上模型服务部署任务 |
| 典型用途 | 新云平台首次联调、模型资产上架验证、部署流程演示、上线前验收和问题定位 |

## 新手理解

从零部署云上模型服务像安排一次云资源接力：运营管理员准备云账号、资源池、授权、模型和调度策略，普通用户选择资源并发起部署。

## 端到端流程图

| 阶段 | 操作方 | 目标 |
| --- | --- | --- |
| 云平台接入 | 运营管理员 | 创建或启用可接入云平台。 |
| 云账号接入 | 运营管理员 | 添加云账号，确保平台可同步云资源。 |
| 资源池准备 | 运营管理员 | 启用地域资源池并确认可用范围。 |
| 授权开放 | 运营管理员 | 配置租户-云平台授权和业务-资源池授权。 |
| 部署资产准备 | 运营管理员 | 维护模型库、推理框架、推理镜像和部署配置。 |
| 策略治理 | 运营管理员 | 配置影响推荐排序的业务策略。 |
| 快速部署 | 普通用户 | 选择云账号、地域、模型、推理引擎和业务策略，发起部署。 |
| 状态验证 | 普通用户 | 查看部署状态、API 调用、监控信息和事件记录。 |
| 排障闭环 | 双方 | 根据失败路径检查账号、授权、资源池、模型资产和策略。 |

## 术语速查

| 术语 | 说明 |
| --- | --- |
| 云账号校验 | 验证云账号密钥、权限和资源同步是否可用。 |
| 资源池启用 | 把云账号内资源纳入可授权和可调度范围。 |
| 业务地域授权 | 控制具体业务能部署到哪些地域。 |
| 我的部署 | 普通用户查看部署状态、Endpoint、事件和监控的入口。 |

## 前提条件

1. 运营管理员具备云平台接入、云账号、资源池、授权、部署资产和策略管理权限。
2. 普通用户具备接入账号、快速部署和我的部署的菜单权限。
3. 目标云平台账号具备创建或管理部署所需的云资源权限。
4. 租户和业务已获得目标云平台与资源池授权。
5. 目标模型、推理框架、推理镜像和部署配置已准备。
6. 本次验证使用的账号和凭据已脱敏记录，不在文档中保存真实 AK/SK。

## 步骤 1：运营管理员接入云平台

1. 进入 `接入管理 > 接入云平台`。
2. 点击 `添加平台`，填写云平台名称、平台类型、标识和描述。
3. 提交后确认云平台出现在列表中，并处于可使用状态。
4. 如需按向导操作，可先进入 [快速接入](../../operator/access-workbench/quick-start/) 查看整体接入进度。

结果校验：

1. 云平台列表中能看到目标云平台。
2. 云平台状态、名称和标识符合预期。
3. 后续云账号和资源池配置中可选择该云平台。

## 步骤 2：运营管理员接入云账号

1. 进入 `接入管理 > 接入账号`。
2. 选择目标云平台，点击 `添加云账号`。
3. 填写账号名称、账号标识、AK/SK 或平台要求的认证信息。
4. 提交后确认账号出现在列表中，并能用于资源同步或部署。

结果校验：

1. 目标云账号在 [接入账号](../../operator/access-management/cloud-accounts/) 列表中可见。
2. 账号状态正常，且归属云平台正确。
3. 账号凭据未暴露在文档、截图或工单中。

## 步骤 3：运营管理员启用资源池

1. 进入 `接入管理 > 接入资源池`。
2. 选择目标云平台，查看地域卡片网格。
3. 启用目标地域资源池，必要时编辑地域显示名称。
4. 确认目标资源池可用于后续授权和部署。

结果校验：

1. [接入资源池](../../operator/access-management/resource-pools/) 中目标地域处于启用状态。
2. 地域名称、云平台归属和资源池信息准确。
3. 普通用户部署时的目标地域与资源池配置一致。

## 步骤 4：运营管理员配置授权

1. 进入 `授权管理 > 租户-云平台授权`，为目标租户新增云平台授权。
2. 进入 `授权管理 > 业务-资源池授权`，为目标业务或用户范围授权目标资源池地域。
3. 确认授权范围覆盖本次验证使用的租户、业务、云平台和资源池。

结果校验：

1. [租户-云平台授权](../../operator/auth-management/tenant-cloud-auth/) 中存在目标授权记录。
2. [业务-资源池授权](../../operator/auth-management/business-region-auth/) 中存在目标资源池授权记录。
3. 普通用户登录后能看到对应云平台和地域范围。

## 步骤 5：运营管理员准备部署资产

1. 进入 `部署资产 > 模型库`，添加或确认目标模型。
2. 在模型库中配置元模型、上架版本、云上部署点、输出配置和计算方案。
3. 进入 `部署资产 > 推理框架`，确认目标推理框架可用。
4. 进入 `部署资产 > 推理镜像`，确认目标运行镜像可用。

结果校验：

1. [模型库](../../operator/deploy-assets/models/) 中目标模型可见。
2. 模型关联的云平台、地域、推理框架、镜像和计算方案完整。
3. 普通用户快速部署页能筛选到目标模型。

## 步骤 6：运营管理员配置调度策略

1. 进入 `调度治理 > 策略管理`。
2. 新增或确认用于部署推荐的业务策略。
3. 检查策略名称、适用范围、排序口径和状态。

结果校验：

1. [策略管理](../../operator/scheduling-governance/policies/) 中目标策略可见。
2. 快速部署中的业务策略选项与策略配置一致。
3. 策略不会把可用资源全部排除。

## 步骤 7：普通用户添加或确认接入账号

1. 进入 `访问管理 > 接入管理`。
2. 点击 `添加云账号` 或确认已有云账号可用。
3. 填写账号名称、云平台、认证信息和必要说明。
4. 返回列表确认账号可见。

结果校验：

1. [接入账号](../../user/access-management/access-accounts/) 中目标账号可见。
2. 账号云平台与运营管理员授权的云平台一致。
3. 账号凭据已脱敏保存。

## 步骤 8：普通用户发起快速部署

1. 进入 `模型服务 > 快速部署`。
2. 在模型库页选择目标云平台、地域和模型。
3. 点击目标模型的 `部署模型`。
4. 在智能推荐部署页选择部署模式、推理引擎和业务策略。
5. 确认模型算力匹配方案后点击 `确认部署`。
6. 在发布部署弹窗中选择云账号，填写任务名称和任务描述。
7. 点击 `发起部署` 提交部署任务。

结果校验：

1. [快速部署](../../user/model-services/quick-deployment/) 能看到目标云账号、地域和模型。
2. 智能推荐页能生成至少一个可部署方案。
3. 发布部署弹窗中必填项填写后可提交。

## 步骤 9：普通用户验证部署状态

1. 进入 `模型服务 > 我的部署`。
2. 在部署任务列表中查找刚提交的任务。
3. 打开详情页，查看基本信息、API 调用、监控信息和事件记录。
4. 如任务支持发布部署，按页面按钮完成后续发布操作。

结果校验：

1. [我的部署](../../user/model-services/my-deployments/) 中能看到目标任务。
2. 任务状态符合预期，失败时能看到事件记录。
3. API 调用信息、监控信息和基本信息可查看。

## 常见失败路径排查

### 失败分支：云账号校验失败

下一跳：[云账号](../../operator/access-management/cloud-accounts/)

**问题现象：**运营管理员添加云账号后校验不通过或无法同步资源。

**排查路径：**

1. 检查 AK/SK、授权策略、账号状态和云厂商 API 权限。
2. 确认密钥未过期，且最小权限覆盖资源查询和部署所需操作。
3. 校验通过后再启用资源池和配置租户授权。

### 失败分支：资源池不可用

下一跳：[资源池](../../operator/access-management/resource-pools/)

**问题现象：**用户已获授权，但快速部署页没有可用地域、规格或资源池。

**排查路径：**

1. 检查资源池是否启用、容量是否同步、地域是否被授权。
2. 核对租户授权和业务地域授权是否同时覆盖当前用户。
3. 确认模型部署资产和调度策略没有限制到其他资源池。

### 失败分支：部署创建失败

下一跳：[我的部署](../../user/model-services/my-deployments/)

**问题现象：**快速部署提交后失败，或我的部署中长时间创建中。

**排查路径：**

1. 查看我的部署事件，区分资源不足、镜像失败、框架异常或认证错误。
2. 核对模型、推理框架、运行镜像、规格和调度策略是否匹配。
3. 将脱敏部署 ID、地域、模型名和错误摘要交给运营管理员复核。

## 完成后检查

| 检查项 | 成功表现 | 下一步 |
| --- | --- | --- |
| 资源可见 | 快速部署页能选择目标模型、地域、账号和规格 | 继续提交云上部署 |
| 状态正常 | 我的部署显示运行中或可用，并生成调用信息 | 进入 API 访问或监控 |
| 排障入口可用 | 事件、状态和监控能定位部署错误 | 按失败分支继续排查 |

## 后续操作

1. 普通用户进入我的部署，持续查看服务状态、Endpoint、事件和监控。
2. 使用脱敏调用示例验证模型输出、延迟和错误码。
3. 运营管理员定期复核云账号授权、资源池容量、调度策略和部署资产版本。
4. 发现费用或容量异常时，结合 On-Cloud 用量、Model Services 调用记录和云厂商账单交叉核对。

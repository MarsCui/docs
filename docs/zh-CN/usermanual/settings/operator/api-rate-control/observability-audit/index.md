## 功能概述

`观测审计` 用于查看 API 流控的分钟统计、拦截日志和审计日志，并提供清理和导出入口。

| 项目 | 内容 |
| --- | --- |
| 适用角色 | 运营方管理员 |
| 导航路径 | API 流控 > 观测审计 |
| 页面路由 | /operator/api-rate-control/observability-audit |
| 管理对象 | 分钟统计、拦截日志和审计日志 |
| 典型用途 | 查看流控统计、拦截记录、审计明细以及清理和导出入口 |

### 新手理解

观测审计页像 API 流控的明细放大镜，用来查看哪些请求被统计或拦截，便于从规则、节点、API 和时间范围定位问题。

### 术语速查

| 术语 | 含义 | 处理建议 |
| --- | --- | --- |
| 审计记录 | 流控统计或拦截产生的明细。 | 排障时按时间过滤。 |
| 节点 | 执行流控规则的服务节点。 | 节点差异时查缓存。 |
| 规则命中 | 请求匹配并触发规则。 | 结合规则管理核对。 |
| 清理数据 | 删除或归档审计数据的动作。 | 执行前确认范围。 |


## 前提条件

1. 当前账号具备 API 流控观测审计权限。
2. 已进入 `API 流控 > 观测审计`。
3. 清理或导出数据前已确认范围和用途。

## 主要操作

### 查看观测审计

1. 进入 `API 流控 > 观测审计`。
2. 选择 `分钟统计`、`拦截日志` 或 `审计日志` 页签。
3. 输入 API Path、节点 ID 或时间范围。
4. 点击 `搜索` 查询目标记录。
5. 如需导出，先确认数据范围，再选择对应导出入口。
6. 对 `清理` 操作，应先确认保留策略和影响范围。

## 踩坑提示

- Overview 有拦截但审计查不到时，先确认时间范围和节点范围。
- 审计明细可能包含 API、IP 或账号线索，导出前要脱敏。
- 清理审计数据前，确认是否仍需用于问题追踪或合规留存。

## 常见问题

### Overview 有拦截但审计查不到

**问题现象：**

Overview 显示拦截升高，观测审计列表没有对应记录。

**可能原因：**

时间范围不一致，或当前页签不是拦截日志。

**处理方式：**

切换到拦截日志，扩大时间范围后重新查询。

### 清理数据前要注意什么

**问题现象：**

页面提供 `清理` 入口。

**可能原因：**

清理会影响后续审计追溯。

**处理方式：**

先确认数据保留要求和排障需求，再由具备权限的人员处理。

### 审计观测记录为什么为空？

**问题现象：**

观测审计页没有 API 命中、限流、拦截或异常记录。

**可能原因：**

时间范围内没有触发频控规则，请求未经过频控网关，或审计采集链路延迟。

**处理方式：**

扩大时间范围并确认请求路径；检查规则发布和网关接入；仍为空时到节点缓存页核对上报状态。

## 功能概述

`调度策略` 用于维护调度规则、资源优先级、地域偏好、限额和回退策略，支撑多云调度、资源授权和模型部署流程。

| 项目 | 内容 |
| --- | --- |
| 适用角色 | 运营方 |
| 导航路径 | 调度治理 > 调度策略 |
| 页面路由 | /operator/scheduling-governance/policies |
| 管理对象 | 调度规则、资源优先级、地域偏好、限额和回退策略 |
| 典型用途 | 治理跨云模型部署的资源选择和调度规则 |

### 新手理解

调度策略像多云部署的路线规则，决定首选哪个资源池、容量不足时怎么回退，以及不同租户或业务地域能不能跨云调度。

### 术语速查

| 术语 | 说明 |
| --- | --- |
| 调度策略 | 决定部署请求如何选择云资源的规则。 |
| 优先级 | 多个资源可用时的排序依据。 |
| 回退策略 | 首选资源不可用时的替代选择。 |
| 限额 | 控制租户或业务使用资源的上限。 |

## 前提条件

1. 资源池、租户授权和业务地域授权已完成。
2. 首选资源池、回退资源池和容量阈值已确认。
3. 跨地域调度策略已评估成本和合规影响。
## 页面说明

页面用于维护多云部署调度规则，包括资源优先级、地域偏好、容量阈值、回退策略和适用范围。运营方应结合资源池水位、业务地域授权和 SLA 要求配置策略。

## 主要操作

### 操作步骤

1. 进入 `调度治理 > 调度策略`。
2. 按租户、业务地域、资源池或状态筛选策略。
3. 新增策略时设置适用范围、首选资源池和优先级。
4. 配置容量阈值、失败回退和限制条件。
5. 保存后通过测试部署查看实际调度结果。

### 参数说明

| 字段名称 | 是否必填 | 字段类型 | 示例 | 说明 |
| --- | --- | --- | --- | --- |
| 策略名称 | 是 | 文本 | `prod-gpu-fallback` | 调度策略展示名称。 |
| 适用范围 | 是 | 多选 | `tenant-a / 华东生产` | 策略作用的租户或业务地域。 |
| 首选资源池 | 是 | 下拉选择 | `gpu-cn-shanghai-prod` | 优先调度的资源池。 |
| 回退资源池 | 否 | 多选 | `gpu-cn-hangzhou-backup` | 首选资源不可用时的替代资源。 |
| 容量阈值 | 否 | 数字 | `80%` | 超过阈值后触发降级或回退。 |

### 踩坑提示

- 回退资源池必须已完成租户和业务地域授权。
- 调度优先级变更只影响后续部署，已运行实例需按平台能力单独处理。
- 跨地域回退可能影响延迟、成本和数据合规边界。

### 结果校验

1. 策略记录启用并绑定正确适用范围。
2. 测试部署事件显示命中预期资源池。
3. 首选资源不可用时能按回退策略处理或给出明确失败原因。

## 常见问题

### 部署没有命中首选资源池

**问题现象：**

调度结果落到回退资源池或其他云地域。

**可能原因：**

- 首选资源池容量不足。
- 业务地域授权不包含首选资源池。
- 策略优先级低于其他规则。

**处理方式：**

1. 检查资源池容量和启用状态。
2. 核对租户及业务地域授权。
3. 查看调度策略优先级和部署事件。

### 回退策略没有生效

**问题现象：**

首选资源池不可用时部署直接失败。

**可能原因：**

- 未配置回退资源池。
- 回退资源池未授权或未启用。
- 限制条件禁止跨地域或跨云回退。

**处理方式：**

1. 补充回退资源池配置。
2. 确认回退资源池授权和容量。
3. 检查策略限制条件。

## 后续操作

1. 监控资源池水位。
2. 复核业务地域授权。
3. 定期用测试部署验证调度策略。

## 注意事项

- 回退资源池必须已授权。
- 优先级变更影响后续部署调度。
- 跨地域回退可能增加延迟和成本。

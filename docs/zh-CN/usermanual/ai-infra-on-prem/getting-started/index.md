## 功能概述

`异构卡纳管` 汇总 AGIOne 面向本地化异构算力资源池的接入、调度、计量和使用能力。它把地域、可用区、集群、节点、规格、镜像、存储、模板、配额和监控串成一条完整链路：运营方先接入和治理资源，普通用户再基于已开放的能力创建模型服务、在线 IDE、运行实例和存储资源。

本文是入门理解页，不是具体操作页。它负责解释 On-Prem 资源层级、角色边界和阅读路径；真实创建、删除、启停、禁用、解绑、额度调整或资源变更应进入对应功能页并按页面风险提示执行。

## AGIOne / 异构卡纳管是什么

AGIOne 面向模型服务、算力资源和运营治理提供一体化平台能力。异构卡纳管指本地化部署或本地资源池纳管场景，平台不会直接替代底层 Kubernetes、Harbor、MinIO、Ceph、NFS 等组件，而会把这些能力纳入统一的资源池、模板、配额和监控视图。

在异构卡纳管场景中，平台负责：

1. 记录资源边界，例如地域、可用区和集群。
2. 维护可调度能力，例如节点、加速卡、资源规格和存储挂载。
3. 管理作业入口，例如部署模板、模型实例、在线 IDE 和运行实例。
4. 计量用户使用量，并提供资源监控和排障入口。

## 资源层级关系

| 层级 | 说明 | 影响范围 |
| --- | --- | --- |
| 地域（Region） | 最大资源边界，通常对应城市、机房、资源池或租户边界。 | 影响用户可见资源、组件绑定和调度范围。 |
| 可用区（Availability Zone） | 地域下的资源分组，通常对应机房区域、网络域或集群组。 | 影响集群归属和故障隔离。 |
| 集群（Cluster） | 被平台纳管的 Kubernetes 集群。 | 决定作业是否有可调度节点。 |
| 节点（Node） | 集群内实际提供 CPU、内存、磁盘和加速卡的服务器。 | 决定运行容量和硬件能力。 |
| 资源池 | 由地域、可用区、集群、规格、镜像和存储组成的可调度能力集合。 | 决定用户能创建哪些实例以及实例能否正常运行。 |
| 作业 / 实例 | 用户创建的模型服务、在线 IDE、运行实例或训练任务。 | 消耗配额和额度，并产生监控与计量数据。 |

推荐配置顺序是：先创建地域，再创建可用区，然后注册集群，之后关联规格、镜像和存储，最后开放模板与配额给用户。

## 全局核心术语

| 术语 | 说明 |
| --- | --- |
| Kubernetes | 容器编排系统，异构卡纳管集群接入和作业调度的基础。 |
| kubeconfig | Kubernetes 连接配置文件，包含 API Server、证书和认证信息，属于敏感材料。 |
| Harbor / Registry | 容器镜像仓库，用于存储和分发作业镜像。 |
| MinIO / S3 | 常见对象存储实现或协议，用于桶和对象读写。 |
| Ceph / RBD | 常见块存储方案，RBD 提供块设备能力。 |
| NFS | 网络文件系统，常用于共享目录挂载。 |
| 资源规格 | CPU、内存、加速卡、显存等资源套餐。 |
| 显存测算 | 根据模型、精度、KV Token、并发等因素估算显存占用。 |
| KV Token | 推理时 Key/Value Cache 相关 Token 数，会影响显存占用。 |
| 动态表达式 | 用于根据参数动态计算推荐规格、显存或表单显示条件。 |
| 配额 | 租户或用户可申请资源上限。 |
| 额度 | 资源使用对应的计费或消耗控制值。 |

## 常见问题

#### 用户侧看不到某个资源

**问题现象：**

用户创建实例、选择镜像、选择规格或查看监控时，目标资源没有出现在列表中。

**可能原因：**

- 运营方尚未在目标地域接入或启用对应资源。
- 资源已接入，但未绑定到用户选择的地域、可用区、集群或租户。
- 当前账号缺少菜单、资源或租户授权。
- 用户侧筛选条件限制了列表范围。

**处理方式：**

1. 确认用户选择的地域、可用区和筛选条件。
2. 请运营方检查地域、集群、规格、镜像、存储和模板的绑定关系。
3. 检查租户配额、额度和账号权限。
4. 资源刚调整后刷新页面或重新进入创建流程。

#### 实例创建后长时间排队

**问题现象：**

模型实例、在线 IDE 或运行实例提交后没有进入运行状态。

**可能原因：**

- 目标规格资源不足。
- 租户配额或额度不足。
- 镜像拉取、存储挂载或集群调度存在异常。
- 目标集群处于维护、不可用或监控数据延迟状态。

**处理方式：**

1. 先查看实例状态、事件和日志。
2. 检查资源配额和用量。
3. 尝试选择更小规格或其他可用地域。
4. 联系运营方检查集群节点、规格关联、镜像服务和存储挂载。

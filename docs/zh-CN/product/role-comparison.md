## 角色速览

| 角色 | 定位 | 负责内容 | 不负责内容 |
| --- | --- | --- | --- |
| `admin` | 平台管理员 | 租户、用户、角色、菜单和平台访问基础 | 日常算力运营、模型发布或模型使用 |
| `operator` | 平台运营方 | 资源准备、治理配置、配额、监控、审核、客户财务、License、设置、审计和 API 流控 | 代替模型提供方发布模型，或作为普通用户消费模型 |
| `provider` | 模型提供方 | 发布和维护模型、聚合模型、审核提交、客户调用、提供方收益和授权范围内的团队设置 | 平台级身份治理、运营财务，或审核自己发布的内容 |
| `enduser` | 普通用户与模型使用方 | 部署可用服务、发现和体验模型、API 调用、个人用量、个人账务和个人设置 | 发布或审核模型，管理平台级资源 |

## 能力对比

| 能力 | `admin` | `operator` | `provider` | `enduser` |
| --- | --- | --- | --- | --- |
| 管理租户、用户和角色分配 | 主要负责 | 否 | 否 | 否 |
| 准备 On-Prem 资源池和模板 | 否 | 主要负责 | 使用已授权资源 | 使用已授权资源 |
| 接入和授权受支持的云资源 | 否 | 主要负责 | 使用已分配访问账户或资源 | 使用已分配访问账户或资源 |
| 维护 Model Services 基础设置 | 否 | 主要负责 | 使用已准备配置 | 否 |
| 维护财务和 License 运营 | 否 | 负责平台和客户财务 | 查看自有收益与结算 | 查看个人账务 |
| 维护设置、审计和 API 流控 | 平台访问基础 | 主要负责平台设置 | 获得授权时管理团队或个人范围 | 获得授权时管理个人或团队范围 |
| 发布单模型或 BYOK 模型 | 否 | 治理与审核 | 主要负责 | 否 |
| 创建聚合模型 | 否 | 治理与审核 | 主要负责 | 否 |
| 审核模型和应用 | 否 | 主要负责 | 提交审核 | 否 |
| 发现模型并使用 Playground | 否 | 按需验收 | 按需验收 | 主要负责 |
| 调用模型 API | 否 | 按需验收 | 验证自有服务 | 主要负责 |
| 查看客户调用和模型收益 | 否 | 获得授权时查看平台范围 | 查看自有模型 | 否 |
| 查看个人调用、用量和账务 | 否 | 获得授权时查看运营范围 | 查看自身范围 | 查看自身范围 |

“主要负责”表示正常任务归属，实际可见范围仍取决于租户、角色配置、资源授权和安装版本。

## `admin`：平台管理员

**典型人员：** 部署方负责人、平台管理员或获得授权的 IT 管理员。

主要职责：

- 维护租户、用户、角色和平台访问配置。
- 分配角色，将平台级权限限制在获得授权的管理团队内。
- 在业务职责已确认后，协助排查菜单缺失和权限边界。

职责边界：

- `admin` 不是资源运营、模型发布、审核或模型使用的默认角色。
- 不应为了绕过业务权限缺失而直接分配宽泛的管理员权限。

## `operator`：平台运营方

**典型人员：** 平台运营、基础设施、模型治理或交付团队。

各子系统主要职责：

| 子系统 | 主要职责 | 手册入口 |
| --- | --- | --- |
| AI Infra On-Prem | 准备资源拓扑、规格、存储、镜像、模板、配额、计量和监控 | [On-Prem 平台入门](../usermanual/ai-infra-on-prem/getting-started/) |
| AI Infra On-Cloud | 维护受支持的云平台接入、账号、资源池、授权、部署资产和调度策略 | [On-Cloud 平台入门](../usermanual/ai-infra-on-cloud/getting-started/) |
| Model Services | 维护元模型、模型来源、模板、标签和币种设置，处理模型与应用审核 | [Model Services 平台入门](../usermanual/model-services/getting-started/) |
| 财务 | 维护客户财务、运营财务、巡检对账、结算、调账和 License 状态 | [财务快速入门](../usermanual/billing/getting-started/) |
| 设置 | 维护成员、角色、组织、操作日志、平台设置、登录配置和 API 流控规则 | [设置快速入门](../usermanual/settings/getting-started/) |

职责边界：

- 运营方负责准备供给和治理条件，但不代替模型提供方发布其自有模型。
- 华为云接入暂不支持，不能作为运营方接入场景。

## `provider`：模型提供方

**典型人员：** 模型团队、AI 开发者、模型服务提供方或负责发布的技术团队。

主要职责：

- 发布和维护单模型或 BYOK Endpoint。
- 使用符合条件的成员模型创建聚合模型，并选择可用路由策略。
- 配置提供方自有发布信息，提交审核并处理审核结果。
- 在授权范围内查看模型用量、模型收益和客户调用数据。
- 在授权范围内查看提供方收益、客户列表和结算记录。
- 仅在账号获得授权时管理个人 Key、账号信息、团队成员、角色、额度或组织设置。
- 部署流程需要时，使用已授权的 On-Prem 或 On-Cloud 资源。

职责边界：

- 模型提供方不能审核自己发布的模型或应用。
- 模型提供方不管理平台级租户、菜单、资源策略或其他提供方的数据。

操作流程见[从发布到调用模型](../usermanual/model-services/end-to-end/publish-and-call-model/)。

## `enduser`：普通用户与模型使用方

**典型人员：** 业务用户、应用开发者、API 使用方，或使用已准备资源进行部署的人员。

主要职责：

- 浏览可见模型并查看模型详情。
- 在 Playground 体验受支持的模型交互。
- 获取所需权限并调用模型 API。
- 查看个人调用总览、分析、日志、用量和部署状态。
- 在入口可见时查看个人账务、交易流水、充值订单、月度账单和个人设置。
- 使用已授权的资源与模板创建 On-Prem 或 On-Cloud 工作负载。

职责边界：

- 普通用户不发布单模型或聚合模型，聚合模型创建属于模型提供方职责。
- 普通用户不处理审核，也不维护平台级资源与身份配置。

操作入口见[用户手册](../usermanual/)和[场景操作指南](../userguide/scenarios)。

## 任务应该由谁完成

| 任务 | 推荐角色 |
| --- | --- |
| 创建租户或分配角色 | `admin` |
| 纳管本地集群或配置配额 | `operator` |
| 接入和授权受支持的云资源池 | `operator` |
| 维护元模型或审核模型 | `operator` |
| 完成账期对账或维护客户财务 | `operator` |
| 维护成员、角色、登录配置或 API 流控规则 | `operator` 或获得授权的管理员 |
| 发布模型或创建聚合模型 | `provider` |
| 查看客户调用和提供方收益 | `provider` |
| 体验模型或调用 API | `enduser` |
| 查看个人用量、个人账务或个人部署 | `enduser` |

小型部署中同一人员可能承担多个职责，但仍应先确认任务所需边界再分配多个角色。平台管理和审核权限应保持最小范围，并保留审计依据。

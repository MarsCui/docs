# 场景概览 - 应用发布与审核

本场景帮助运营方核对应用绑定模型、调用入口和客户可见范围，完成应用发布、审核及客户侧验证。

## 适用角色

- 平台运营方、应用发布管理人员、应用审核人员

## 场景目标

- 应用绑定模型处于可用状态。
- 调用入口、参数映射和客户可见范围清晰。
- 审核意见和发布状态能够被追溯。
- 审核通过后，仅目标客户能够看到并调用应用。

## 场景流程

**主线：** 准备应用材料 → 核对发布信息 → 处理应用审核 → 验证可见性与调用 → 跟踪调用结果

| 阶段 | 核心结果 |
| --- | --- |
| 1. 准备材料 | 应用说明、绑定模型、调用入口和客户范围完整 |
| 2. 执行审核 | 审核结论、意见、处理人和时间已记录 |
| 3. 完成发布 | 发布状态和目标可见范围正确 |
| 4. 验证结果 | 客户调用可用，失败原因可追溯 |

## 开始前准备

- 具备应用发布管理或应用审核权限。
- 已准备应用说明、绑定模型、调用入口和客户范围。
- 已确认绑定模型状态及客户授权边界。

## 推荐阅读顺序

1. 应用列表核对发布信息。
2. 应用审核检查材料和范围。
3. 客户视角验证可见性和调用入口。
4. 使用调用日志继续排查异常。

## 完成检查

> **用途：** 以下检查是本场景的退出条件，用于确认应用只对目标客户可见，并具备可用、可追溯的调用路径。

| 检查项 | 通过标准 |
| --- | --- |
| 1 | 应用说明、绑定模型、调用入口和客户范围完整。 |
| 2 | 审核状态、意见、处理人和时间可追溯。 |
| 3 | 应用只对目标客户可见。 |
| 4 | 客户侧调用入口可用，异常可从日志定位。 |

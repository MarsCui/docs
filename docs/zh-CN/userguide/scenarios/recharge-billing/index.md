# 场景概览 - 充值与计费

本场景帮助用户区分充值记录、资源配额、资源用量和模型计费，并核对额度是否到账、资源是否可用以及消耗为何产生。

## 适用角色

- 平台用户、模型提供方、平台运营方

## 场景目标

- 确认充值记录和账户额度变化。
- 区分资源配额不足与账户额度不足。
- 将扣减记录对应到实例、作业或模型调用。

## 场景流程

**主线：** 获得可用额度 → 确认资源配额 → 产生并记录用量 → 核对扣减结果

| 阶段 | 核心结果 |
| --- | --- |
| 1. 获得额度 | 充值或分配记录形成可消费余额 |
| 2. 确认配额 | 租户可申请的算力和存储上限明确 |
| 3. 记录用量 | 实例、作业或模型调用产生可追溯明细 |
| 4. 核对扣减 | 余额、额度、用量、价格和账期汇总相互一致 |

## 开始前准备

- 明确要核对模型调用还是 本地算力平台资源使用。
- 明确租户、账号、时间范围、账期和计费单位。

## 推荐阅读顺序

1. 查看充值记录
2. 查看资源配额和账户额度
3. 查看资源或模型用量
4. 按账期核对扣减明细

## 完成检查

> **用途：** 以下检查是本场景的退出条件，用于判断是否已经取得可观察、可复核的结果，以及是否可以进入下一场景。它不是操作步骤的重复；任一项不满足时，请返回对应功能文档的“常见失败分支”排查。

| 检查项 | 通过标准 |
| --- | --- |
| 1 | 充值或额度分配记录已生效，可消费余额按预期变化。 |
| 2 | 租户配额支持目标工作负载，实例、作业或模型调用产生可追溯用量。 |
| 3 | 同一账期内的余额、额度、用量、价格和扣减结果能够相互对应。 |

## 功能概述

`AI Infra On-Cloud` 是 AGIOne 面向多云资源的接入、授权、调度和模型部署能力。它把云平台、云账号、资源池、业务授权、模型库、推理框架、推理镜像和调度策略串成一条完整链路：运营管理员先接入和治理多云资源，普通用户再基于已开放的云资源快速部署模型服务，并在部署后查看 API、监控和事件。

| 项目 | 内容 |
| --- | --- |
| 适用角色 | 运营管理员、普通用户 |
| 推荐入口 | 本文档、[多云调度平台概览](../)、[从零开始部署云上模型服务](../end-to-end/deploy-cloud-model-service/) |
| 关键对象 | 云平台、云账号、资源池、租户授权、业务授权、模型库、推理框架、推理镜像、调度策略、部署任务 |
| 典型用途 | 建立 On-Cloud 资源理解框架，明确运营方与用户的操作边界，并选择正确阅读路径 |

## 新手理解

AI Infra On-Cloud 像多云资源接入手册：先把云账号、资源池和授权关系理顺，再让用户基于被授权资源部署模型服务。

## 前提条件

1. 已明确当前账号属于运营管理员还是普通用户。
2. 运营管理员应准备云账号授权、资源池规划、部署资产和调度策略。
3. 普通用户应确认自己具备接入账号、地域资源和模型服务部署权限。
4. 所有截图、示例和工单中不得暴露 AK/SK、API Key、Endpoint、账号 ID 或内部资源编号。

## AGIOne / On-Cloud 是什么

AGIOne 是面向模型服务、算力资源和运营治理的一体化平台。On-Cloud 指平台纳管公有云或第三方云资源后，为模型部署提供跨云资源选择、智能推荐、部署任务管理和调用观测能力的场景。

在 On-Cloud 场景中，平台负责：

1. 记录可接入的云平台和资源边界，例如云厂商、地域和资源池。
2. 维护可部署能力，例如云账号、推理框架、推理镜像、模型库和调度策略。
3. 管理用户侧部署入口，例如快速部署、我的部署、API 调用和监控事件。
4. 通过授权关系控制租户、业务和用户能使用哪些云平台与资源池。

| 角色 | 主要职责 | 常用章节 |
| --- | --- | --- |
| 运营管理员 | 接入云平台、云账号和资源池；配置租户与业务授权；维护模型库、框架、镜像和调度策略。 | [快速接入](../operator/access-workbench/quick-start/)、[接入云平台](../operator/access-management/access-cloudtype/)、[模型库](../operator/deploy-assets/models/) |
| 普通用户 | 添加或使用已授权云账号；从模型库选择模型；基于智能推荐发起部署；查看部署详情、API、监控和事件。 | [接入账号](../user/access-management/access-accounts/)、[快速部署](../user/model-services/quick-deployment/)、[我的部署](../user/model-services/my-deployments/) |

运营管理员决定“哪些云资源和模型资产可以部署”，普通用户决定“把哪个模型部署到哪个已授权云资源上”。如果用户侧看不到某个模型、云账号或地域，通常需要从账号接入、租户授权、业务授权、资源池状态和部署资产几个方向排查。

## 资源与授权层级

| 层级 | 说明 | 影响范围 |
| --- | --- | --- |
| 云平台 | 可接入的云厂商或云服务类型。 | 影响云账号、资源池和模型部署点的归属。 |
| 云账号 | 访问目标云平台资源的账号凭据或账号实体。 | 影响资源同步、模型部署和费用归属。 |
| 资源池 | 云平台下可启用的地域或资源集合。 | 影响用户可部署的地域、规格和库存范围。 |
| 租户-云平台授权 | 控制租户是否可以使用目标云平台。 | 影响租户下用户能否看到或使用某云平台。 |
| 业务-资源池授权 | 控制业务或用户范围能使用哪些资源池。 | 影响快速部署中的可部署范围。 |
| 部署资产 | 模型库、推理框架和推理镜像等可部署基础数据。 | 影响模型是否能被选择、推荐和发起部署。 |
| 部署任务 | 用户提交的云上模型服务部署实例。 | 产生状态、API 调用、监控和事件数据。 |

推荐配置顺序是：先接入云平台，再添加云账号和启用资源池，然后配置租户与业务授权，之后维护模型库、框架、镜像和策略，最后开放快速部署给用户。

## 用户侧与运营方侧功能边界

| 能力 | 运营管理员 | 普通用户 |
| --- | --- | --- |
| 云平台 | 新增、编辑、启用、禁用云平台类型。 | 选择已授权云平台作为部署范围。 |
| 云账号 | 接入和管理运营侧云账号。 | 添加或使用个人可用的接入账号。 |
| 资源池 | 启用、禁用资源池地域，维护显示名称。 | 在快速部署中选择已授权资源池。 |
| 授权 | 配置租户-云平台授权和业务-资源池授权。 | 通过可见范围感知授权结果。 |
| 部署资产 | 维护模型库、推理框架和推理镜像。 | 从可部署模型中选择目标模型。 |
| 调度策略 | 维护策略管理和候选排序规则。 | 在快速部署中选择业务策略并查看推荐结果。 |
| 部署任务 | 查看接入与资源治理结果。 | 发起部署、启停部署、发布部署、查看 API、监控和事件。 |

| 术语 | 说明 |
| --- | --- |
| 云平台 | 被 AGIOne 接入的云厂商或云资源平台。 |
| 接入账号 | 用于访问云平台资源的账号配置，可能包含 AK/SK 等敏感凭据。 |
| 资源池 | 云平台下可用于部署的地域、算力、规格和库存能力集合。 |
| 租户授权 | 将云平台能力开放给指定租户的授权关系。 |
| 业务授权 | 将资源池能力开放给指定业务或用户范围的授权关系。 |
| 模型库 | 运营方维护的可部署模型资产集合。 |
| 推理框架 | 模型运行所需的推理引擎或服务框架，例如 vLLM、SGLang。 |
| 推理镜像 | 部署模型服务时使用的运行时镜像。 |
| 业务策略 | 快速部署中影响候选资源排序的策略，例如经济适用、卓越性能或现货速达。 |
| 部署任务 | 用户发起的一次云上模型部署记录。 |

## 常见问题

### 快速部署看不到目标云账号或地域

**问题现象：**

用户进入快速部署页面后，目标云账号、云平台或地域不在可部署范围中。

**可能原因：**

- 用户尚未添加接入账号，或账号状态异常。
- 运营管理员未启用对应云平台或资源池。
- 租户-云平台授权或业务-资源池授权未配置。
- 页面筛选条件限制了云平台、地域或模型范围。

**处理方式：**

1. 用户先检查 [接入账号](../user/access-management/access-accounts/) 是否存在且可用。
2. 请运营管理员检查云平台、云账号和资源池状态。
3. 检查租户授权和业务授权是否覆盖当前用户或业务。
4. 清空快速部署筛选条件后刷新资源。

### 部署任务提交后长时间无结果

**问题现象：**

部署任务提交后长期处于创建中、部署中或无可用 API 状态。

**可能原因：**

- 目标云资源库存不足或规格不可用。
- 推理框架、镜像或模型部署点配置不完整。
- 云账号权限不足或云平台接口调用失败。
- 调度策略筛选范围过窄。

**处理方式：**

1. 在 [我的部署](../user/model-services/my-deployments/) 查看部署详情、事件记录和监控信息。
2. 尝试切换推理引擎、业务策略或地域后重新部署。
3. 请运营管理员检查模型库、推理框架、推理镜像和资源池配置。
4. 检查云账号权限和云平台侧资源状态。

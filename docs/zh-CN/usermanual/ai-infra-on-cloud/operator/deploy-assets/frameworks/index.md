# 框架资产

## 功能概述

`框架资产` 用于维护运行框架、框架版本、启动命令、端口和镜像关系，支撑多云调度、资源授权和模型部署流程。

| 项目 | 内容 |
| --- | --- |
| 适用角色 | 运营方 |
| 导航路径 | 部署资产 > 框架 |
| 页面路由 | /operator/deploy-assets/frameworks |
| 管理对象 | 运行框架、框架版本、启动命令、端口和镜像关系 |
| 典型用途 | 维护模型部署时使用的运行框架 |

### 新手理解

部署框架像模型服务的启动器，定义模型怎么启动、监听哪个端口、需要哪些环境变量，以及和运行镜像、模型资产如何配合。

### 术语速查

| 术语 | 说明 |
| --- | --- |
| 运行框架 | 承载模型推理服务的框架或服务进程。 |
| 启动命令 | 容器启动后加载模型并启动服务的命令。 |
| 服务端口 | 模型服务监听端口。 |
| 框架版本 | 框架能力和兼容性的版本标识。 |

## 前提条件

1. 目标运行框架、启动命令和服务端口已确认。
2. 兼容运行镜像和资源规格已准备。
3. 健康检查和环境变量要求已确认。
## 页面说明

页面用于维护云上部署框架，包括框架类型、启动命令、服务端口、健康检查、默认环境变量和兼容模型。运营方应保证框架与运行镜像、模型资产和云资源规格匹配。

## 主要操作

### 操作步骤

1. 进入 `部署资产 > 部署框架`。
2. 按框架类型、启用状态或关键字筛选记录。
3. 新增框架时填写名称、启动命令、端口和健康检查。
4. 关联可兼容的运行镜像、模型类型或资源规格。
5. 保存后用测试模型创建部署验证启动结果。

关键步骤截图：

![新增推理框架](./images/add-framework.png)

新增时核对启动命令、端口和健康检查。

### 参数说明

| 字段名称 | 是否必填 | 字段类型 | 示例 | 说明 |
| --- | --- | --- | --- | --- |
| 框架名称 | 是 | 文本 | `vllm-openai` | 部署页展示的框架名称。 |
| 启动命令 | 是 | 文本 | `python -m vllm.entrypoints.openai.api_server` | 容器启动模型服务的命令。 |
| 服务端口 | 是 | 数字 | `8000` | 模型服务监听端口。 |
| 健康检查路径 | 否 | 文本 | `/health` | 用于判断服务是否就绪。 |
| 兼容镜像 | 条件必填 | 多选 | `vllm-runtime:latest` | 可配合该框架使用的运行镜像。 |

### 踩坑提示

- 启动命令不要硬编码真实模型路径、密钥或内部下载地址。
- 端口与运行镜像暴露端口不一致会导致部署成功但服务不可访问。
- 框架变更会影响后续新部署，已运行实例是否受影响需按平台机制确认。

### 结果校验

1. 框架记录处于启用状态。
2. 部署页能选择该框架和兼容镜像。
3. 测试部署事件显示服务健康检查通过。

## 常见问题

### 框架启动失败

**问题现象：**

部署实例创建后进入失败或反复重启。

**可能原因：**

- 启动命令参数错误。
- 镜像缺少框架依赖。
- 模型路径或环境变量不匹配。

**处理方式：**

1. 查看部署事件和容器日志。
2. 核对启动命令、端口和环境变量。
3. 使用兼容镜像重新验证。

### 框架在部署页不可选

**问题现象：**

框架已维护，但用户创建部署时看不到。

**可能原因：**

- 框架未启用。
- 未关联当前模型类型或运行镜像。
- 用户所在业务地域没有对应部署资产权限。

**处理方式：**

1. 确认框架启用状态。
2. 补齐模型类型和镜像关联。
3. 核对业务地域和租户授权。

## 后续操作

1. 维护运行镜像。
2. 关联模型资产。
3. 创建测试部署验证框架可用。

## 注意事项

- 启动命令不要硬编码密钥或内部路径。
- 端口和镜像暴露端口需一致。
- 框架变更后用测试部署验证。

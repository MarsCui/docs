# 检查和维护 License 生命周期

本任务用于确认 License 当前状态，完成必要激活，并建立有效期和额度巡检。

## 适用角色

- 平台运营方、License 管理员、资源运营人员

## 开始前准备

1. 确认当前环境、实例和目标授权类型。
2. 记录现有状态、有效期、总额度、已用和剩余额度。
3. 选择在线支付或注册码/激活码方式，并准备相应审批。
4. 注册码、激活码和支付凭据不得出现在文档、截图或工单中。

## 操作流程

### 1. 检查当前 License 状态

进入[License 管理](../../../usermanual/billing/operator/license/license/)，核对当前状态、有效期、总额度、已用额度、剩余额度和纳管对象。先确认当前页面属于目标环境和实例。

![查看 License 状态和纳管对象](../../../usermanual/billing/operator/license/license/images/license-managed-objects.png)

### 2. 选择并执行激活方式

- 具备在线购买条件时，参考[在线支付激活](../../../license/online-payment-activation)。
- 需要线下获取激活码或在线激活不可用时，参考[激活码激活](../../../license/activation-code-activation)。

提交前再次核对环境、实例、注册码和授权类型。激活失败时先检查状态和错误信息，不重复提交未知激活材料。

### 3. 核对详情和授权构成

激活后刷新 License 管理页面，打开详情和授权构成，确认授权类型、生效时间、到期时间、总额度和各类资源占用。

![核对 License 授权构成](../../../usermanual/billing/operator/license/license/images/license-composition.png)

### 4. 建立有效期和额度巡检

定期检查剩余有效期、剩余额度和纳管对象变化。根据资源扩容计划设置内部预警时间；预计不足时提前准备续期或扩容，并评估现有实例和新建任务的影响。

## 完成检查

> **用途：** 以下检查确认 License 不仅被激活，而且在正确环境提供了可解释、可持续的授权容量。

| 检查项 | 通过标准 |
| --- | --- |
| 实例匹配 | 环境、实例、注册码/激活材料和授权类型一致。 |
| 状态有效 | 页面显示有效状态和正确有效期。 |
| 额度清晰 | 总额度、已用、剩余和授权构成可解释。 |
| 对象一致 | 纳管对象与实际资源和业务计划一致。 |
| 后续计划 | 到期、扩容和额度不足的责任人及处理时间明确。 |

## 常见失败分支

| 现象 | 优先检查 |
| --- | --- |
| 找不到激活入口或注册码 | 账号权限、授权区域、页面加载和实例状态 |
| 激活码无效 | 是否匹配当前注册码、授权类型、有效期及复制完整性 |
| 激活后状态未更新 | 页面缓存、同步延迟和当前授权区域 |
| 剩余额度异常 | 纳管对象、授权构成、历史占用和资源同步 |
| License 即将到期或不足 | 业务扩容计划、续期周期、受影响实例和审批进度 |

# 接入账号

本任务把平台运营方管理的云账号接入 AGIOne，用于同步可部署的地域、规格和模型资源。

## 场景目标

账号校验成功，AGIOne 只能发现该账号被授权使用的云资源。

## 适用角色

- 平台运营方

## 开始前准备

- 云平台已经接入并处于可用状态。
- 已准备专用于 AGIOne 的最小权限云凭证，并确认凭证有效期。

## 操作步骤

### 添加云账号

1. 进入平台首页，点击左侧导航栏的 **"接入管理 > 接入账号"** 菜单，进入接入账号管理页面。
2. 点击页面右上角的 **"添加云账号"** 按钮，弹出「新增账号」窗口。
![接入账号](./images/cloud-accounts-list.png)
3. 配置账号信息：
   - 填写 **"账号名称"**，用于标识该云账号；
   - 从下拉列表中选择 **"选择云平台"**（如阿里云、亚马逊、华为云等）；
   - 输入目标云平台的 **"Access Key ID"**；
   - 输入目标云平台的 **"Access Key Secret"**。
5. 确认所有信息配置无误后，点击 **"确定"** 按钮完成添加；如需放弃，点击 **"取消"**。
![新增账号](./images/add-account.png)
#### 参数说明

| 字段名称 | 字段类型 | 示例 | 说明 |
|----------|----------|------|------|
| 账号名称 | 文本 | `aliyun-wh-dev` | 必填，自定义账号标识 |
| 选择云平台 | 下拉选择 | `阿里云` | 必填，选择目标云平台 |
| Access Key ID | 文本 | `LTAI5tM8xHnXoLuBW...` | 必填，云平台访问凭证 ID |
| Access Key Secret | 密码 | `flsBCIDPLksdaNh05J...` | 必填，云平台访问凭证密钥 |

## 完成检查

> **用途：** 以下检查是当前功能任务的退出条件，用于判断操作结果是否可观察、可复核，以及是否可以继续当前场景的下一步。它不是操作步骤的重复；任一项不满足时，请按下方“常见失败分支”继续排查。

| 检查项 | 通过标准 |
| --- | --- |
| 1 | 账号校验成功。 |
| 2 | 只授予业务所需的最小云权限。 |
| 3 | 预期地域和资源同步成功。 |

## 常见失败分支

| 现象 | 优先检查 |
| --- | --- |
| 账号校验失败 | 凭证状态、云权限、时间同步和网络访问 |
| 地域或资源缺失 | 云权限范围、账号地域范围和同步状态 |
